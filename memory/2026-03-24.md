## 📚 第13次学习（13:00）

### 一、技能学习：compass-analysis（数据分析）

**技能定位**：指南针工程步骤8 - 数据分析技能（知微版）

**核心能力**：
- 数据分析：描述性分析（发生了什么）、诊断性分析（为什么发生）、预测性分析（将发生什么）
- 用户画像：标签体系、RFM模型、用户分群方法
- 可视化设计：图表设计、BI报表、数据看板
- 流失预测：预警模型、策略建议

**方法论掌握**：
1. **RFM模型**
   - R (Recency): 最近一次消费时间
   - F (Frequency): 消费频率
   - M (Monetary): 消费金额
   - 用户价值分层：高价值(R↑F↑M↑)、潜力(R↓F↑M↑)、流失预警(R↓F↓M↓)

2. **电信业务指标体系**
   - 收入指标：ARPU、收入增长率、收入构成
   - 用户指标：用户规模、新增用户、离网率、活跃率
   - 业务指标：DOU（户均流量）、MOU（户均通话时长）、套餐渗透率

3. **流失预测模型**
   - 预警指标：话费余额<10元、最近充值>30天、通话次数下降>50%、流量使用下降>50%
   - 风险等级：中风险⚠️、高风险🔴

**分析流程**：
```
问题定义 → 数据获取 → 数据分析 → 可视化展示 → 报告输出
```

**应用思考**：
- 可用于湖北电信用户流失预警系统
- 可用于智能配案效果评估（指南针工程步骤8）
- 与知微协作，为需求分析、售前方案提供数据支撑

### 三、少帅智慧学习：提示词工程框架

**核心方法论**：

1. **CRISPE框架**
   - C (Capacity): 能力定位
   - R (Role): 角色定义
   - I (Insight): 洞察视角
   - S (Statement): 任务陈述
   - P (Personality): 人格特质
   - E (Experiment): 实验迭代

2. **四层结构模型**
   - Layer 1: 身份定义（Who）- 你是谁、你的背景、你的专长
   - Layer 2: 任务目标（What）- 你要做什么、达到什么目标
   - Layer 3: 方法规范（How）- 你用什么方法、遵循什么标准
   - Layer 4: 输出要求（Output）- 输出格式、质量标准、约束条件

3. **角色定义六要素**
   - 身份定位：名称、代号、领域
   - 背景设定：行业经验、专业背景、核心能力
   - 专业方法论：核心方法、分析框架、工具技术
   - 思维模式：分析视角、决策逻辑、优先级判断
   - 沟通风格：语言特点、表达习惯、互动方式
   - 约束条件：不做什么、必须做什么、质量标准

**关键发现**：
- 提示词质量决定AI输出质量
- 角色定义要清晰：做什么、不做什么、何时转交
- 持续优化循环：测试 → 识别问题 → 针对性优化 → 再次测试

**应用实践**：
- 九星智囊团角色定义已应用此框架
- 采薇、织锦等Agent提示词优化后达到A级
- 每个角色都有明确的边界和协作关系

**提示词优化技巧**：
| 技巧 | 说明 |
|------|------|
| Few-shot示例 | 提供参考示例 |
| 思维链 | 引导分步思考 |
| 角色强化 | 强调专业视角 |
| 约束明确 | 设置边界条件 |
| 格式指定 | 明确输出格式 |
