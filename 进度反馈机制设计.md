# 执行进度反馈机制设计

## 问题

用户无法看到AI执行指令的过程，导致误以为卡死。

## 解决方案

### 方案1：进度日志文件

创建一个进度日志文件，用户可以通过命令查看：

```bash
# 查看最新进度
tail -f /root/.openclaw/workspace/progress.log
```

### 方案2：进度状态文件

创建JSON格式的状态文件，便于程序化读取：

```json
{
  "timestamp": "2026-03-13 17:15:00",
  "status": "running",
  "task": "生成需求文档",
  "progress": "60%",
  "message": "正在进行多维度分析..."
}
```

### 方案3：实时消息推送

每完成一个步骤，主动发送消息通知用户：

```
✅ 步骤1完成：智能解析需求
✅ 步骤2完成：多维度分析
🔄 步骤3进行中：风险识别...
```

## 推荐方案

**方案3最直观**，但会发送多条消息。

**方案1+2组合**：创建日志文件 + 在关键节点发送简要消息。

## 实施代码

```python
class ProgressTracker:
    def __init__(self, log_file="/root/.openclaw/workspace/progress.log"):
        self.log_file = log_file

    def update(self, task, status, progress, message):
        log_entry = {
            "timestamp": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
            "task": task,
            "status": status,
            "progress": progress,
            "message": message
        }

        # 写入日志文件
        with open(self.log_file, 'a') as f:
            f.write(json.dumps(log_entry, ensure_ascii=False) + "\n")

        # 返回消息（可选发送给用户）
        return f"🔄 {task}: {message} ({progress})"
```

## 使用示例

```python
tracker = ProgressTracker()

tracker.update("生成需求文档", "running", "20%", "正在智能解析需求...")
# 执行解析...

tracker.update("生成需求文档", "running", "40%", "正在进行多维度分析...")
# 执行分析...

tracker.update("生成需求文档", "complete", "100%", "文档生成完成")
```
