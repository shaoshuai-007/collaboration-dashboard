## 二、Agent SDK框架

### 主流框架对比

| 框架 | 特点 | 适用场景 | 复杂度 |
|-----|------|---------|-------|
| **LangChain** | 模块化、生态丰富、文档完善 | 通用Agent开发 | 中等 |

| **CrewAI** | 角色扮演、任务分工清晰 | 模拟团队工作流 | 中等 |
| **LlamaIndex** | RAG专家、数据检索强 | 知识库Agent | 中等 |
| **Semantic Kernel** | 微软生态、企业集成 | 企业应用 | 较高 |

### 框架选型决策树

```
需要多Agent协作？
├── 是 → AutoGen 或 CrewAI
└── 否 → 需要RAG能力？
    ├── 是 → LlamaIndex
    └── 否 → LangChain (最通用)
```

## 三、代码生成技术

### 核心技术栈

1. **Prompt Engineering（提示工程）**
   - Chain-of-Thought（思维链）
   - Few-shot Learning（少样本学习）
   - Self-Consistency（自一致性）

2. **RAG（检索增强生成）**
   - 向量数据库：Pinecone、Milvus、Chroma
   - Embedding模型：OpenAI、BGE、M3E
   - 分块策略：语义分块、滑动窗口

3. **Fine-tuning（微调）**
   - LoRA/QLoRA（低秩适配）
   - 领域数据集构建
   - 评估与迭代

## 五、代码示例

### 示例1：使用LangChain构建简单Agent

```python
from langchain.agents import create_openai_functions_agent, AgentExecutor
from langchain.tools import Tool
from langchain_openai import ChatOpenAI

# 定义工具
def calculate(expression: str) -> str:
    """执行数学计算"""
    return str(eval(expression))

tools = [
    Tool(
        name="calculator",
        func=calculate,
        description="用于数学计算，输入数学表达式"
    )
]

# 创建Agent
llm = ChatOpenAI(model="gpt-4", temperature=0)
agent = create_openai_functions_agent(llm, tools)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 执行任务
result = agent_executor.invoke({
    "input": "计算 (25 + 17) * 3 的结果"
})
print(result["output"])  # 输出: 126
```

### 示例2：AutoGen多Agent协作

```python
import autogen

# 配置LLM
config_list = [{
    "model": "gpt-4",
    "api_key": "your-api-key"
}]

# 创建Agent
assistant = autogen.AssistantAgent(
    name="assistant",
    llm_config={"config_list": config_list}
)

user_proxy = autogen.UserProxyAgent(
    name="user_proxy",
    human_input_mode="NEVER",
    max_consecutive_auto_reply=10,
    code_execution_config={"work_dir": "coding"}
)

# 发起对话
user_proxy.initiate_chat(
    assistant,
    message="写一个Python脚本，爬取指定网页的所有链接"
)
```

## 六、关键学习要点

### 核心公式

```
AI编程效率 = 工具选择 × 提示质量 × 代码组织

其中：
- 工具选择：根据项目需求选择合适的工具
- 提示质量：清晰、具体、结构化的提示词
- 代码组织：模块化、类型化、文档化
```

### 避坑指南

1. **不要盲目信任AI生成的代码** → 必须测试验证
2. **不要忽视安全性** → 敏感信息不要输入AI
3. **不要过度依赖** → 理解原理比复制代码更重要
4. **不要忽视代码审查** → AI也可能引入Bug
