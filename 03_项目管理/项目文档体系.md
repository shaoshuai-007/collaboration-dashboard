## 一、体系架构

```
项目文档体系
├── 01_项目档案/           # 项目基本信息
├── 02_参考文档/           # 用户提供的参考资料
├── 03_工程输出/           # 各工程产出的文档
│   ├── 指南针工程/        # 指南针工程输出
│   ├── 美学工坊/          # 美学工坊输出
│   └── 智汇工程/          # 智汇工程输出
├── 04_会议记录/           # 项目会议纪要
├── 05_进度跟踪/           # 项目进度管理
└── 06_复盘沉淀/           # 项目复盘与知识沉淀
```

## 三、文档命名规范

### 3.1 文件命名

```
[类型]_[名称]_[日期].md

类型：
- 分析：问题分析、需求分析
- 方案：方案文档
- 会议：会议纪要
- 参考：参考资料
- 模板：模板文件
```

### 3.2 版本管理

```
文档_v1.0.md    # 初始版本
文档_v1.1.md    # 小幅修订
文档_v2.0.md    # 大幅修订
文档_终稿.md    # 最终版本
```

## 六、文档质量标准

### 6.1 文档规范

| 要素 | 标准 |
|------|------|
| 标题层级 | 最多三级标题，层级清晰 |
| 列表编号 | 有序列表用数字，无序列表用`-` |
| 表格格式 | 表头加粗，对齐整齐 |
| 代码块 | 指定语言，格式规范 |

### 6.2 审核流程

```
创建文档 → 自检 → 提交审核 → 少帅审核 → 归档
```
