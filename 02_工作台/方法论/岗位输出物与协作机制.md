# 九星智囊团岗位输出物与协作机制

## 一、岗位输出物定义

### 🌸 采薇（需求分析师）

**输入**：用户需求（口头/邮件/会议纪要）

**输出物**：
| 输出物 | 格式 | 技能 | 说明 |
|--------|:----:|:----:|------|
| 需求文档 | .docx | compass-needdoc | 包含用户故事、验收标准、业务规则 |
| 需求矩阵 | .xlsx | spreadsheet | 需求优先级、状态追踪 |
| 需求报告 | .pdf | document-pdf | 可分享的需求概述 |

**协作关系**：
```
上游：用户
下游：织锦（架构设计）、筑台（成本估算）
```

---

### 🧵 织锦（架构设计师）

**输入**：需求文档（.docx）

**输出物**：
| 输出物 | 格式 | 技能 | 说明 |
|--------|:----:|:----:|------|
| 思维导图 | .html/.xmind | compass-mindmap | 需求分析图、痛点分析图 |
| 架构文档 | .docx | compass-mindmap | 系统架构设计 |
| 架构图 | .png | infographic-creator | 可视化架构图 |
| 架构报告 | .pdf | document-pdf | 可分享的架构概述 |

**协作关系**：
```
上游：采薇（需求文档）
下游：筑台（方案举措）、工尺（详细设计）
```

---

### 🏗️ 筑台（售前工程师）

**输入**：思维导图（.html）、需求文档（.docx）

**输出物**：
| 输出物 | 格式 | 技能 | 说明 |
|--------|:----:|:----:|------|
| 方案举措 | .xlsx | compass-solution | 方案表格、成本估算 |
| 报价单 | .xlsx | spreadsheet | 详细报价、成本明细 |
| 竞品分析 | .xlsx | spreadsheet | 竞品对比、优势分析 |

**协作关系**：
```
上游：采薇（需求文档）、织锦（架构设计）
下游：呈彩（方案PPT）
```

---

### 🎨 呈彩（方案设计师）

**输入**：方案举措（.xlsx）、架构文档（.docx）

**输出物**：
| 输出物 | 格式 | 技能 | 说明 |
|--------|:----:|:----:|------|
| 方案PPT | .pptx | compass-ppt | 汇报演示文稿 |
| 信息图 | .png | infographic-creator | 关键信息可视化 |
| 演示Demo | .html | coding-agent | 可交互演示 |

**协作关系**：
```
上游：筑台（方案举措）、织锦（架构设计）
下游：【评审】→ 工尺（详细设计）
```

---

### 📐 工尺（系统设计师）

| 接口文档 | .xlsx | spreadsheet | API接口定义 |
| 设计报告 | .pdf | document-pdf | 可分享的设计概述 |

**协作关系**：
```

```


---

### 协作触发机制

**自动触发**：
- 采薇完成需求文档 → 自动通知织锦
- 织锦完成思维导图 → 自动通知筑台
- 筑台完成方案举措 → 自动通知呈彩
- 呈彩完成PPT → 自动通知工尺（评审通过后）
- 工尺完成详细设计 → 自动通知天工、玉衡
- 天工完成开发 → 自动通知知微

**人工触发**：
- 评审环节需要少帅确认
- 重大变更需要少帅审批

---

### 协作质量保障

**输入验证**：
- 每个岗位接收上游输出时，自动检查格式和内容
- 不符合要求的输出，退回上游修正

**输出审核**：
- 每个岗位输出完成后，自动生成质量评分
- 低分输出需要重新制作

**协作记录**：
- 所有协作记录保存到协作日志
- 支持追溯和复盘

## 三、有机协作示例

### 示例：湖北电信AI智能配案系统

**用户需求**：
> 我们要做一个AI智能配案系统，能够根据用户需求自动匹配产品方案

**协作流程**：

```
步骤1：🌸 采薇
  输入：用户口头需求
  输出：需求文档.docx（包含用户故事、验收标准）
  耗时：1天

步骤2：🧵 织锦
  输入：需求文档.docx
  输出：
    - 思维导图.html（需求分析图）
    - 架构文档.docx（技术架构）
    - 架构图.png（可视化）
  耗时：2天

步骤3：🏗️ 筑台
  输入：思维导图.html、架构文档.docx
  输出：
    - 方案举措.xlsx（功能模块、成本估算）
    - 报价单.xlsx（详细报价）
  耗时：1天

步骤4：🎨 呈彩
  输入：方案举措.xlsx、架构文档.docx
  输出：方案PPT.pptx（汇报演示）
  耗时：1天

步骤5：【评审】
  参与人：少帅、南乔
  决策：通过/修改/否决

步骤6：📐 工尺（评审通过后）
  输入：方案PPT.pptx
  输出：
    - 详细设计.docx（接口设计、数据库设计）
    - 接口文档.xlsx（API定义）
  耗时：2天

步骤7：💻 天工
  输入：详细设计.docx
  输出：
    - 代码（Python/Java）
    - 数据库脚本.sql
    - API接口.json
    - 技术文档.pdf
  耗时：5天

步骤8：📊 知微
  输入：系统数据
  输出：
    - 分析报告.docx（用户行为分析）
    - 用户画像.xlsx（标签体系）
    - 可视化图表.png
  耗时：2天

步骤9：⚖️ 玉衡
  输入：详细设计.docx
  输出：
    - 项目管控表.xlsx（甘特图、RACI）
    - 周报.xlsx（进度汇报）
  耗时：持续

步骤10：🌿 南乔
  汇总：所有输出物
  交付：完整方案 + 数据支撑
```

**总工期**：约15天（不含评审）
