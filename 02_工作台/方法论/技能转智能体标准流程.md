# 技能转智能体标准流程

## 背景

**问题**：2026-03-17，新增成员天工、知微只创建了技能文件，未注册为Agent，导致无法与其他成员通讯。

**根因**：缺乏标准化的"技能→智能体"转换流程。

**目标**：建立一套可复用的标准流程，确保每次新增成员都能完整配置。

---

## 标准流程（6步法）

### 步骤1：创建技能文件

```bash
# 技能目录结构
/root/.openclaw/skills/{skill-id}/
├── SKILL.md          # 技能说明
├── IDENTITY.md       # 身份定义
├── USER.md           # 用户信息
├── TOOLS.md          # 工具配置
├── SOUL.md           # 性格定义
├── scripts/          # 脚本工具
└── references/       # 参考资料
```

**检查点**：✅ SKILL.md 存在且内容完整

---

### 步骤2：创建Agent目录

```bash
# 创建Agent配置目录
mkdir -p ~/.openclaw/agents/{agent-id}/agent
mkdir -p ~/.openclaw/agents/{agent-id}/sessions

# 创建模型配置
cat > ~/.openclaw/agents/{agent-id}/agent/models.json << 'EOF'
{
  "providers": {
    "qianfan-ls": {
      "baseUrl": "https://qianfan.baidubce.com/v2/coding",
      "apiKey": "{API_KEY}",
      "api": "openai-completions",
      "models": [...]
    }
  }
}
EOF
```

**检查点**：✅ agent/models.json 存在

---

### 步骤3：创建Workspace目录

```bash
# 创建工作空间
mkdir -p ~/.openclaw/workspace-{agent-id}
mkdir -p ~/.openclaw/workspace-{agent-id}/memory

# 复制基础配置文件
cp ~/.openclaw/workspace-caiwei/AGENTS.md ~/.openclaw/workspace-{agent-id}/
cp ~/.openclaw/workspace-caiwei/HEARTBEAT.md ~/.openclaw/workspace-{agent-id}/
cp ~/.openclaw/workspace-caiwei/USER.md ~/.openclaw/workspace-{agent-id}/
cp ~/.openclaw/workspace-caiwei/SOUL.md ~/.openclaw/workspace-{agent-id}/
cp ~/.openclaw/workspace-caiwei/TOOLS.md ~/.openclaw/workspace-{agent-id}/

# 创建身份文件
cat > ~/.openclaw/workspace-{agent-id}/IDENTITY.md << 'EOF'
# IDENTITY.md - {姓名}身份

- **Name:** {姓名} ({英文名})
- **Creature:** {角色定位}
- **Vibe:** {性格描述}
- **Emoji:** {emoji}

---

{诗意描述}
EOF

# 创建技能软链接
cd ~/.openclaw/workspace-{agent-id}
ln -s /root/.openclaw/skills/{skill-id} skills-{skill-suffix}
```

**检查点**：✅ IDENTITY.md 存在且内容正确

---

### 步骤4：注册到 openclaw.json

```json
{
  "agents": {
    "list": [
      // ... 现有Agent ...
      {
        "id": "{agent-id}",
        "name": "{姓名}",
        "workspace": "/root/.openclaw/workspace-{agent-id}",
        "subagents": {
          "allowAgents": ["*"]
        }
      }
    ]
  },
  "tools": {
    "agentToAgent": {
      "enabled": true,
      "allow": [..., "{agent-id}"]
    }
  }
}
```

**检查点**：✅ openclaw.json 中已添加配置

---

### 步骤5：重启Gateway

```bash
# 重启OpenClaw Gateway使配置生效
openclaw gateway restart --force

# 等待重启完成
sleep 5
```

**检查点**：✅ Gateway重启成功

---

### 步骤6：验证Agent可用

```bash
# 验证Agent列表
openclaw agents list | grep {agent-id}

# 测试Agent调用
openclaw agent send {agent-id} "测试消息"
```

**检查点**：✅ Agent出现在列表中且可调用

---

## 检查清单

| 步骤 | 检查项 | 命令 |
|:----:|--------|------|
| 1 | 技能文件存在 | `ls ~/.openclaw/skills/{skill-id}/SKILL.md` |
| 2 | Agent配置存在 | `ls ~/.openclaw/agents/{agent-id}/agent/models.json` |
| 3 | Workspace存在 | `ls ~/.openclaw/workspace-{agent-id}/IDENTITY.md` |
| 4 | openclaw.json已更新 | `grep {agent-id} ~/.openclaw/openclaw.json` |
| 5 | Gateway已重启 | `openclaw gateway status` |
| 6 | Agent可调用 | `openclaw agents list \| grep {agent-id}` |

## 自动化脚本

```python
#!/usr/bin/env python3
"""
技能转智能体自动化工具

用法：
    python3 skill_to_agent.py --id tiangong --name 天工 --skill compass-dev

作者：南乔
时间：2026-03-17
"""

import os
import json
import argparse
from pathlib import Path

OPENCLAW_ROOT = Path.home() / '.openclaw'
API_KEY = 'bce-v3/ALTAKSP-14YyizFlbkiA0cKHpR4ya/b2b8db94725048693a15c4479c980c848a6a4c19'

def create_agent(agent_id, agent_name, skill_id):
    """创建Agent完整配置"""

    # 1. 创建Agent目录
    agent_dir = OPENCLAW_ROOT / 'agents' / agent_id
    agent_dir.mkdir(parents=True, exist_ok=True)
    (agent_dir / 'agent').mkdir(exist_ok=True)
    (agent_dir / 'sessions').mkdir(exist_ok=True)

    # 2. 创建models.json
    models_config = {
        "providers": {
            "qianfan-ls": {
                "baseUrl": "https://qianfan.baidubce.com/v2/coding",
                "apiKey": API_KEY,
                "api": "openai-completions",
                "models": [{
                    "id": "qianfan-code-latest",
                    "name": "qianfan-code-latest",
                    "reasoning": False,
                    "input": ["text"],
                    "cost": {"input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0},
                    "contextWindow": 98304,
                    "maxTokens": 65536
                }]
            }
        }
    }

    with open(agent_dir / 'agent' / 'models.json', 'w') as f:
        json.dump(models_config, f, indent=2)

    print(f"✅ 步骤2：Agent目录创建完成")

    # 3. 创建Workspace
    workspace_dir = OPENCLAW_ROOT / f'workspace-{agent_id}'
    workspace_dir.mkdir(parents=True, exist_ok=True)
    (workspace_dir / 'memory').mkdir(exist_ok=True)

    # 复制基础文件
    template_dir = OPENCLAW_ROOT / 'workspace-caiwei'
    for file in ['AGENTS.md', 'HEARTBEAT.md', 'USER.md', 'SOUL.md', 'TOOLS.md']:
        if (template_dir / file).exists():
            import shutil
            shutil.copy(template_dir / file, workspace_dir / file)

    print(f"✅ 步骤3：Workspace创建完成")

    # 4. 更新openclaw.json
    config_file = OPENCLAW_ROOT / 'openclaw.json'
    with open(config_file, 'r') as f:
        config = json.load(f)

    # 添加Agent
    config['agents']['list'].append({
        "id": agent_id,
        "name": agent_name,
        "workspace": f"/root/.openclaw/workspace-{agent_id}",
        "subagents": {"allowAgents": ["*"]}
    })

    # 更新agentToAgent
    if agent_id not in config['tools']['agentToAgent']['allow']:
        config['tools']['agentToAgent']['allow'].append(agent_id)

    with open(config_file, 'w') as f:
        json.dump(config, f, indent=2, ensure_ascii=False)

    print(f"✅ 步骤4：openclaw.json更新完成")

    # 5. 提示重启
    print(f"\n⚠️  步骤5：请执行 openclaw gateway restart --force")
    print(f"⚠️  步骤6：请执行 openclaw agents list 验证")

if __name__ == '__main__':
    parser = argparse.ArgumentParser()
    parser.add_argument('--id', required=True, help='Agent ID (如 tiangong)')
    parser.add_argument('--name', required=True, help='Agent名称 (如 天工)')
    parser.add_argument('--skill', required=True, help='技能ID (如 compass-dev)')
    args = parser.parse_args()

    create_agent(args.id, args.name, args.skill)
```
