# 九星智囊团调度指南

## 一、调度流程

### 1.1 标准流程（指南针工程6步）

```
用户需求 → 南乔分析 → 扶摇调度
                ↓
        ┌───────┴───────┐
        ↓               ↓
    单任务            多任务
        ↓               ↓
    单Agent        多Agent协作
        ↓               ↓
    执行任务       依次执行
        ↓               ↓
    输出成果       输出成果
        ↓               ↓
    质量检查       质量检查
        ↓               ↓
    完成汇报       完成汇报
```

### 1.2 任务类型与调度

| 任务类型 | 主导角色 | 协作角色 | 调度方式 |
|----------|:--------:|----------|:--------:|
| 需求分析 | 🌸 采薇 | 折桂、知微 | 手动/自动 |
| 架构设计 | 🧵 织锦 | 天工、工尺 | 手动/自动 |
| 方案举措 | 🏗️ 筑台 | 知微、采薇 | 手动/自动 |
| 方案PPT | 🎨 呈彩 | 筑台、织锦 | 手动/自动 |
| 详细设计 | 📐 工尺 | 织锦、天工 | 手动/自动 |
| 项目管控 | ⚖️ 玉衡 | 全体 | 手动/自动 |
| 开发实现 | 💻 天工 | 工尺、织锦 | 手动/coding-agent |
| 数据分析 | 📊 知微 | 采薇、筑台 | 手动/自动 |
| 全流程 | 🌀 扶摇 | 全体 | 自动 |

### 1.3 调度方式

#### 方式1：手动调度（南乔直接执行）

**适用场景**：单任务、紧急任务、需要精细控制

**示例**：
```python
# 让采薇生成需求文档
exec command:"python /root/.openclaw/skills/compass-needdoc/scripts/generate_needdoc.py --project '项目名' --input '需求'"

# 让知微分析数据
exec command:"python /root/.openclaw/skills/compass-analysis/scripts/data_analyzer.py /path/to/data.csv"
```

#### 方式2：自动调度（扶摇协调器）

**适用场景**：多步骤任务、需要按流程执行

**示例**：
```bash
# 一键执行全流程
exec command:"python /root/.openclaw/skills/compass-coordinator/scripts/run_full_pipeline.py --project '项目名' --input '需求' --output '/path/to/output'"
```

#### 方式3：Agent调度（独立会话）

**适用场景**：需要独立思考、长时间任务

**示例**：
```python
# 调用采薇Agent
sessions_spawn agentId="caiwei" task="分析湖北电信AI配案系统需求"
```

## 三、质量保障

### 3.1 质量检查流程

```
任务完成
    ↓
自动检查 → 代码规范/格式规范/内容完整
    ↓
人工审核 → 少帅/南乔审核
    ↓
评分记录 → 记录评分和反馈
    ↓
知识沉淀 → 优秀成果入库
```

### 3.2 评分标准

| 维度 | 权重 | 说明 |
|------|:----:|------|
| 完整性 | 30% | 内容是否完整 |
| 正确性 | 30% | 内容是否正确 |
| 规范性 | 20% | 格式是否规范 |
| 时效性 | 20% | 是否按时完成 |

### 3.3 评分等级

| 等级 | 分数 | 说明 |
|:----:|:----:|------|
| A级 | ≥90分 | 优秀，可直接使用 |
| B级 | 70-89分 | 良好，小修改后可用 |
| C级 | 60-69分 | 及格，需要返工 |
| D级 | <60分 | 不合格，需要重做 |

## 四、异常处理

### 4.1 任务异常

| 异常类型 | 处理方式 |
|----------|----------|
| 工具报错 | 检查工具配置，尝试修复 |
| 输入不足 | 向少帅请求补充信息 |
| 能力不足 | 调用其他Agent协助 |
| 时间不足 | 申请延期或简化范围 |

### 4.2 协作异常

| 异常类型 | 处理方式 |
|----------|----------|
| 依赖阻塞 | 联系上游角色催促 |
| 意见分歧 | 提交给南乔仲裁 |
| 进度延误 | 及时预警，调整计划 |
