## 1. 文档排版七要素 ⭐⭐⭐

### 核心要素

| 要素 | 标准 | 说明 |
|------|------|------|
| **字体** | 正文16-18px | 易读为主 |
| **行距** | 1.5-1.8倍 | 不拥挤 |
| **段落** | 3-5行一段 | 不冗长 |
| **标题** | 层级分明 | H1/H2/H3清晰 |
| **列表** | 用项目符号 | 清晰易读 |
| **留白** | 段落间距充足 | 视觉呼吸 |
| **对齐** | 左对齐为主 | 阅读习惯 |

### 详细说明

#### 字体选择

**中文字体推荐**：
| 类型 | 推荐字体 | 特点 |
|------|---------|------|
| 正文 | 微软雅黑、苹方 | 清晰易读 |
| 标题 | 黑体、思源黑体 | 稳重大气 |
| 强调 | 粗体 | 突出醒目 |

**英文字体推荐**：
| 类型 | 推荐字体 | 特点 |
|------|---------|------|
| 正文 | Arial、Helvetica | 简洁专业 |
| 标题 | Arial Bold | 稳重醒目 |
| 数据 | Roboto Mono | 等宽对齐 |

**字体搭配原则**：
- 一份文档最多2种字体
- 正文用易读字体
- 标题可用粗体强调

---

#### 行距设置

**标准**：
- 正文：1.5-1.8倍行距
- 标题：1.2-1.5倍行距
- 列表：1.3-1.5倍行距

**行距计算**：
```
行距 = 字号 × 倍数
例：16px正文 × 1.5倍 = 24px行距
```

---

#### 段落设计

**段落长度**：
- 最短：1行（强调）
- 适中：3-5行
- 最长：不超过8行

**段落间距**：
- 段落间距 ≥ 行距
- 段落之间留白，便于区分

---

#### 标题层级

**标准层级**：
```
H1（一级标题）── 字号最大，用于章节
    ├── H2（二级标题）── 用于小节
    │   ├── H3（三级标题）── 用于段落主题
    │   │   └── H4（四级标题）── 用于细分
```

**层级区分**：
| 层级 | 字号 | 样式 |
|:----:|:----:|------|
| H1 | 最大 | 加粗，颜色区分 |
| H2 | 较大 | 加粗 |
| H3 | 中等 | 加粗或常规 |
| H4 | 较小 | 常规 |

---

#### 列表使用

**有序列表**：有顺序要求的内容
```
1. 第一步
2. 第二步
3. 第三步
```

**无序列表**：并列关系的内容
```
• 要点一
• 要点二
• 要点三
```

**列表格式**：
- 每项不超过2行
- 项与项之间留白
- 层级缩进清晰

---

#### 留白设计

**留白类型**：
| 类型 | 位置 | 比例 |
|------|------|------|
| 页边距 | 页面四周 | 2-3cm |
| 段落间距 | 段落之间 | ≥ 行距 |
| 章节间距 | 章节之间 | ≥ 2倍行距 |
| 元素间距 | 图表与文字 | ≥ 1.5倍行距 |

---

#### 对齐方式

| 对齐方式 | 适用场景 | 特点 |
|---------|---------|------|
| 左对齐 | 正文、列表 | 符合阅读习惯（推荐） |
| 居中对齐 | 标题、短文本 | 正式、平衡 |
| 右对齐 | 日期、数字 | 对齐数字位 |
| 两端对齐 | 英文文档 | 整齐美观 |

## 2. 文档结构设计法 ⭐⭐⭐

### 核心原理

好的文档结构让读者快速定位、理解内容。

### 标准结构

**总分总结构**：
```
总：开篇概述（背景、目的、结论）
    ↓
分：主体内容（分点阐述）
    ├── 要点1
    ├── 要点2
    └── 要点3
    ↓
总：结尾总结（回顾、行动建议）
```

---

### 开篇设计

**开篇三要素**：
| 要素 | 说明 | 篇幅 |
|------|------|------|
| 背景 | 为什么写这份文档 | 1-2段 |
| 目的 | 文档要解决什么问题 | 1段 |
| 结论 | 核心观点（可选） | 1段 |

**开篇技巧**：
- 用简洁的语言说明背景
- 明确文档的目的和价值
- 可先给出核心结论

---

### 主体设计

**分点阐述**：
- 每个要点独立成章
- 用小标题分隔
- 逻辑递进

**常用逻辑顺序**：
| 顺序 | 说明 | 适用场景 |
|------|------|---------|
| 时间顺序 | 按时间先后 | 流程、计划 |
| 结构顺序 | 从整体到局部 | 系统、架构 |
| 重要性顺序 | 从重要到次要 | 建议、问题 |

---

### 结尾设计

**结尾三要素**：
| 要素 | 说明 |
|------|------|
| 总结回顾 | 重申核心观点 |
| 行动建议 | 下一步要做什么 |
| 附录参考 | 相关资料、数据来源 |

---

### 目录设计

**目录层次**：
- 一般文档：2-3层
- 长文档：3-4层
- 最深层级不超过4层

**目录格式**：
```
1. 一级标题 ┈┈┈┈┈┈┈ 1
   1.1 二级标题 ┈┈┈┈┈ 2
      1.1.1 三级标题 ┈┈ 3
```

## 3. 文档视觉层次法 ⭐⭐

### 核心原理

通过视觉手段建立信息层次，引导读者视线。

### 层次建立方式

| 方式 | 方法 | 效果 |
|------|------|------|
| **字号** | 标题 > 正文 | 层级分明 |
| **粗细** | 标题加粗 | 突出重点 |
| **颜色** | 标题用深色/品牌色 | 区分层级 |
| **间距** | 章节间距 > 段落间距 | 分组清晰 |
| **缩进** | 层级缩进 | 结构清晰 |

### 视觉权重排序

```
标题（最大、最粗、最深）
    ↓
小标题（较大、较粗）
    ↓
正文（标准字号、常规）
    ↓
注释/说明（最小、最浅）
```

## 4. 文档格式规范 ⭐⭐

### 通用格式规范

| 项目 | 标准 | 说明 |
|------|------|------|
| 页边距 | 上下2.54cm，左右3.17cm | Word默认 |
| 页眉页脚 | 简洁，含页码 | 不喧宾夺主 |
| 字体 | 正文宋体/微软雅黑 | 易读为主 |
| 字号 | 正文小四/12pt | 舒适阅读 |
| 行距 | 1.5倍 | 不拥挤 |
| 段落间距 | 段前0.5行，段后0.5行 | 分隔清晰 |

### 商务文档规范

**页眉**：
- 左侧：公司Logo或文档标题
- 右侧：日期或版本号

**页脚**：
- 居中：页码（第X页，共X页）
- 可选：保密等级

**封面**：
- 文档标题
- 作者/部门
- 日期
- 版本号

---

### Markdown格式规范

**标题格式**：
```markdown
# 一级标题
## 二级标题
### 三级标题
```

**强调格式**：
```markdown
**粗体强调**
*斜体*
`代码或关键词`
```

**列表格式**：
```markdown
- 无序列表
  - 子列表

1. 有序列表
2. 第二项
```

**引用格式**：
```markdown
> 引用内容
> 多行引用
```

**表格格式**：
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

## 使用指南

### 文档设计流程

| 步骤 | 动作 | 说明 |
|:----:|------|------|
| 1 | 定结构 | 规划文档框架 |
| 2 | 写大纲 | 列出章节标题 |
| 3 | 设格式 | 确定字体、行距等 |
| 4 | 填内容 | 按大纲填充内容 |
| 5 | 修格式 | 调整排版细节 |
| 6 | 检查 | 检查格式一致性 |

### 文档类型适配

| 文档类型 | 结构特点 | 格式要点 |
|---------|---------|---------|
| 方案报告 | 总分总结构 | 图文并茂，标题清晰 |
| 会议纪要 | 时间+议题 | 要点列表，决议突出 |
| 需求文档 | 模块化结构 | 层级清晰，表格为主 |
| 技术文档 | 结构化组织 | 代码块、图表配合 |

### 南乔心法

> 文档如建筑，结构为骨架，内容为血肉，格式为装饰。
> 
> 好的文档让读者：
> - 一眼看懂结构
> - 快速找到重点
> - 轻松阅读全文
> 
> **结构清晰如树，层次分明如塔，排版舒适如家。**
