## 1. 信息架构法 ⭐⭐⭐

### 核心原理

让信息结构清晰，逻辑流畅，用户能快速找到所需信息。

### 四步法

| 步骤 | 动作 | 说明 |
|:----:|------|------|
| 1 | **分类** | 将信息按主题分组 |
| 2 | **层级** | 建立主次关系 |
| 3 | **路径** | 设计阅读顺序 |
| 4 | **导航** | 提供位置指引 |

### 分类原则

**MECE原则**：相互独立，完全穷尽

- 相互独立：各组之间不重叠
- 完全穷尽：所有信息都有归属

**分类维度**：
- 按时间：过去、现在、未来
- 按结构：总-分、上-下、内-外
- 按重要性：核心、重要、辅助
- 按流程：步骤1、步骤2、步骤3

### 层级设计

**三层结构**：

```
第一层：核心信息（标题、结论）
    ↓ 用户首先看到
第二层：支撑信息（论据、数据）
    ↓ 支撑核心结论
第三层：补充信息（细节、背景）
    ↓ 可选深入了解
```

### 阅读路径设计

**F型阅读**：网页、文档
```
用户视线从左上开始，向右扫视，然后向下移动
→ 标题放左上，重要信息放左边
```

**Z型阅读**：PPT、海报
```
用户视线从左上→右上→左下→右下移动
→ 按Z型安排信息流
```

### 导航设计

| 导航类型 | 适用场景 | 设计要点 |
|---------|---------|---------|
| 目录导航 | 长文档 | 清晰层级、可跳转 |
| 面包屑 | 网站 | 显示当前位置 |
| 页码 | PPT、报告 | 总页数+当前页 |
| 标签 | 分类内容 | 清晰的分类标签 |

### 南乔心法

> 信息如水，架构如渠。渠成则水畅，架构明则信息通。

## 2. 数据可视化法 ⭐⭐⭐

### 核心原理

用图表让数据"说话"，让复杂信息一目了然。

### 图表选择指南

| 数据类型 | 推荐图表 | 说明 | 示例 |
|---------|---------|------|------|
| **对比** | 柱状图、条形图 | 直观对比数值大小 | 各部门业绩对比 |
| **趋势** | 折线图、面积图 | 展示变化趋势 | 月度销售趋势 |
| **占比** | 饼图、环形图 | 展示比例关系 | 市场份额占比 |
| **关系** | 散点图、气泡图 | 展示相关性 | 价格与销量关系 |
| **分布** | 直方图、箱线图 | 展示数据分布 | 成绩分布 |
| **流程** | 漏斗图、桑基图 | 展示流程转化 | 销售漏斗 |
| **地理** | 地图、热力图 | 展示地理分布 | 各省销售分布 |

### 图表设计原则

| 原则 | 说明 | 实现方式 |
|------|------|---------|
| **一图一主题** | 每个图表只表达一个核心观点 | 标题即结论 |
| **数据可读** | 数据标签清晰，便于阅读 | 添加数值标签 |
| **重点突出** | 核心数据用颜色/大小突出 | 高亮核心数据 |
| **简洁美观** | 去除不必要的装饰元素 | 减少网格线、背景 |

### 数据可视化配色

**单色渐变**：同色系深浅变化
- 适用：对比类、趋势类
- 示例：蓝浅→蓝深

**对比配色**：不同颜色对比
- 适用：分组对比、分类展示
- 示例：蓝色组 vs 橙色组

**警示配色**：红黄绿状态
- 适用：状态展示、进度展示
- 示例：红=风险、黄=警告、绿=正常

### 常见错误

| 错误 | 问题 | 正确做法 |
|------|------|---------|
| 3D效果 | 视觉干扰，数据不准确 | 使用2D图表 |
| 饼图切块过多 | 难以比较 | 最多5-6块 |
| Y轴不从0开始 | 视觉误导 | Y轴从0开始 |
| 图表类型选择错误 | 信息表达不清 | 按数据类型选图表 |

## 3. 逻辑图法 ⭐⭐

### 核心原理

用图形化方式展示逻辑关系，让复杂关系一目了然。

### 常见逻辑图类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **流程图** | 展示步骤和顺序 | 业务流程、操作步骤 |
| **结构图** | 展示层级和关系 | 组织架构、系统架构 |
| **矩阵图** | 展示多维度对比 | SWOT分析、决策矩阵 |
| **循环图** | 展示闭环流程 | PDCA循环、闭环管理 |
| **关系图** | 展示关联关系 | 概念关系、知识图谱 |
| **时间轴** | 展示时间顺序 | 项目历程、里程碑 |

### 流程图设计

**标准符号**：
- 矩形：处理步骤
- 菱形：判断/决策
- 圆角矩形：开始/结束
- 箭头：流程方向

**设计要点**：
- 从上到下、从左到右
- 每个步骤只有一个入口
- 判断节点有两个出口（是/否）

### 结构图设计

**层级结构**：
```
第一层：顶层概念
    ├── 第二层：子概念1
    │   ├── 第三层：细节1
    │   └── 第三层：细节2
    └── 第二层：子概念2
        └── 第三层：细节3
```

**设计要点**：
- 层级不超过4层
- 同级元素对齐
- 用线条/颜色区分层级

### 矩阵图设计

**经典矩阵**：
- SWOT矩阵：优势、劣势、机会、威胁
- 波士顿矩阵：明星、金牛、问题、瘦狗
- 决策矩阵：多因素评估

**设计要点**：
- 坐标轴标签清晰
- 四象限用不同颜色区分
- 元素位置反映其属性

## 4. 金字塔原理 ⭐⭐

### 核心原理

芭芭拉·明托的经典方法论：结论先行，以上统下。

### 结构模型

```
           ┌───────┐
           │ 结论  │  顶层：核心观点
           └───┬───┘
        ┌──────┼──────┐
        ↓      ↓      ↓
    ┌──────┐┌──────┐┌──────┐
    │论据1 ││论据2 ││论据3 │  中层：支撑论据
    └───┬──┘└───┬──┘└───┬──┘
        ↓      ↓      ↓
    ┌──────┐┌──────┐┌──────┐
    │事实1 ││事实2 ││事实3 │  底层：数据/事实
    └──────┘└──────┘└──────┘
```

### 四个原则

| 原则 | 说明 | 应用 |
|------|------|------|
| **结论先行** | 核心观点放在最前面 | 标题即结论 |
| **以上统下** | 上层统领下层 | 上层是下层的总结 |
| **归类分组** | 同类信息归为一组 | 按主题分类 |
| **逻辑递进** | 按逻辑顺序排列 | 时间/结构/重要性 |

### 应用场景

| 场景 | 应用方式 |
|------|---------|
| **PPT设计** | 标题=结论，内容=论据 |
| **报告写作** | 首段=结论，正文=论据 |
| **汇报沟通** | 先说结论，再说原因 |

### 南乔心法

> 金字塔倒过来是沙堆，立起来是丰碑。
> 结论先行，让听者先知方向；层层支撑，让信者心服口服。

### 设计流程

1. **理解信息** → 分析信息类型和关系
2. **选择方法** → 根据场景选择方法论
3. **构建架构** → 建立信息层级和结构
4. **可视化** → 用图表/图形呈现
5. **检验** → 用户能快速理解吗？
