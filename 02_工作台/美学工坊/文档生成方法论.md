# 文档生成方法论

> **更新时间**：2026-03-12
> **使命**：建立标准化文档生成流程，确保高质量输出

---

## 一、文档生成流程

### 总体流程

```
需求理解 → 内容规划 → 结构设计 → 内容生成 → 美化优化 → 质量检查 → 输出交付
```

### 各阶段说明

| 阶段 | 输入 | 输出 | 关键动作 |
|------|------|------|---------|
| **需求理解** | 用户需求 | 需求文档 | 明确目标、受众、场景 |
| **内容规划** | 需求文档 | 内容大纲 | 确定章节、关键内容 |
| **结构设计** | 内容大纲 | 结构框架 | 选择模板、设计结构 |
| **内容生成** | 结构框架 | 初稿文档 | 填充内容、图表 |
| **美化优化** | 初稿文档 | 美化文档 | 应用设计方法论 |
| **质量检查** | 美化文档 | 最终文档 | 多维度检查 |
| **输出交付** | 最终文档 | 交付物 | 导出、打包、交付 |

## 二、PPT生成方法论

### Step 1：确定PPT类型

| 类型 | 特点 | 页数 | 适用场景 |
|------|------|:----:|---------|
| **方案汇报** | 结构完整、逻辑清晰 | 25-35页 | 项目方案、立项汇报 |
| **竞品分析** | 对比清晰、数据支撑 | 15-20页 | 竞品对比、市场分析 |
| **项目总结** | 成果突出、经验沉淀 | 15-25页 | 项目复盘、工作总结 |
| **产品介绍** | 视觉冲击、卖点突出 | 10-15页 | 产品发布、客户演示 |
| **数据报告** | 图表丰富、结论明确 | 15-25页 | 数据分析、业务报告 |

### Step 2：选择模板

根据PPT类型，从模板库选择合适模板：
- 方案汇报 → 方案汇报模板
- 竞品分析 → 竞品分析模板
- 项目总结 → 项目总结模板
- 产品介绍 → 产品介绍模板
- 数据报告 → 数据报告模板

### Step 3：确定配色方案

| 行业/风格 | 推荐配色 |
|----------|---------|
| **科技/AI** | 科技蓝配色 |
| **电信** | 电信标准配色 |
| **金融** | 金融标准配色 |
| **商务** | 商务深蓝配色 |
| **健康** | 健康清新配色 |

### Step 4：生成页面内容

#### 封面页

**必需元素**：
- 方案标题（核心主题）
- 副标题（可选，补充说明）
- 版本号
- 日期
- 编制人/团队

**设计要点**：
- 标题居中或左对齐
- 字体大、加粗
- 可加品牌Logo
- 背景与主题协调

**内容模板**：
```
[方案名称]
[副标题/定位语]

版本：V1.0
日期：XXXX-XX-XX
编制：[编制人/团队]
```

---

#### 目录页

**必需元素**：
- 章节编号
- 章节名称
- 页码

**设计要点**：
- 对齐整齐
- 编号+标题+页码
- 可用图标装饰

**内容模板**：
```
目录

01  背景与问题 ┈┈┈┈┈┈┈ 03
02  目标与策略 ┈┈┈┈┈┈┈ 08
03  解决方案 ┈┈┈┈┈┈┈┈┈ 15
04  实施计划 ┈┈┈┈┈┈┈┈┈ 25
05  预期收益 ┈┈┈┈┈┈┈┈┈ 30
```

---

#### 内容页

**设计原则**：
- **一页一主题**：每页只讲一个核心观点
- **标题即结论**：标题 = 核心结论
- **少即是多**：每页不超过6行字
- **视觉优先**：图 > 表 > 文

**页面结构**：
```
┌─────────────────────────────────────┐
│ [章节标题]                          │
├─────────────────────────────────────┤
│                                     │
│ [核心结论/观点]                     │
│                                     │
│ [支撑内容：文字/图表/图片]          │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

**内容填充模板**：
```
## [页面标题：结论式]

[核心观点：一句话总结本页核心内容]

### 关键要点
- 要点1
- 要点2
- 要点3

### 数据支撑
- 数据1：[具体数值]
- 数据2：[具体数值]

[图表：根据内容选择合适图表类型]
```

---

#### 结尾页

**必需元素**：
- 感谢语
- Q&A
- 联系方式（可选）

**设计要点**：
- 简洁大方
- 与封面风格呼应

**内容模板**：
```
感谢聆听

Q & A

联系方式：XXX@xxx.com
```

### Step 5：应用设计方法论

#### 视觉设计

| 方法论 | 应用方式 |
|--------|---------|
| **CRAP原则** | 对比突出重点、重复统一风格、对齐整齐、亲密分组 |
| **视觉层次法** | 大小区分层级、颜色突出重点、位置引导视线 |
| **留白艺术法** | 内容留白60:40、重点周围多留白 |

#### 色彩设计

| 方法论 | 应用方式 |
|--------|---------|
| **60-30-10法则** | 主色60%、辅助色30%、点缀色10% |
| **色彩心理学** | 根据情感选择颜色 |
| **行业配色规范** | 按行业选择配色 |

#### 信息设计

| 方法论 | 应用方式 |
|--------|---------|
| **金字塔原理** | 结论先行、以上统下 |
| **信息架构法** | 分类清晰、层级分明 |
| **数据可视化法** | 按数据类型选择图表 |

### Step 6：质量检查

#### 内容检查

| 检查项 | 标准 | 状态 |
|--------|------|:----:|
| 标题即结论 | 每页标题表达核心观点 | ✅/❌ |
| 一页一主题 | 每页只讲一个观点 | ✅/❌ |
| 逻辑清晰 | 章节之间逻辑递进 | ✅/❌ |
| 数据准确 | 所有数据准确无误 | ✅/❌ |

#### 视觉检查

| 检查项 | 标准 | 状态 |
|--------|------|:----:|
| 风格统一 | 字体、配色、风格一致 | ✅/❌ |
| 对齐整齐 | 所有元素对齐 | ✅/❌ |
| 留白适度 | 不拥挤、有呼吸感 | ✅/❌ |
| 重点突出 | 核心信息醒目 | ✅/❌ |

#### 技术检查

| 检查项 | 标准 | 状态 |
|--------|------|:----:|
| 字体嵌入 | 字体正确显示 | ✅/❌ |
| 图片清晰 | 图片无模糊 | ✅/❌ |
| 文件大小 | ≤20MB | ✅/❌ |
| 格式正确 | 符合交付要求 | ✅/❌ |

## 三、Word文档生成方法论

### Step 1：确定文档类型

| 类型 | 特点 | 页数 | 适用场景 |
|------|------|:----:|---------|
| **方案报告** | 结构完整、内容详实 | 20-50页 | 项目方案、立项报告 |
| **需求文档** | 层级清晰、表格为主 | 10-30页 | 需求分析、产品规划 |
| **会议纪要** | 要点突出、行动明确 | 2-5页 | 会议记录、决议跟踪 |
| **复盘报告** | 结构化、可复用 | 5-15页 | 项目复盘、经验沉淀 |

### Step 2：选择模板

从文档模板库选择合适模板，按模板结构填充内容。

### Step 3：格式规范

| 要素 | 规范 |
|------|------|
| **字体** | 正文：微软雅黑 12pt，标题：黑体 加粗 |
| **行距** | 1.5倍行距 |
| **页边距** | 上下2.54cm，左右3.17cm |
| **段落** | 段前0.5行，段后0.5行 |

### Step 4：质量检查

| 检查项 | 标准 | 状态 |
|--------|------|:----:|
| 结构完整 | 所有章节齐全 | ✅/❌ |
| 格式统一 | 字体、行距、间距一致 | ✅/❌ |
| 内容准确 | 信息准确无误 | ✅/❌ |
| 语法正确 | 无语病、错别字 | ✅/❌ |
