# 步骤0_美化工坊赋能模板

> **美化工坊赋能层 - 专门为指南针工程步骤0提供支撑**
> **版本**：V1.0
> **更新时间**：2026-03-12
> **输出格式**：Word + 图片（PNG/JPG）

---

## 一、赋能定位

### 1.1 输出物定义

| 输出物 | 格式 | 说明 |
|--------|------|------|
| 问题分析报告 | Word文档 | 正式的问题分析文档 |
| 问题树图 | 图片（PNG/JPG） | 问题分解可视化 |
| 因果关系图 | 图片（PNG/JPG） | 根因分析可视化 |

### 1.2 为什么用Word + 图片？

**Word的优势**：
- ✅ 正式文档格式
- ✅ 支持丰富的排版
- ✅ 支持图表插入
- ✅ 支持目录结构
- ✅ 适合报告类输出
- ✅ 便于打印存档

**图片的优势**：
- ✅ 可视化呈现
- ✅ 直观易懂
- ✅ 支持复杂图形
- ✅ 便于分享展示

**vs MD文件**：
- ❌ md只能展示简单文本
- ❌ 缺少丰富排版
- ❌ 缺少可视化图表
- ❌ 内容不够丰满

### 2.2 问题树图设计

#### 图形结构

**树状结构**：
```
                    [核心问题]
                        │
           ┌────────────┼────────────┐
           │            │            │
       [子问题1]    [子问题2]    [子问题3]
           │            │            │
    ┌──────┼──────┐ ...    ...
    │      │      │
[原因1][原因2][原因3]
```

#### 节点设计

**节点类型**：

| 节点类型 | 形状 | 颜色 | 字体 | 大小 |

| 原因 | 椭圆 | #FF6B6B（珊瑚红） | 白色常规 10pt | 宽度自动 |

**连接线**：
- 类型：直线或折线
- 颜色：#808080（中灰）
- 粗细：1.5pt

## 三、结构化赋能

### 3.1 问题分析报告框架

#### 标准章节结构

**封面**：
- 报告标题
- 报告副标题
- 编制单位
- 编制日期
- 版本号

**目录**：
- 自动生成目录
- 显示页码

**正文章节**：

```
一、问题背景
    （一）项目背景
    （二）相关背景
    （三）问题提出

二、问题定义
    （一）核心问题
    （二）问题表述
    （三）问题影响

三、问题分解
    （一）问题树分析
    （二）子问题识别
    （三）问题归类

四、根因分析
    （一）5Why分析
    （二）因果关系图
    （三）根本原因

五、影响评估
    （一）业务影响
    （二）用户影响
    （三）经济影响

六、结论建议
    （一）结论
    （二）建议
    （三）下一步行动
```

---

### 3.2 内容扩充要求

#### 问题背景章节

**项目背景**：≥100字
- 项目起因
- 项目目标
- 项目范围

**相关背景**：≥100字
- 政策背景
- 市场背景
- 技术背景

**问题提出**：≥50字
- 问题来源
- 问题紧迫性

---

#### 问题定义章节

**核心问题**：一句话（≤30字）
- 清晰、明确、可理解

**问题表述**：≥150字
- 问题现状描述
- 问题具体表现
- 问题关键要素

**问题影响**：≥100字
- 对业务的影响
- 对用户的影响
- 对组织的影响

---

#### 问题分解章节

**问题树分析**：
- 问题树图（PNG/JPG）
- 问题树说明（≥50字）

**子问题识别**：≥6个子问题
- 每个子问题包含：
  - 编号
  - 名称
  - 描述（≥30字）

**问题归类**：
- 按类别分组
- 每类问题数量

---

#### 根因分析章节

**5Why分析**：≥5层
```
第1层：为什么会出现这个问题？
    ↓
第2层：为什么会出现第1层的原因？
    ↓
第3层：为什么会出现第2层的原因？
    ↓
第4层：为什么会出现第3层的原因？
    ↓
第5层：为什么会出现第4层的原因？
```

**因果关系图**：
- 鱼骨图（PNG/JPG）
- 因果关系说明（≥50字）

**根本原因**：≥3个
- 每个根本原因包含：
  - 编号
  - 名称
  - 描述（≥30字）

---

### 3.3 方法论应用

#### 5W2H分析法

| 要素 | 问题 | 内容要求 |
|------|------|----------|
| What | 是什么 | 问题的具体内容 |
| Why | 为什么 | 问题的原因 |
| Who | 谁 | 涉及的人员 |
| When | 何时 | 问题发生的时间 |
| Where | 何地 | 问题发生的地点 |
| How | 如何 | 问题如何解决 |
| How much | 多少 | 问题的规模、成本 |

---

#### SWOT分析法

| 维度 | 内容 |
|------|------|
| S（优势） | 内部优势因素 |
| W（劣势） | 内部劣势因素 |
| O（机会） | 外部机会因素 |
| T（威胁） | 外部威胁因素 |

**问题定义示例**：

```
二、问题定义

（一）核心问题

一线营业员业务办理效率低、客户体验差。

（二）问题表述

当前一线营业员在业务办理过程中，存在流程复杂、操作繁琐、知识分散等问题。客户平均等待时间超过15分钟，业务办理平均耗时20分钟，客户满意度持续下降。问题主要表现在：
1. 业务流程复杂：办理一个业务需要跳转多个系统，操作繁琐。
2. 知识获取困难：业务知识散落在各个系统，查询路径复杂。
3. 问题定位困难：缺少智能诊断工具，问题定位耗时长。

（三）问题影响

1. 业务影响：业务办理效率低，客户排队时间长。
2. 用户影响：客户满意度下降，投诉率上升。
3. 经济影响：人力成本高，运营效率低。
```

## 五、赋能应用流程

### 5.1 自动化赋能机制

```
指南针工程步骤0触发
        ↓
调用美化工坊赋能模板
        ↓
应用三层赋能
    ├─ 视觉化赋能：Word排版+图表设计
    ├─ 结构化赋能：报告框架+内容扩充
    └─ 专业化赋能：质量检查+最佳实践
        ↓
生成文件
    ├─ 问题分析报告.docx
    ├─ 问题树图.png
    └─ 因果关系图.png
        ↓
执行质量检查
        ↓
输出最终文件
```

### 5.2 南乔执行标准

当执行指南针工程步骤0时，南乔必须：

3. **应用结构化赋能**：

4. **应用专业化赋能**：
