## 设计方法论体系框架

> **2026-03-16 新增**：基于24个方法论构建的三层设计体系

```
┌─────────────────────────────────────────────────────────────┐
│                   🎨 设计方法论体系框架                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   【战略层】决定"做什么"                                     │
│   ├── 设计四大原则（CRAP）：对比、重复、对齐、亲密          │

│   └── 信息架构法：分类、层级、路径、导航                    │
│                                                             │
│   【执行层】决定"怎么做"                                     │
│   ├── 视觉设计：视觉层次法、留白艺术法、对比统一法          │

│                                                             │

│   ├── 质量检查清单：内容维度 + 视觉维度                     │
│   └── 美学价值评估：视觉40% + 信息35% + 情感25%            │
│                                                             │
```

## 案例1：苹果产品发布会PPT

### 基本信息

| 项目 | 内容 |
|------|------|
| **案例名称** | 苹果产品发布会PPT |
| **类型** | PPT设计 |
| **评级** | ⭐⭐⭐ S级 |
| **学习价值** | 极简主义设计典范 |

### 设计亮点

#### 1. 极致的留白

**特点**：
- 每页PPT只有1-2个元素
- 大量留白，让产品成为唯一焦点
- 留白占比超过60%

**方法论提炼**：
> **留白艺术法** - 留白不是浪费，而是让核心更突出

---

#### 2. 视觉层次分明

**特点**：
- 产品图片占据主要视觉空间
- 文字精简到极致（通常只有产品名称）
- 用大小、位置建立明确层次

**方法论提炼**：
> **视觉层次法** - 通过大小、位置建立信息层级

---

#### 3. 文字精简

**特点**：
- 每页通常不超过10个字
- 文字只用于标注产品名称或核心卖点
- 用图片传递信息

**方法论提炼**：
> **少即是多** - 用图片说话，文字只是点缀

---

#### 4. 动画克制

**特点**：
- 动画极少使用
- 只有在展示产品细节时才使用简单的缩放效果
- 动画服务于内容，不喧宾夺主

**方法论提炼**：
> **动画原则** - 动画要克制，有目的性

---

### 可复用设计要点

| 要点 | 应用方式 |
|------|---------|
| 一页一焦点 | 每页只放一个核心元素 |
| 极致留白 | 核心元素周围大量留白 |
| 文字极简 | 标题/名称足够，不要解释性文字 |
| 图片为王 | 用高质量大图传递信息 |

## 案例10：无障碍设计系统

### 基本信息

| 项目 | 内容 |
|------|------|
| **案例名称** | 无障碍设计系统（A11y Design System） |

| **学习价值** | 包容性设计、合规性、社会责任 |

### 设计亮点

#### 1. 对比度规范

**特点**：
- 文本对比度≥4.5:1（WCAG AA）
- 大文本对比度≥3:1
- 非文本元素对比度≥3:1

**方法论提炼**：
> **对比度检验法** - 设计完成后必做对比度检查

---

#### 2. 色彩无障碍

**特点**：
- 不仅依赖颜色传递信息
- 添加图标、纹理等辅助识别
- 色盲模式支持

**方法论提炼**：
> **色彩+形态双通道法** - 颜色+形状/图标=无障碍

---

#### 3. 键盘导航

**特点**：
- 所有交互元素可键盘访问
- 焦点状态清晰可见
- Tab顺序符合逻辑

**方法论提炼**：
> **键盘优先设计法** - 设计时先用键盘测试

---
