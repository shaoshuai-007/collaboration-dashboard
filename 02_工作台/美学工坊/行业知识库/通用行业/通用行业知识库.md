## 一、通用方案框架

### 方案标准结构

```
1. 封面页
   └── 方案标题 + 副标题 + 版本信息

2. 目录页
   └── 清晰的章节索引

3. 背景分析
   ├── 行业背景
   ├── 现状问题
   └── 需求分析

4. 目标策略
   ├── 目标设定（SMART原则）
   └── 策略路径

5. 解决方案
   ├── 总体架构
   ├── 核心功能
   └── 创新亮点

6. 实施计划
   ├── 阶段划分
   ├── 时间规划
   └── 资源需求

7. 预期收益
   ├── 经济收益
   └── 能力提升

8. 风险预案
   └── 风险识别与应对

9. 总结页
   └── 核心观点 + 行动建议
```

## 二、通用分析方法论

### SWOT分析

| 维度 | 说明 | 分析要点 |
|------|------|---------|
| **S (Strengths)** | 优势 | 内部优势资源 |
| **W (Weaknesses)** | 劣势 | 内部不足之处 |
| **O (Opportunities)** | 机会 | 外部机会因素 |
| **T (Threats)** | 威胁 | 外部威胁因素 |

### 5W2H分析

| 维度 | 说明 | 示例问题 |
|------|------|---------|
| **What** | 是什么 | 做什么事情？ |
| **Why** | 为什么 | 为什么要做？ |
| **Who** | 谁 | 谁来做？给谁做？ |
| **When** | 何时 | 什么时候做？ |
| **Where** | 何地 | 在哪里做？ |
| **How** | 怎么做 | 如何实施？ |
| **How much** | 多少 | 需要多少资源？ |

### SMART目标

| 维度 | 说明 | 示例 |
|------|------|------|
| **S (Specific)** | 具体的 | 提升用户活跃度 |
| **M (Measurable)** | 可衡量的 | 活跃率从30%提升到50% |
| **A (Achievable)** | 可实现的 | 基于现有资源可实现 |
| **R (Relevant)** | 相关的 | 与业务目标相关 |
| **T (Time-bound)** | 有时限的 | 3个月内完成 |

## 三、通用数据指标

### 业务指标

| 指标类型 | 常见指标 | 说明 |
|---------|---------|------|
| **规模指标** | 用户数、收入、交易量 | 业务规模 |
| **增长指标** | 增长率、环比、同比 | 增长趋势 |
| **效率指标** | 转化率、完成率、成功率 | 运营效率 |
| **质量指标** | 满意度、留存率、复购率 | 业务质量 |
| **成本指标** | 成本、ROI、CAC | 成本效益 |

### 技术指标

| 指标类型 | 常见指标 | 说明 |
|---------|---------|------|
| **性能指标** | 响应时间、QPS、TPS | 系统性能 |
| **可用性指标** | 可用性、故障率 | 系统稳定性 |
| **安全指标** | 安全事件数、漏洞数 | 安全状况 |

## 四、通用设计规范

### 配色原则

| 原则 | 说明 |
|------|------|
| **行业适配** | 配色与行业特点匹配 |
| **品牌一致** | 与客户品牌色协调 |
| **视觉层次** | 用颜色区分信息层级 |
| **色盲友好** | 考虑色盲用户 |

### 排版原则

| 原则 | 说明 |
|------|------|
| **对齐** | 元素对齐整齐 |
| **留白** | 适度留白，呼吸感 |
| **层次** | 字号、粗细区分层次 |
| **一致** | 风格统一 |
