## 一、设计流程

### 标准流程

```
1. 明确需求 → 了解用户期望的展示形式
2. 内容梳理 → 整理工程核心内容（理念、成果、流程等）
3. 工具选择 → 根据需求选择合适工具
4. 设计迭代 → 多轮沟通优化
5. 最终定稿 → 存档保存
```

### 关键节点

| 阶段 | 产出 | 注意事项 |
|------|------|---------|
| 需求确认 | 明确展示形式 | 参考已有案例，确认格式 |
| 内容整理 | 工程总结文档 | 方法论、成果、流程、配色 |
| 初版设计 | HTML/SVG文件 | 先出初稿，再迭代优化 |
| 用户反馈 | 调整方向 | 根据反馈快速调整 |
| 最终定稿 | PNG+HTML存档 | 保留可编辑版本 |

## 二、工具选择

### 方案对比

| 工具 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **AntV Infographic** | 快速生成、模板丰富 | 格式固定、定制性弱 | 标准化展示 |
| **SVG手绘** | 完全自定义、矢量图 | 中文需特殊处理 | 复杂布局设计 |
| **HTML+浏览器截图** | 中文友好、灵活度高 | 需要前端知识 | **推荐方案** |

### 推荐方案：HTML + 浏览器截图

**原因**：
1. 中文完美显示（使用系统字体）
2. 布局灵活，完全可控
3. 支持渐变、阴影等高级效果
4. 可迭代修改

**技术栈**：
- HTML5 + CSS3
- Puppeteer 截图
- 字体：PingFang SC / Microsoft YaHei / Noto Sans SC

**截图脚本**：
```javascript
const puppeteer = require('puppeteer');
```

## 四、图标设计

### 设计原则

1. **与代号匹配**：图标要与工程代号含义呼应
2. **简洁明了**：一眼能识别核心含义
3. **配色统一**：使用工程主色调

### 美学工坊图标 · Prism（棱镜）

| 元素 | 寓意 |
|------|------|
| 棱镜主体 | 将白光分解为七彩，象征设计将方案分解为多维价值 |
| 入射光线 | 白色光代表问题与需求 |
| 折射光线 | 五彩光代表视觉、信息、情感等多维价值 |
| 金色核心 | 智慧中心，价值汇聚 |

## 五、迭代经验

### 本次迭代过程

| 轮次 | 问题 | 解决方案 |
|------|------|---------|
| 第1轮 | 使用树状图，不符合预期 | 改用网格列表 |
| 第2轮 | 网格列表仍不满意 | 参考智汇工程格式 |
| 第3轮 | 格式统一但内容不足 | 独立设计，完整填充 |
| 第4轮 | SVG中文不显示 | 改用HTML+浏览器截图 |

### 关键经验

1. **先确认格式**：参考已有案例，避免方向错误
2. **工具选对**：中文场景用HTML，不用SVG直接转换
3. **迭代耐心**：多轮沟通是正常的，不抛弃不放弃
4. **存档规范**：保留可编辑版本（HTML），删除废弃版本
