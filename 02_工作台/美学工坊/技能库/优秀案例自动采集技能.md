# 优秀案例自动采集技能

> **技能名称**：case-collector
> **版本**：V1.0
> **创建时间**：2026-03-13
> **用途**：自动从网上采集优秀案例（PPT、文档、设计作品）

---

## 一、采集目标

### 1.1 采集内容

| 类型 | 格式 | 来源 | 优先级 |
|------|------|------|:------:|
| **PPT案例** | .pptx | 演界网、PPTStore、SlideModel | P1 |
| **设计作品** | 图片 | Dribbble、Behance、站酷 | P2 |
| **文档模板** | Word/PDF | Template.net、Canva | P3 |
| **配色方案** | JSON/图片 | ColorHunt、Coolors | P3 |

### 1.2 采集标准

**质量标准**：
- 下载量 ≥ 1000
- 评分 ≥ 4.5/5.0
- 设计专业、风格现代

**分类标准**：
- 按行业分类：电信、金融、制造、零售、医疗
- 按场景分类：方案汇报、竞品分析、项目总结、产品介绍
- 按风格分类：商务、科技、简约、创意

## 三、采集流程

### 3.1 自动采集流程

```
触发采集任务
    ↓
访问采集源网站
    ↓
搜索优秀案例（关键词+筛选条件）
    ↓
评估案例质量（下载量+评分+设计质量）
    ↓
下载案例文件
    ↓
分类入库（按行业/场景/风格）
    ↓
更新案例库索引
    ↓
生成采集报告
```

### 3.2 采集关键词

**按行业**：
- 电信行业：电信、运营商、5G、宽带、客服
- 金融行业：银行、保险、证券、金融科技
- 制造行业：制造、工业、生产、供应链
- 零售行业：零售、电商、营销、客户
- 医疗行业：医疗、健康、医院、医药

**按场景**：
- 方案汇报：方案、规划、建设、实施
- 竞品分析：竞品、对比、分析、市场
- 项目总结：总结、复盘、报告、成果
- 产品介绍：产品、功能、介绍、展示

**按风格**：
- 商务风格：商务、专业、稳重、大气
- 科技风格：科技、未来、创新、智能
- 简约风格：简约、极简、干净、清爽
- 创意风格：创意、个性、独特、新颖

## 四、评估模型

### 4.1 质量评估维度

| 维度 | 权重 | 评估标准 |
|------|:----:|---------|
| **下载量** | 30% | ≥1000下载 |
| **评分** | 30% | ≥4.5/5.0 |
| **设计质量** | 25% | 专业、现代、美观 |
| **实用性** | 15% | 可复用、可修改 |

### 4.2 分级标准

| 等级 | 得分 | 说明 | 处理方式 |
|:----:|:----:|------|----------|
| **S级** | ≥90 | 核心案例，必入库 | 立即入库 |
| **A级** | 80-89 | 优秀案例，推荐入库 | 入库 |
| **B级** | 70-79 | 良好案例，可选入库 | 可选入库 |
| **C级** | <70 | 一般案例，不入库 | 不入库 |

## 六、自动化任务

### 6.1 Cron定时任务

**任务1：每周PPT案例采集**
```bash
openclaw cron add \
  --name "每周PPT案例采集" \
  --cron "0 10 * * 1" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "执行优秀PPT案例自动采集任务，采集电信、金融、制造行业方案汇报类PPT模板，评估质量后入库" \
  --deliver
```

**任务2：每月设计作品采集**
```bash
openclaw cron add \
  --name "每月设计作品采集" \
  --cron "0 10 1 * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "执行优秀设计作品自动采集任务，采集Dribbble、Behance、站酷的海报设计、UI设计作品，评估质量后入库" \
  --deliver
```

### 6.2 手动触发

**触发方式**：
- 少帅指示时立即执行
- 发现优秀案例时立即入库
- 案例库不足时补充采集

## 九、注意事项

### 9.1 版权问题

- ✅ 免费案例：可直接使用
- ⚠️ 付费案例：需要购买授权
- ❌ 受版权保护案例：不可使用

### 9.2 质量控制

- 只采集高质量案例（S/A级）
- 避免采集低质量案例
- 定期清理过时案例
