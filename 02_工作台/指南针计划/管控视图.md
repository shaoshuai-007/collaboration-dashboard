## 四、方向发现引擎

### 4.1 自动发现机制

**方向来源**：

| 来源 | 触发条件 | 发现逻辑 |
|------|----------|----------|
| **商机洞察** | 新商机入库 | 识别关联项目，生成攻坚方向 |
| **知识库分析** | 知识库更新 | 发现知识缺口，生成补全方向 |
| **项目跟进** | 项目状态变化 | 识别关键节点，生成跟进方向 |
| **用户指令** | 少帅明确指示 | 直接创建方向 |
| **外部事件** | 行业动态、政策变化 | 识别影响，生成响应方向 |

### 4.2 方向评估模型

**五维评分**：

| 维度 | 权重 | 评分标准 |
|------|:----:|----------|
| 商业价值 | 30% | 潜在收益规模 |
| 可行性 | 25% | 技术/资源可行性 |
| 紧急度 | 20% | 时间窗口紧迫性 |
| 关联度 | 15% | 与当前商机关联程度 |
| 成熟度 | 10% | 知识/资源准备程度 |

**优先级判定**：

```
总分 ≥ 80分 → ⭐⭐⭐⭐⭐ 立即启动
总分 60-79分 → ⭐⭐⭐⭐ 优先规划
总分 40-59分 → ⭐⭐⭐ 正常排队
总分 < 40分 → ⭐⭐ 暂缓考虑
```

## 七、工作机制

### 7.1 日常运行

| 时间 | 动作 |
|------|------|
| **启动时** | 扫描方向池，识别待处理方向 |
| **执行中** | 按步骤推进，生成输出物 |
| **完成时** | 沉淀知识，更新看板 |
| **定期** | 回顾进度，调整优先级 |

### 7.2 触发机制

**自动触发**：
- 新商机入库 → 自动评估是否生成方向
- 知识库更新 → 自动发现知识缺口
- 项目状态变化 → 自动生成跟进方向

**手动触发**：
- 少帅指令 → 立即创建方向
- 南乔建议 → 等待确认后执行
