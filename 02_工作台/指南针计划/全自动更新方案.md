### 3.2 实现方式二：钩子驱动

**技术栈**：Git Hooks / 文件钩子

```bash
# post-commit hook 示例
#!/bin/bash
# 检测步骤文件变化并自动更新状态

# 1. 检测变更的文件
changed_files=$(git diff --name-only HEAD^ HEAD)

# 2. 判断是否为步骤输出文件
for file in $changed_files; do
  if [[ $file == *"步骤"* ]]; then
    # 3. 更新执行状态跟踪文件
    node update-status.js $file
  fi
done
```

**优点**：
- 与Git集成
- 版本可追溯

**缺点**：
- 依赖Git
- 需要提交才能触发

### 3.3 实现方式三：南乔内置

**技术栈**：南乔内置逻辑

```
南乔执行步骤 → 内置状态更新逻辑 → 自动更新跟踪文件
```

**实现步骤**：
1. 南乔在执行每个步骤前，先更新状态为🔄进行中
2. 执行步骤，生成输出文件
3. 南乔验证检查项是否通过
4. 更新检查项状态
5. 更新统计表
6. 标记步骤完成✅

**优点**：
- 无需额外开发
- 与南乔深度集成
- 灵活性高

**缺点**：
- 需要南乔主动执行
