## 十、自动更新机制

### 10.1 机制说明

| 项目 | 内容 |
|------|------|
| 更新方式 | 半自动更新（南乔驱动） |
| 更新时机 | 每个步骤完成后 |
| 更新内容 | 检查项状态、通过率统计、步骤执行状态 |
| 跟踪文件 | `执行状态跟踪.md` |

### 10.2 执行流程

```
步骤开始 → 南乔更新状态为🔄进行中 →
执行步骤 → 生成输出文件 →
检查质量 → 南乔更新检查项状态✅/❌ →
更新通过率 → 步骤完成标记✅
```

### 10.3 状态说明

| 状态 | 符号 | 说明 |
|------|:----:|------|
| 未开始 | ⏳ | 尚未开始执行 |
| 进行中 | 🔄 | 正在执行中 |
| 已完成 | ✅ | 已完成并通过检查 |
| 待修正 | 🔶 | 已完成但需修正 |
| 已跳过 | ⏭️ | 已跳过（适用于可选步骤） |

## 四、三大闭环

### 4.1 问题到结果

```
问题定义 → 方向计划 → 头脑风暴 → 需求分析 → 举措梳理
    ↓
明确问题本质，找到解决路径
```

### 4.2 设计到落地

```
可行性验证 → 方案编写 → 方案评审 → 执行落地
    ↓
确保方案可行，推动落地执行
```

### 4.3 执行到沉淀

```
效果评估 → 经验萃取 → 知识入库 → 复用推广
    ↓
形成知识沉淀，支撑下一轮循环
```

## 八、工作原则

### 核心原则

1. **先思考，再行动**
   - 充分分析，明确方向
   - 方法先行，工具支撑

2. **问题导向**
   - 从问题出发，到结果落地
   - 不为做方案而做方案

3. **闭环思维**
   - 每一步都要有输出
   - 每一轮都要有沉淀

4. **方法论支撑**
   - 不凭感觉，凭方法
   - 不靠经验，靠工具
