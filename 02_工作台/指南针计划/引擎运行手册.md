## 二、执行层引擎

### 2.1 方向发现引擎（Discover）

#### 基本信息

| 属性 | 内容 |
|------|------|
| 引擎代号 | Discover |
| 引擎名称 | 方向发现引擎 |
| 所属层级 | 执行层 |
| 触发条件 | 知识库更新 / 商机入库 / 用户指令 / 外部事件 |

#### 运行流程

```
触发信号 → 来源识别 → 内容分析 → 五维评估 → 优先级排序 → 方向入库
```

#### 输入标准

| 输入项 | 来源 | 格式 |
|--------|------|------|
| 知识更新 | 知识库 | 知识卡片 |
| 商机信息 | 商机库 | 商机档案 |
| 用户指令 | 用户 | 文字指令 |
| 外部事件 | 外部 | 事件描述 |

#### 处理逻辑

**步骤1：来源识别**

| 来源 | 权重 | 处理方式 |
|------|:----:|----------|
| 商机洞察 | 40% | 识别关联项目 |
| 知识库分析 | 25% | 发现知识缺口 |
| 项目跟进 | 20% | 识别关键节点 |
| 用户指令 | 15% | 直接创建 |

**步骤2：五维评估**

| 维度 | 权重 | 评分标准 |
|------|:----:|----------|
| 商业价值 | 30% | 1-10分 |
| 可行性 | 25% | 1-10分 |
| 紧急度 | 20% | 1-10分 |
| 关联度 | 15% | 1-10分 |
| 成熟度 | 10% | 1-10分 |

**步骤3：优先级判定**

| 总分 | 优先级 | 标记 |
|:----:|:------:|:----:|
| ≥80分 | 立即启动 | ⭐⭐⭐⭐⭐ |
| 60-79分 | 优先规划 | ⭐⭐⭐⭐ |
| 40-59分 | 正常排队 | ⭐⭐⭐ |
| <40分 | 暂缓考虑 | ⭐⭐ |

#### 输出标准

| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 方向清单 | Markdown | 方向名称、评分、优先级、建议顺序 |

#### 示例输出

```markdown
# 方向发现报告

## 发现时间
2026-03-11 22:00

## 方向清单

| 编号 | 方向名称 | 评分 | 优先级 | 来源 |
|:----:|----------|:----:|:------:|------|
| N-001 | 商机深度攻坚 | 85 | ⭐⭐⭐⭐⭐ | 商机洞察 |
| N-002 | 知识库深度激活 | 72 | ⭐⭐⭐⭐ | 知识库分析 |
| N-003 | 电渠项目跟进 | 58 | ⭐⭐⭐ | 项目跟进 |

## 建议启动顺序
1. N-001 商机深度攻坚（评分最高，建议优先）
2. N-002 知识库深度激活（支撑其他方向）
3. N-003 电渠项目跟进（关注预算风险）
```

### 2.2 计划制定引擎（Planner）

#### 基本信息

| 属性 | 内容 |
|------|------|
| 引擎代号 | Planner |
| 引擎名称 | 计划制定引擎 |
| 所属层级 | 执行层 |
| 触发条件 | 方向确认启动 |

#### 运行流程

```
方向确认 → 调用步骤0标准 → 调用步骤1标准 → 创建工作目录 → 输出管控文档
```

#### 输入标准

| 输入项 | 来源 | 格式 |
|--------|------|------|
| 方向信息 | 方向发现引擎 | 方向清单 |
| 方法论工具箱 | 知识库 | 方法论文档 |

#### 处理逻辑

**步骤1：执行步骤0（问题定义）**

| 序号 | 动作 | 方法论 | 输出 |
|:----:|------|--------|------|
| 0.1 | 问题陈述 | 5W2H | 7个维度分析 |
| 0.2 | 问题分析 | SWOT | 四象限分析 |
| 0.3 | 根因追溯 | 5Why | 至少5层追问 |
| 0.4 | 原因梳理 | 鱼骨图 | 多维度归类 |
| 0.5 | 问题定义 | 总结 | 一句话定义 |

**步骤2：执行步骤1（方向计划）**

| 序号 | 动作 | 方法论 | 输出 |
|:----:|------|--------|------|
| 1.1 | 目标设定 | SMART | 5要素目标 |
| 1.2 | 目标分解 | OKR | 目标+关键结果 |
| 1.3 | 计划制定 | WBS | 工作分解结构 |
| 1.4 | 责任分配 | RACI | 责任矩阵 |
| 1.5 | 时间规划 | 甘特图 | 时间节点 |

**步骤3：创建工作目录**

```
02_工作台/指南针计划/[方向名称]/
├── 00_问题分析.md
├── 01_管控文档.md
├── 02_头脑风暴/
├── 03_举措表/
├── 04_方案PPT/
└── 05_知识卡片.md
```

#### 输出标准

| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 问题分析文档 | Markdown | 5W2H/SWOT/根因/鱼骨图 |
| 管控文档 | Markdown | SMART/OKR/WBS/RACI/时间规划 |

### 2.3 方案生成引擎（Generator）

#### 基本信息

| 属性 | 内容 |
|------|------|
| 引擎代号 | Generator |
| 引擎名称 | 方案生成引擎 |
| 所属层级 | 执行层 |
| 触发条件 | 步骤1完成并审核通过 |

#### 运行流程

```
步骤2 → 步骤2.5 → 步骤3 → 步骤3.5 → 步骤4
  ↓        ↓        ↓        ↓        ↓
 xmind   需求规格   举措表   可行性报告  方案PPT
```

#### 输入标准

| 输入项 | 来源 | 格式 |
|--------|------|------|
| 问题分析文档 | 步骤0 | Markdown |
| 管控文档 | 步骤1 | Markdown |
| 方法论工具箱 | 知识库 | 方法论文档 |

#### 处理逻辑

| 步骤 | 动作 | 方法论 | 输出 |
|:----:|------|--------|------|
| 2 | 头脑风暴 | 六顶思考帽、SCAMPER | 思维导图.xmind |
| 2.5 | 需求分析 | 干系人地图、KANO、MoSCoW | 需求规格说明书 |
| 3 | 举措梳理 | WBS、RACI、优先级矩阵 | 详细举措.xlsx |

#### 输出标准

| 输出项 | 格式 | 质量标准 |
|--------|------|----------|
| 思维导图 | XMIND | 六顶思考帽完整、方案≥3个 |
| 需求规格说明书 | Markdown | 干系人地图、用户故事≥5个 |
| 详细举措表 | Excel | WBS≥4层、RACI明确 |
| 可行性评估报告 | Markdown | 技术经济操作三维度 |
| 方案PPT | PPTX | 标准章节结构完整 |

#### 基本信息

| 属性 | 内容 |
|------|------|

| 引擎名称 | 效果评估引擎 |

## 三、审核层引擎

### 3.1 流程审核引擎（Auditor）

#### 基本信息

| 属性 | 内容 |
|------|------|
| 引擎代号 | Auditor |
| 引擎名称 | 流程审核引擎 |
| 所属层级 | 审核层 |
| 触发条件 | 每个步骤完成后自动触发 |

#### 运行流程

```
步骤输出 → 读取步骤标准 → 检查必需项 → 计算质量分 → 判定结果 → 输出报告
```

#### 审核维度

| 审核项 | 权重 | 检查内容 |
|--------|:----:|----------|
| 步骤完整性 | 30% | 关键步骤是否完成 |
| 方法论正确性 | 30% | 必需方法论是否使用 |
| 输出物质量 | 30% | 必需内容是否包含 |
| 时间节点合规 | 10% | 是否按时交付 |

#### 判定标准

| 质量分 | 结果 | 处理 |
|:------:|:----:|------|
| ≥90分 | 🟢 通过 | 进入下一步 |
| 70-89分 | 🟡 需修正 | 修正后通过 |
| <70分 | 🔴 退回 | 返回重做 |

#### 输出标准

| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 审核报告 | Markdown | 审核项、检查结果、评分、结论 |

#### 审核报告模板

```markdown
# 流程审核报告

## 审核对象
步骤X：[步骤名称]

## 审核时间
YYYY-MM-DD HH:mm

## 审核结果
| 审核项 | 权重 | 得分 | 说明 |
|--------|:----:|:----:|------|

## 总分
XX分

## 审核结论
🟢 通过 / 🟡 需修正 / 🔴 退回

## 改进建议
- [ ] 建议1
- [ ] 建议2
```

### 3.2 价值审核引擎（Validator）

#### 基本信息

| 属性 | 内容 |
|------|------|
| 引擎代号 | Validator |
| 引擎名称 | 价值审核引擎 |
| 所属层级 | 审核层 |

#### 运行流程

```
全部输出 → 问题解决评估 → 目标达成评估 → 方案可行性评估 → 知识沉淀评估 → 价值评级
```

#### 审核维度

| 维度 | 权重 | 评估内容 | 评估标准 |
|------|:----:|----------|----------|
| 问题解决度 | 30% | 问题是否解决 | 问题分析→解决方案→效果验证 |
| 目标达成度 | 25% | 目标是否达成 | OKR对比→达成率计算 |
| 方案可行性 | 25% | 方案可落地性 | 可行性报告→风险评估 |
| 知识沉淀度 | 20% | 是否形成沉淀 | 知识卡片→入库保存 |

#### 价值评级标准

| 总分 | 评级 | 说明 |
|:----:|:----:|------|
| ≥90分 | S级 | 价值连城，可直接推广 |
| 80-89分 | A级 | 高价值，建议采纳 |
| 70-79分 | B级 | 有价值，优化后采纳 |
| <70分 | C级 | 待优化，需重新打磨 |

#### 输出标准

| 输出项 | 格式 | 内容要求 |
|--------|------|----------|
| 价值评估报告 | Markdown | 四维评分、总分、评级、改进建议 |

## 四、引擎协同机制

### 4.1 执行层协同

```
Discover → Planner → Generator → Executor → Evaluator
   ↓          ↓          ↓           ↓           ↓
 方向清单   管控文档    方案PPT     执行报告    评估报告
```

### 4.2 审核层介入

```
每步完成 → Auditor审核 → 通过 → 下一步
                    ↓
                不通过 → 返回修正
```

### 4.3 最终审核

```
全部完成 → Validator价值审核 → 评级 → 入库
```

---

## 五、异常处理

### 5.1 审核不通过

| 情况 | 处理 |
|------|------|
| 质量分70-89分 | 返回修正，重新提交 |
| 质量分<70分 | 返回重做，重新执行 |

### 5.2 方向变更

| 情况 | 处理 |
|------|------|
| 方向取消 | 终止流程，归档处理 |
| 方向调整 | 重新评估，更新管控文档 |

### 5.3 时间延期

| 情况 | 处理 |
|------|------|
| 合理延期 | 更新时间规划，继续执行 |
| 不合理延期 | 上报预警，调整资源 |
