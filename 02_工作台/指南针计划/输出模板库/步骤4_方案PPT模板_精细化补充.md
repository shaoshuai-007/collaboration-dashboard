## 一、每页必含三要素标准

### 1.1 三要素定义

**要素1：标题**
- **格式**：主标题：副标题
- **主标题**：明确本页核心主题
- **副标题**：一句话概括本页核心内容，不超过10字
- **示例**：项目背景：满足考核解决痛点

**要素2：建设目标**
- **内容**：一句话说明本页的核心目标
- **要求**：清晰、明确、可衡量
- **位置**：标题下方
- **示例**：满足集团云运〔2025〕4号考核要求，解决一线营业员效率低、体验差的核心痛点。

**要素3：具体举措**
- **内容**：3-5条具体可行的举措
- **格式**：每条举措包含"举措标题"+"举措内容"
- **位置**：左侧内容区
- **示例**：
  1. **深度调研分析**：深入12个分公司开展需求调研，覆盖营业厅、客服、渠道等40+业务场景。
  2. **考核指标对标**：云运〔2025〕4号文件明确要求AI嵌入10个关键业务场景，自动化率达70%。

## 二、内容扩充要求

### 2.1 问题分析页扩充标准

**要求**：
- 问题表象：不少于50字，详细描述问题现状
- 核心根因：至少6项，每项包含"概括+描述"
- 解决方案：至少6项，每项包含"概括+描述"
- 解决方案设计：不少于150字，分段排列

**示例**：

**标题**：问题分析：定位根因精准施策

**建设目标**：深度剖析一线营业员效率低、体验差的根因，提出针对性解决方案。

**问题表象分析**（不少于50字）：
效率低：业务办理平均耗时20分钟，高度依赖人工操作，流程繁琐易出错。体验差：客户平均等待15分钟以上，排队时间长，满意度持续下降。知识获取难：业务知识散落在各个系统，查询路径复杂，耗时耗力。

**核心根因定位**（至少6项）：

| 概括 | 描述 |
|------|------|
| 缺乏工具 | 无AI助手赋能一线营业员，全部依赖人工操作 |
| 知识分散 | 业务知识散落在CRM、随翼选、知识库等多个系统 |
| 路径复杂 | 查询路径复杂，需要多次跳转，操作繁琐 |
| 流程冗长 | 办理流程长，涉及多个系统，易出错 |
| 定位困难 | 问题定位困难，缺少智能诊断工具 |
| 数据孤岛 | 系统数据不互通，信息孤岛严重 |

**解决方案设计**（不少于150字，分段排列）：

基于大模型+RAG技术，建设营业受理数字员工系统，通过四大智能体集群作战，全面提升一线营业员工作效率和客户体验。

第一，通过智能问答能力，准确率≥90%，减少人工查询时间70%，让营业员快速获取业务知识。

第二，通过智能查询能力，效率提升50%，一键获取客户360度视图，让营业员快速了解客户信息。

第三，通过智能办理能力，时长缩短40%，填单准确率≥98%，让营业员快速完成业务办理。

第四，通过智能诊断能力，时间缩短60%，一键报障闭环处理，让营业员快速定位和解决问题。

### 2.2 架构页扩充标准

**要求**：
- 每层名称
- 每层功能：不少于50字，详细说明功能定位
- 每层组成：3-5项，每项包含"名称+描述"

**示例**：

**标题**：整体架构：六层分层清晰可落地

**建设目标**：构建"交互-智能体-能力-模型-数据-集成"六层架构，实现技术可落地、业务可支撑。

**架构理念**：
采用分层解耦设计，每层职责清晰、接口标准，支持快速迭代和持续优化。

**六层架构详解**：

**交互层**
- **功能**（不少于50字）：提供统一对话入口，支持PC端、移动端、语音等多种交互方式，实现多终端全覆盖，让一线营业员随时随地获取AI助手支持。
- **组成**：
  - ChatUI入口：统一对话界面，支持自然语言输入
  - PC端/移动端：多终端支持，随时随地访问
  - 语音入口：语音交互，解放双手

**智能体层**
- **功能**（不少于50字）：部署四大智能体（Q问答、S查询、H办理、D诊断），分工协作，协同作战，形成完整的营业受理智能体集群。
- **组成**：
  - Q-Agent：智能问答，知识快速获取
  - S-Agent：智能查询，信息快速检索
  - H-Agent：智能办理，业务快速处理
  - D-Agent：智能诊断，问题快速定位

**能力层**
- **功能**（不少于50字）：构建意图识别、知识检索、智能填单三大核心能力，支撑四大智能体的业务逻辑实现。
- **组成**：
  - 意图识别：NLU引擎，准确理解用户意图
  - 知识检索：RAG检索，快速检索业务知识
  - 智能填单：表单自动填充，减少人工录入

**模型层**
- **功能**（不少于50字）：集成大语言模型（GPT/文心/通义）、RAG检索引擎、向量数据库，构建强大的AI基础设施。
- **组成**：
  - 大语言模型：GPT/文心/通义，支持多模型切换
  - RAG检索：向量检索+重排序，提高检索准确率
  - 向量数据库：Milvus/ES，存储向量化的业务知识

**数据层**
- **功能**（不少于50字）：整合业务知识库、客户数据、系统数据，为AI提供丰富的数据支撑。
- **组成**：
  - 业务知识库：规则/资费/流程/产品，覆盖核心业务
  - 客户数据：档案/套餐/余额/订单，360度客户视图
  - 系统数据：配置/日志/接口/报文，系统运行数据

**集成层**
- **功能**（不少于50字）：对接CRM、随翼选、云哨等八大核心系统，实现数据互通和业务协同。
- **组成**：
  - CRM系统：客户关系管理，客户信息查询
  - 随翼选：营业办理平台，业务办理支撑
  - 云哨平台：智能诊断平台，问题诊断分析

### 2.3 能力页扩充标准

**要求**：
- 核心业务场景：3-4个，每个包含"场景名称+场景描述"
- 技术实现路径：5-6步
- 关键指标：3-4个
- 业务描述：不少于100字

**示例（智能问答能力页）**：

**标题**：智能问答能力 (Q-Agent)：大模型加RAG技术

**建设目标**：准确率≥90%，基于大模型+RAG+向量检索，实现业务知识快速问答。

**核心业务场景**（3-4个）：
1. **业务规则查询**：办理条件、办理流程、办理材料
2. **资费政策解答**：套餐详情、优惠活动、资费标准
3. **流程指引**：操作步骤、注意事项、常见问题
4. **故障排查**：问题定位、解决方案、报障流程

**技术实现路径**（5-6步）：
1. 用户自然语言输入
2. 意图智能识别（NLU引擎）
3. 知识向量检索（RAG检索）
4. 结果重排序（提高准确率）
5. 大模型生成答案（GPT/文心/通义）
6. 结构化输出（便于阅读）

**关键指标**（3-4个）：
- 准确率≥90%
- 响应时间<3秒
- 日均咨询量5000+次
- 用户满意度≥85%

**业务描述**（不少于100字）：
覆盖业务规则查询、资费政策解答、流程指引、故障排查、政策解读等高频场景，日均咨询量超过5000次。通过智能问答能力，让一线营业员快速获取业务知识，准确率≥90%，响应时间<3秒，大幅提升服务效率和客户满意度，减少人工客服压力。
