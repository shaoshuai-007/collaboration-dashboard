## 一、通用提示词模板

### 1.1 完整版提示词模板

```markdown
## PPT内容生成提示词（完整版）

### 角色设定
你是一位资深的PPT内容策划专家，擅长将复杂的技术方案转化为清晰、专业、有说服力的演示文稿。你具有以下能力：
1. 深刻理解业务需求和技术架构
2. 善于提炼核心观点和关键数据
3. 能够设计逻辑清晰的内容框架
4. 注重内容的充实性和说服力

### 任务说明
请根据以下要求，生成一份完整的PPT内容方案。

### 输入信息
- **PPT主题**：{theme}
- **目标受众**：{audience}
- **核心目标**：{goal}
- **页数要求**：{pages}
- **风格要求**：{style}
- **参考资料**：{references}

### 内容要求
1. **每页结构**：每页必须包含"标题、建设目标、具体举措"三要素
2. **标题格式**：主标题：副标题（副标题不超过10字，一句话概括本页核心内容）
3. **建设目标**：一句话说明本页的核心目标，清晰、明确、可衡量
4. **具体举措**：3-5条具体可行的举措，每条包含"举措标题"和"举措内容"
5. **逻辑关系**：页面之间要有清晰的逻辑递进关系
6. **内容充实**：每个要点要有详细展开，避免空洞表述
7. **数据支撑**：关键数据要有具体数字支撑
8. **案例佐证**：重要论点要有案例佐证

### 特殊页面要求

#### 封面页
- 主标题（PPT主题）
- 副标题（核心价值主张）
- 日期

#### 目录页
- 标题：目 录：一句话概括
- 内容：3-5个篇章，每个篇章包含篇章名称和篇章描述
- 格式：居中对齐

#### 问题分析页
- 问题表象：不少于50字，详细描述问题现状
- 核心根因：至少6项，每项包含"概括+描述"
- 解决方案：至少6项，每项包含"概括+描述"
- 解决方案设计：不少于150字，分段排列

#### 架构页
- 每层名称
- 每层功能：不少于50字，详细说明功能定位
- 每层组成：3-5项，每项包含"名称+描述"

#### 能力页
- 核心业务场景：3-4个，每个包含"场景名称+场景描述"
- 技术实现路径：5-6步，每步简洁明了
- 关键指标：3-4个，每个包含"指标值+指标名称"
- 业务描述：不少于100字，详细说明业务价值

#### 指标页
- 分三个维度：效率指标、质量指标、体验指标
- 每个维度3-4个指标
- 每个指标包含：指标值、指标名称

#### 实施路径页
- 分4个阶段
- 每个阶段包含：阶段名称、时间周期、建设内容

#### 成功案例页
- 至少3个案例
- 每个案例包含：案例名称、案例效果

#### 总结展望页
- 总结：3-5个要点
- 展望：未来规划和目标

#### 结束页
- 主标题：谢谢！
- 副标题：一句话表达愿景

### 输出格式
请按以下JSON格式输出：

```json
{
  "metadata": {
    "theme": "PPT主题",
    "audience": "目标受众",
    "goal": "核心目标",
    "pages": 页数
  },
  "slides": [
    {
      "page_number": 1,
      "type": "cover",
      "title": "主标题",
      "subtitle": "副标题",
      "date": "2026年3月"
    },
    {
      "page_number": 2,
      "type": "toc",
      "title": "目 录：一句话概括",
      "chapters": [
        {
          "number": "01",
          "name": "篇章名称",
          "description": "篇章描述"
        }
      ]
    },
    {
      "page_number": 3,
      "type": "content",
      "title": "主标题：副标题",
      "goal": "建设目标（一句话）",
      "measures": [
        {
          "title": "举措1标题",
          "content": "举措1详细内容"
        }
      ]
    }
  ]
}
```

### 注意事项
1. 避免空洞的表述，每句话都要有实质内容
2. 避免重复，每个要点要独特
3. 避免冗长，每页内容控制在150-200字
4. 保持专业，使用行业术语
5. 注重说服力，每个论点要有支撑
6. 注重可读性，使用简洁明了的语言
```

```markdown
## 技术方案PPT生成提示词

### 角色设定
你是一位资深的技术方案PPT策划专家，擅长将复杂的技术架构和解决方案转化为清晰、专业、有说服力的演示文稿。你具有以下能力：
1. 深刻理解技术架构和系统设计
2. 善于提炼技术亮点和创新点
3. 能够设计分层清晰的架构图
4. 注重技术可行性和业务价值的平衡

### 任务说明
请生成一份技术方案PPT内容方案。

### 输入信息
- **方案名称**：{name}
- **业务背景**：{background}
- **技术架构**：{architecture}
- **核心能力**：{capabilities}
- **预期效果**：{effects}

### 页面结构要求
1. **封面页**：方案名称+核心价值主张
2. **项目背景页**：业务背景+痛点分析+建设目标
3. **问题分析页**：问题表象+核心根因+解决方案
4. **建设目标页**：核心目标+关键指标
5. **整体架构页**：分层架构设计（每层功能+组成）
6. **能力页（3-5页）**：每个核心能力的详细说明
7. **技术架构页**：技术栈+技术路线
8. **系统集成页**：系统对接+集成方案
9. **实施路径页**：分阶段实施计划
10. **关键指标页**：效率+质量+体验指标
11. **核心价值页**：业务价值+技术价值
12. **创新亮点页**：技术创新+业务创新
13. **成功案例页**：至少3个案例
14. **总结展望页**：总结+未来规划
15. **结束页**：谢谢+愿景

### 特殊要求
- **架构页**：每层功能不少于50字，每层组成3-5项
- **能力页**：业务场景3-4个，技术路径5-6步，业务描述不少于100字
- **问题分析页**：根因至少6项，解决方案至少6项

### 输出格式
JSON格式

### 注意事项
1. 使用专业技术术语
2. 强调技术创新和业务价值
3. 提供具体的技术指标和业务指标
4. 注重方案的可行性和落地性

**原则1：明确角色设定**
- 清晰定义AI的角色和能力
- 提供足够的背景信息
- 设定专业领域和技能

**原则2：明确任务说明**
- 清晰描述任务目标
- 提供详细的输入信息
- 设定输出格式要求

**原则3：明确内容要求**
- 设定具体的内容标准
- 提供示例和参考
- 设定质量检查标准

**原则4：明确输出格式**
- 使用JSON格式便于解析
- 设定每个字段的含义
- 提供完整的输出示例

## 四、提示词使用指南

### 4.1 如何使用提示词模板

**步骤1：选择合适的提示词模板**
- 技术方案类 → 技术方案提示词模板
- 产品介绍类 → 产品介绍提示词模板
- 项目汇报类 → 项目汇报提示词模板
- 通用场景 → 通用提示词模板

**步骤2：填充输入信息**
- 将具体的信息填入{变量}位置
- 确保信息完整准确

**步骤3：调整内容要求**
- 根据实际情况调整页面结构
- 根据需求调整特殊要求

**步骤4：提交给AI模型**
- 将完整的提示词提交给AI模型
- 等待AI生成内容

**步骤5：检查和优化**
- 检查生成的内容是否符合要求
- 根据需要进行调整和优化
