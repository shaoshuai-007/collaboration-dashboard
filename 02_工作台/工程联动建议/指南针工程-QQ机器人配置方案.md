## 一、部署模式选择

### 模式对比

| 模式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **会话隔离模式** | 配置简单、易管理、成本小 | 只有一个机器人账号 | 推荐⭐ |
| **多实例模式** | 每个技能独立机器人 | 需要多个QQ机器人账号 | 独立团队协作 |
| **子代理模式** | 主机器人分发任务 | 配置复杂 | 大规模部署 |

## 三、备选方案：多实例模式

### 3.1 架构说明

```
┌─────────────────────────────────────────────────────────────┐
│                    多机器人独立部署                           │
└─────────────────────────────────────────────────────────────┘

QQ机器人1：采薇（需求分析师）
├── AppID: 1903219462_1
└── 技能：compass-needdoc

QQ机器人2：织锦（架构设计师）
├── AppID: 1903219462_2
└── 技能：compass-mindmap

QQ机器人3：筑台（售前工程师）
├── AppID: 1903219462_3
└── 技能：compass-solution

...（共8个机器人）

QQ机器人8：扶摇（总指挥）
├── AppID: 1903219462_8
└── 技能：compass-coordinator
```

### 3.2 配置步骤

#### 步骤1：申请多个QQ机器人

在 [QQ机器人平台](https://bot.q.qq.com) 创建多个机器人：

| 机器人名称 | AppID | 用途 |
|-----------|-------|------|
| 采薇-需求分析师 | 待申请 | 需求文档 |
| 织锦-架构设计师 | 待申请 | 思维导图 |
| 筑台-售前工程师 | 待申请 | 方案举措 |
| 呈彩-方案设计师 | 待申请 | PPT |
| 工尺-系统设计师 | 待申请 | 详细设计 |
| 玉衡-项目经理 | 待申请 | 项目管控 |
| 折桂-资源管家 | 待申请 | 通用资源 |
| 扶摇-总指挥 | 待申请 | 全流程 |

#### 步骤2：部署多个OpenClaw实例

```bash
# 实例1：采薇
cd /root/compass-robots/caiwei
openclaw start --port 18790 --config openclaw-caiwei.json

# 实例2：织锦
cd /root/compass-robots/zhijin
openclaw start --port 18791 --config openclaw-zhijin.json

# ... 其他实例
```

#### 步骤3：配置各实例

```json
// openclaw-caiwei.json（采薇实例）
{
  "channels": {
    "qqbot": {
      "appId": "采薇的AppID",
      "clientSecret": "采薇的Secret"
    }
  },
  "skills": {
    "entries": {
      "compass-needdoc": { "enabled": true }
    }
  }
}
```

### 3.3 优缺点

**优点**：
- 每个技能独立运行，互不干扰
- 可以分配给不同团队使用
- 故障隔离，一个挂了不影响其他

**缺点**：
- 需要申请8个QQ机器人账号
- 维护成本高（8个实例）
- 资源占用大
