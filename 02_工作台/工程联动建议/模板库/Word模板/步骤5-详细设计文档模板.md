## 二、接口设计

### 2.1 接口规范

#### 2.1.1 RESTful API规范

| 规范项 | 说明 |
|--------|------|
| **协议** | HTTPS |
| **请求格式** | JSON |
| **响应格式** | JSON |
| **字符编码** | UTF-8 |
| **版本控制** | URL路径版本 /api/v1/ |

#### 2.1.2 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 业务数据
  },
  "timestamp": 1703275200000
}
```

#### 2.1.3 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 2.2 接口列表

#### 2.2.1 智能问答接口

**接口1：智能问答**

| 维度 | 说明 |
|------|------|
| **接口名称** | 智能问答 |
| **接口路径** | POST /api/v1/qa/ask |
| **接口描述** | 提交问题，获取智能回答 |

```json
  "message": "success",
  "data": {

    "confidence": 0.95,
    "sources": [
      {
        "title": "宽带业务办理指南",
        "content": "..."
      }
    ]
  },
  "timestamp": 1703275200000
}
```

### 3.2 逻辑模型

#### 3.2.1 用户表（t_user）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| user_id | VARCHAR(32) | 用户ID | PRIMARY KEY |
| name | VARCHAR(50) | 用户姓名 | NOT NULL |
| phone | VARCHAR(11) | 手机号 | UNIQUE |
| email | VARCHAR(100) | 邮箱 | |
| status | TINYINT | 状态（1正常，0禁用） | DEFAULT 1 |
| create_time | DATETIME | 创建时间 | NOT NULL |
| update_time | DATETIME | 更新时间 | |

**建表语句**：

```sql
CREATE TABLE t_user (
  user_id VARCHAR(32) PRIMARY KEY COMMENT '用户ID',
  name VARCHAR(50) NOT NULL COMMENT '用户姓名',
  phone VARCHAR(11) UNIQUE COMMENT '手机号',
  email VARCHAR(100) COMMENT '邮箱',
  status TINYINT DEFAULT 1 COMMENT '状态',
  create_time DATETIME NOT NULL COMMENT '创建时间',
  update_time DATETIME COMMENT '更新时间'
) COMMENT='用户表';
```

---

#### 3.2.2 会话表（t_session）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| session_id | VARCHAR(32) | 会话ID | PRIMARY KEY |
| user_id | VARCHAR(32) | 用户ID | FOREIGN KEY |
| type | VARCHAR(20) | 会话类型 | NOT NULL |
| status | TINYINT | 状态 | DEFAULT 1 |
| create_time | DATETIME | 创建时间 | NOT NULL |
| update_time | DATETIME | 更新时间 | |

**建表语句**：

```sql
CREATE TABLE t_session (
  session_id VARCHAR(32) PRIMARY KEY COMMENT '会话ID',
  user_id VARCHAR(32) NOT NULL COMMENT '用户ID',
  type VARCHAR(20) NOT NULL COMMENT '会话类型',
  status TINYINT DEFAULT 1 COMMENT '状态',
  create_time DATETIME NOT NULL COMMENT '创建时间',
  update_time DATETIME COMMENT '更新时间',
  FOREIGN KEY (user_id) REFERENCES t_user(user_id)
) COMMENT='会话表';
```

---

#### 3.2.3 消息表（t_message）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| message_id | VARCHAR(32) | 消息ID | PRIMARY KEY |
| session_id | VARCHAR(32) | 会话ID | FOREIGN KEY |
| content | TEXT | 消息内容 | NOT NULL |
| type | VARCHAR(20) | 消息类型 | NOT NULL |
| sender | VARCHAR(20) | 发送者 | NOT NULL |
| create_time | DATETIME | 创建时间 | NOT NULL |

**建表语句**：

```sql
CREATE TABLE t_message (
  message_id VARCHAR(32) PRIMARY KEY COMMENT '消息ID',
  session_id VARCHAR(32) NOT NULL COMMENT '会话ID',
  content TEXT NOT NULL COMMENT '消息内容',
  type VARCHAR(20) NOT NULL COMMENT '消息类型',
  sender VARCHAR(20) NOT NULL COMMENT '发送者',
  create_time DATETIME NOT NULL COMMENT '创建时间',
  FOREIGN KEY (session_id) REFERENCES t_session(session_id)
) COMMENT='消息表';
```

### 5.2 安全策略

#### 5.2.1 认证授权

| 策略 | 说明 |
|------|------|
| 认证方式 | JWT Token + OAuth2.0 |
| 密码策略 | 8位以上，包含大小写字母、数字、特殊字符 |
| 会话管理 | Token有效期2小时，自动续期 |

#### 5.2.2 数据安全

| 策略 | 说明 |
|------|------|
| 数据加密 | AES-256加密 |
| 传输加密 | HTTPS/TLS 1.3 |
| 数据脱敏 | 敏感数据脱敏展示 |

#### 5.2.3 接口安全

| 策略 | 说明 |
|------|------|
| 接口鉴权 | Token验证 |
| 限流控制 | 每分钟100次 |
| 防重放攻击 | 请求时间戳 + 签名 |

---

### 5.3 安全措施

| 安全措施 | 说明 |
|---------|------|
| 防SQL注入 | 参数化查询 |
| 防XSS攻击 | 输入过滤、输出编码 |
| 防CSRF攻击 | Token验证 |
| 防DDoS攻击 | 限流、熔断、降级 |
