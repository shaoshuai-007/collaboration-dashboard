## 三、使用流程

### 3.1 步骤1：需求文档

**输入**：客户需求（口头、邮件、会议纪要）

**使用模板**：`步骤1-需求文档模板.md`

**操作步骤**：
1. 打开模板文件
2. 根据客户需求填写各章节
3. 重点填写：需求背景、需求描述、需求目标
4. 导出为Word格式

**输出**：需求文档

---

### 3.2 步骤2：需求分析 → 思维导图

**输入**：步骤1输出的需求文档

**使用模板**：`步骤2-思维导图模板.md`

**操作步骤**：
1. 打开模板文件，理解思维导图结构
2. 使用Xmind或其他思维导图工具
3. 按照模板结构填写：
   - 需求理解（核心需求、次要需求、隐性需求）
   - 痛点分析（效率痛点、体验痛点、成本痛点）
   - 解决方案想法（至少3个）
   - 优先级评估（影响度、可行性）
4. 导出为Xmind/PNG格式

**输出**：思维导图

---

### 3.3 步骤3：方案举措Excel

**输入**：步骤2输出的思维导图

**使用模板**：`步骤3-方案举措Excel模板.xlsx`

**操作步骤**：
1. 打开Excel模板
2. 在Sheet1（方案举措清单）填写：
   - 方案标题
   - 建设目标
   - 具体举措
   - 预期效果
   - PPT页码
3. 在Sheet2（PPT大纲）调整PPT结构
4. 在Sheet3（方案汇总）填写汇总信息
5. 保存Excel文件

**输出**：方案举措Excel（PPT素材）

---

### 3.4 步骤4：方案PPT

**输入**：步骤3输出的方案举措Excel + PPT模板

**使用模板**：`步骤4-PPT模板说明.md`

**操作步骤**：
1. 阅读PPT模板说明
2. 选择合适的PPT模板（技术方案/业务方案/产品方案/综合方案）
3. 根据步骤3的PPT大纲填写PPT内容
4. 重点填写：
   - 方案背景（来自步骤1需求文档）
   - 方案目标（来自步骤2思维导图）
   - 解决举措（来自步骤3方案举措Excel）
   - 实施计划（来自步骤2思维导图）
   - 预期效果（来自步骤3方案举措Excel）
5. 美化设计，导出为PPTX/PDF格式

**输出**：客户评审方案PPT

---

### 3.5 步骤5：详细设计文档（评审通过后）

**输入**：评审通过的PPT方案

**使用模板**：`步骤5-详细设计文档模板.md`

**操作步骤**：
1. 打开模板文件
2. 根据评审通过的PPT方案填写各章节
3. 重点填写：
   - 系统架构设计（总体架构、技术架构、部署架构）
   - 接口设计（接口规范、接口列表、接口示例）
   - 数据库设计（概念模型、逻辑模型、物理模型）
   - UI设计（界面原型、交互设计、视觉设计）
   - 安全设计（安全架构、安全策略、安全措施）
4. 导出为Word格式

**输出**：详细设计文档

---

### 3.6 步骤6：项目管控计划

**输入**：步骤5输出的详细设计文档

**使用模板**：`步骤6-项目管控计划模板.xlsx`

**操作步骤**：
1. 打开Excel模板
2. 在Sheet1（甘特图）填写：
   - 任务编号、任务名称
   - 负责人、开始时间、结束时间
   - 工期、前置任务、状态、完成度
3. 在Sheet2（资源表）填写：
   - 人力资源、设备资源、预算资源
4. 在Sheet3（RACI矩阵）填写：
   - 任务、角色责任分配
5. 在Sheet4（风险清单）填写：
   - 风险项、风险等级、应对措施
6. 在Sheet5（沟通计划）填写：
   - 沟通事项、沟通频率、沟通对象
7. 保存Excel文件

**输出**：项目管控计划

## 四、方法论库使用

**文档**：`指南针工程方法论库.md`

**内容**：
- 需求阶段方法论：访谈法、问卷法、用户故事法、用例分析法、痛点分析法
- 方案阶段方法论：SCAMPER法、六顶思考帽、SMART原则、金字塔原理
- 项目阶段方法论：WBS分解法、RACI矩阵、甘特图、风险管理方法

**使用方式**：
- 每个步骤开始前，先阅读对应的方法论
- 按照方法论指导开展工作
- 确保输出质量符合标准

## 六、总结

### 6.1 核心价值

| 价值 | 说明 |
|------|------|
| **标准化** | 统一模板、统一格式、统一标准 |
| **专业化** | 专业内容、专业方法、专业输出 |
| **效率化** | 快速上手、高效执行、减少返工 |
| **质量化** | 质量有保障、结果可预期 |

### 6.2 使用建议

1. **先理解后使用**：先阅读模板和方法论，理解后再使用
2. **灵活调整**：模板是参考，可根据实际情况调整
3. **持续优化**：使用中发现问题及时反馈，持续优化模板
