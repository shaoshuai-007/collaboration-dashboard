### 2.4 优先级评估

```
4. 优先级评估
├─ 4.1 影响度评估
│  ├─ 评分标准：
│  │  ├─ 5分：影响所有用户，核心业务
│  │  ├─ 4分：影响大部分用户，重要业务
│  │  ├─ 3分：影响部分用户，一般业务
│  │  ├─ 2分：影响少量用户，次要业务
│  │  └─ 1分：影响个别用户，辅助业务
│  └─ 评估结果：
│     ├─ 想法1：[分数]
│     ├─ 想法2：[分数]
│     └─ 想法3：[分数]
├─ 4.2 可行性评估
│  ├─ 评分标准：
│  │  ├─ 5分：技术成熟，资源充足
│  │  ├─ 4分：技术较成熟，资源较充足
│  │  ├─ 3分：技术一般，资源一般
│  │  ├─ 2分：技术较新，资源较紧张
│  │  └─ 1分：技术不成熟，资源不足
│  └─ 评估结果：
│     ├─ 想法1：[分数]
│     ├─ 想法2：[分数]
│     └─ 想法3：[分数]
└─ 4.3 综合评估
   ├─ 评估方法：综合得分 = 影响度 × 0.6 + 可行性 × 0.4
   └─ 评估结果：
      ├─ 想法1：[综合得分] - [优先级]
      ├─ 想法2：[综合得分] - [优先级]
      └─ 想法3：[综合得分] - [优先级]
```

**示例**：

```
4. 优先级评估
├─ 4.1 影响度评估
│  ├─ 想法1：RAG检索增强问答系统 - 5分
│  ├─ 想法2：知识图谱增强查询系统 - 4分
│  ├─ 想法3：流程自动化办理系统 - 5分
│  └─ 想法4：规则引擎诊断系统 - 4分
├─ 4.2 可行性评估
│  ├─ 想法1：RAG检索增强问答系统 - 4分
│  ├─ 想法2：知识图谱增强查询系统 - 3分
│  ├─ 想法3：流程自动化办理系统 - 4分
│  └─ 想法4：规则引擎诊断系统 - 5分
└─ 4.3 综合评估
   ├─ 想法1：5×0.6 + 4×0.4 = 4.6 - P1
   ├─ 想法2：4×0.6 + 3×0.4 = 3.6 - P2
   ├─ 想法3：5×0.6 + 4×0.4 = 4.6 - P1
   └─ 想法4：4×0.6 + 5×0.4 = 4.4 - P1
```

## 三、方法论说明

### 3.1 需求分析方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 用户故事 | 作为xxx，我想要xxx，以便xxx | 功能需求分析 |
| 用例分析 | 参与者、用例、流程、结果 | 业务流程分析 |
| 痛点分析 | 痛点识别、痛点分析、痛点解决 | 问题导向分析 |

### 3.2 优先级评估方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 影响度评估 | 评估对业务的影响程度 | 所有想法 |
| 可行性评估 | 评估技术可行性、资源可行性 | 所有想法 |
| 综合评估 | 影响度 × 0.6 + 可行性 × 0.4 | 最终优先级确定 |
