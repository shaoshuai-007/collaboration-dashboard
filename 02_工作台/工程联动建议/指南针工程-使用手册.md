## 二、八大技能使用指南

### 2.1 采薇（需求分析师）🌸

**触发词**：需求文档、需求分析

**使用方式**：
```
用户：帮我生成需求文档

采薇：请提供以下信息：
      1. 项目背景
      2. 核心需求
      3. 目标用户
      
用户：项目是xxx，核心需求是xxx...

采薇：正在生成需求文档...
      ✅ 需求文档已生成：需求文档.docx
```

**输出格式**：Word文档（.docx）

---

### 2.2 织锦（架构设计师）🧵

**触发词**：思维导图、分析图

**使用方式**：
```
用户：生成思维导图

织锦：请提供需求文档或需求描述...

用户：（上传需求文档或输入需求）

织锦：正在生成思维导图...
      ✅ 思维导图已生成：思维导图.html
```

**输出格式**：HTML网页（可在浏览器中打开）

---

### 2.3 筑台（售前工程师）🏗️

**触发词**：方案举措、方案表格

**使用方式**：
```
用户：生成方案举措

筑台：正在根据思维导图生成方案举措...
      ✅ 方案举措已生成：方案举措.xlsx
```

**输出格式**：Excel表格（.xlsx）

---

### 2.4 呈彩（方案设计师）🎨

**触发词**：PPT、方案PPT、汇报PPT

**使用方式**：
```
用户：生成PPT

呈彩：正在根据方案举措生成PPT...
      ✅ 方案PPT已生成：方案PPT.pptx
      
      注意：PPT采用三原色定理配色
      - 主色：电信红（60%）
      - 辅色：深蓝（30%）
      - 中性色：灰色（10%）
```

**输出格式**：PowerPoint演示文稿（.pptx）

**特色**：
- 严格遵守三原色定理
- 每页有故事主线
- 左右布局，图文呼应

---

### 2.5 工尺（系统设计师）📐

**触发词**：详细设计、设计文档

**使用方式**：
```
用户：生成详细设计

工尺：请确认PPT方案已评审通过...
      回复"确认"继续

用户：确认

工尺：正在生成详细设计文档...
      ✅ 详细设计已生成：详细设计.docx
```

**输出格式**：Word文档（.docx）

**包含内容**：
- 系统架构设计
- 接口设计
- 数据库设计
- UI设计
- 安全设计

---

### 2.6 玉衡（项目经理）⚖️

**触发词**：项目管控、甘特图、RACI

**使用方式**：
```
用户：生成项目管控

玉衡：正在生成项目管控计划...
      ✅ 项目管控已生成：项目管控.xlsx
      
      包含：
      - 甘特图（Sheet1）
      - RACI矩阵（Sheet2）
      - 风险清单（Sheet3）
      - 资源表（Sheet4）
      - 沟通计划（Sheet5）
```

**输出格式**：Excel表格（.xlsx，5个Sheet）

---

### 2.7 折桂（资源管家）📚

**触发词**：配色方案、审核标准、方法论

**使用方式**：
```
用户：配色方案是什么？

折桂：三原色定理（60-30-10）：
      - 主色（60%）：电信红 #C93832
      - 辅色（30%）：深蓝 #006EBD
      - 中性色（10%）：灰色 #595959

用户：审核标准是什么？

折桂：审核等级：
      - A级（90-100分）：直接通过
      - B级（80-89分）：小幅优化
      - C级（70-79分）：重新审核
      - D级（<70分）：重新编写
```

---

**使用方式**：

## 四、常见问题

### Q1：如何修改配色方案？

**A**：所有配色由折桂统一管理，修改 `/root/.openclaw/skills/compass-shared/references/配色方案.md` 即可全局生效。

### Q2：如何自定义PPT模板？

**A**：修改 `/root/.openclaw/skills/compass-ppt/scripts/generate_ppt.py` 脚本中的模板代码。

### Q3：如何添加新的审核标准？

**A**：修改 `/root/.openclaw/skills/compass-shared/references/审核标准.md` 文件。

### Q4：如何在群聊中使用？

**A**：在群聊中 @对应的机器人，然后发送指令即可。
