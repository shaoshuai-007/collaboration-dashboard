# 六步骤审核清单（快速检查版）

> **使用说明**：每个步骤完成后，逐项检查，全部✅才能进入下一步

---

## 步骤1：需求文档审核清单

### 输入审核

```
□ 需求来源明确（客户访谈/邮件/会议纪要）
□ 需求描述清晰，无模糊表述
□ 需求信息完整，包含背景、目标、范围
□ 需求方已确认需求内容
□ 需求优先级已明确
```

### 输出审核

```
□ 需求背景（项目背景、业务现状、问题分析）
□ 需求描述（功能需求、非功能需求、业务场景）
□ 需求目标（业务目标、技术目标、量化指标）
□ 约束条件（时间、成本、资源、技术）
□ 验收标准（功能、性能、业务）
□ 文档格式符合模板
```

### 质量检查

```
□ 内容正确，无错误
□ 逻辑清晰，因果关系明确
□ 表述规范，专业术语正确
□ 数据准确，无事实错误
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 步骤2：思维导图审核清单

### 输入审核

```
□ 步骤1需求文档已完成（评分≥70分）
□ 已充分理解需求文档内容
□ 已提取核心需求、痛点、目标
□ 已准备头脑风暴素材
```

### 输出审核

```
□ 需求理解（核心需求、次要需求、隐性需求）
□ 痛点分析（效率痛点、体验痛点、成本痛点）
□ 解决方案想法（≥3个）
□ 优先级评估（影响度、可行性、综合评估）
□ 结构完整，符合模板
```

### 质量检查

```
□ 想法可行，技术和资源可行
□ 评估合理，标准和方法正确
□ 优先级正确，P1方案明确
□ 与需求文档一致
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 步骤3：方案举措Excel审核清单

### 输入审核

```
□ 步骤2思维导图已完成（评分≥70分）
□ 优先级评估已完成，P1方案已确定
□ 已提取解决方案想法
□ 已准备方案举措素材
```

### 输出审核

```
□ 方案标题明确、简洁
□ 建设目标明确、可量化（SMART原则）
□ 具体举措详细、可执行（≥3条）
□ 预期效果量化、可衡量
□ PPT页码对应正确
□ PPT大纲结构完整（20页）
```

### 质量检查

```
□ 目标量化，有具体数字
□ 举措可行，技术和资源可行
□ 效果量化，有具体指标
□ 与思维导图一致
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 步骤4：PPT审核清单

### 输入审核

```
□ 步骤3方案举措Excel已完成（评分≥70分）
□ PPT大纲已明确（20页结构）
□ 方案举措素材已准备
□ 设计风格已确定
```

### 输出审核

```
□ 封面完整（标题、副标题、日期）
□ 目录清晰（章节明确）
□ 方案背景完整（背景、痛点、必要性）
□ 方案目标明确（业务、技术、量化目标）
□ 解决举措完整（≥4个方案，每个2页）
□ 实施计划可行（阶段、里程碑、资源）
□ 预期效果量化（有具体指标）
□ 风险保障完善（风险、应对措施）
□ 总结完整（核心价值、下一步）
□ 页码连续（1-20页）
```

### 质量检查

```
□ 格式统一（字体、颜色、布局）
□ 图表清晰（可读、数据准确）
□ 逻辑清晰（因果关系明确）
□ 视觉美观（设计专业、配色协调）
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 步骤5：详细设计文档审核清单

### 输入审核

```
□ PPT方案已评审通过（客户/领导确认）
□ 方案内容已确认
□ 评审意见已整理
□ 设计要求已明确
```

### 输出审核

```
□ 系统架构完整（总体、技术、部署架构）
□ 接口设计完整（规范、列表、示例）
□ 数据库设计完整（概念、逻辑、物理模型）
□ UI设计完整（原型、交互、视觉设计）
□ 安全设计完整（架构、策略、措施）
□ 附录完整（名词解释、参考文档）
```

### 质量检查

```
□ 架构合理，满足业务需求
□ 接口规范，符合RESTful规范
□ 设计可行，技术和资源可行
□ 与PPT方案一致
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 步骤6：项目管控计划审核清单

### 输入审核

```
□ 步骤5详细设计文档已完成（评分≥70分）
□ 项目资源已确认（人力、预算、时间）
□ 项目团队已组建
□ 项目启动条件已满足
```

### 输出审核

```
□ 甘特图完整（任务、时间、负责人、进度）
□ 资源表完整（人力、设备、预算）
□ RACI矩阵完整（任务、角色责任）
□ 风险清单完整（风险、应对措施）
□ 沟通计划完整（频率、对象、内容）
```

### 质量检查

```
□ 计划可行，时间和资源可行
□ 风险可控，应对措施有效
□ 责任明确，RACI矩阵清晰
□ 与详细设计文档一致
```

**审核结论**：□ A级（≥90分） □ B级（80-89分） □ C级（70-79分） □ D级（<70分）

---

## 审核等级说明

| 等级 | 得分范围 | 说明 | 处理方式 |
|:----:|:--------:|------|----------|
| A级 | 90-100分 | 优秀 | 直接进入下一步 |
| B级 | 80-89分 | 良好 | 小幅优化后进入下一步 |
| C级 | 70-79分 | 合格 | 需要优化后进入下一步 |
| D级 | <70分 | 不合格 | 需要重新编写 |
