## 二、审核流程

### 2.1 标准审核流程

```
步骤开始
    ↓
输入审核 ←← 不合格 ←← 返工
    ↓
通过
    ↓
执行步骤
    ↓
输出审核 ←← 不合格 ←← 返工
    ↓
通过
    ↓
评分
    ↓
┌──────────────────┐
│  A级（≥90分）   │→ 直接进入下一步
│  B级（80-89分） │→ 小幅优化后进入
│  C级（70-79分） │→ 优化后进入
│  D级（<70分）   │→ 重新编写
└──────────────────┘
```

---

### 2.2 审核触发条件

| 触发条件 | 触发动作 |
|----------|----------|
| 步骤开始前 | 触发输入审核 |
| 步骤完成后 | 触发输出审核 |
| 评分<70分 | 触发返工流程 |
| 评分≥70分 | 进入下一步骤 |

## 三、审核等级

| 等级 | 得分范围 | 说明 | 处理方式 |
|:----:|:--------:|------|----------|
| A级 | 90-100分 | 优秀，可直接进入下一步 | 直接进入下一步 |
| B级 | 80-89分 | 良好，小幅优化后进入下一步 | 小幅优化 |
| C级 | 70-79分 | 合格，需要优化后进入下一步 | 需要优化 |
| D级 | <70分 | 不合格，需要重新编写 | 重新编写 |

## 四、使用指南

### 4.1 快速审核（人工）

**工具**：`六步骤审核清单.md`

**步骤**：
1. 打开对应的步骤审核清单
2. 逐项检查输入审核项
3. 逐项检查输出审核项
4. 进行质量检查
5. 确定审核等级
6. 根据等级决定是否进入下一步

---

### 4.2 自动审核（工具）

**工具**：`compass_auditor.py`

**步骤**：
1. 运行Python脚本
2. 输入各审核项得分
3. 自动计算总分和等级
4. 生成审核报告

**示例**：

```python
from compass_auditor import CompassAuditor

# 创建审核工具实例
auditor = CompassAuditor()

# 输入审核
input_checks = ["需求来源明确", "需求描述清晰", ...]
input_result = auditor.audit_input("step1", input_checks)

# 输出审核
scores = {
    "需求背景": 18,
    "需求描述": 25,
    ...
}
output_result = auditor.audit_output("step1", scores)

# 生成审核报告
report = auditor.generate_report("step1", input_result, output_result)
print(report)
```

---

### 4.3 详细审核（标准）

**工具**：`指南针工程质量审核模型.md`

**步骤**：
1. 阅读审核模型，理解审核标准
2. 按照审核标准逐项审核
3. 计算各项得分
4. 确定审核等级

## 六、质量保障机制

### 6.1 审核责任

| 角色 | 职责 |
|------|------|
| **执行者** | 自检，确保输出符合标准 |
| **审核者** | 审核输出，确定审核等级 |
| **项目经理** | 监督审核流程，处理不合格情况 |

---

### 6.2 审核频率

| 审核类型 | 频率 | 触发条件 |
|----------|------|----------|
| 输入审核 | 每次步骤开始前 | 步骤开始触发 |
| 输出审核 | 每次步骤完成后 | 步骤完成触发 |
| 专项审核 | 按需 | 发现问题时 |

---

### 6.3 问题处理

| 问题等级 | 处理方式 |
|----------|----------|
| **A级问题**（评分≥90） | 无需处理，直接进入下一步 |
| **B级问题**（评分80-89） | 小幅优化，优化后进入下一步 |
| **C级问题**（评分70-79） | 需要优化，优化后重新审核 |
| **D级问题**（评分<70） | 重新编写，重新审核 |
