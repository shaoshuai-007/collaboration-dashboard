## 一、审核模型概述

### 1.1 审核目标

| 目标 | 说明 |
|------|------|
| **质量保障** | 确保每个步骤的输出符合标准 |
| **流程规范** | 确保流程按标准执行 |
| **风险预防** | 提前发现问题，避免返工 |
| **持续改进** | 通过审核发现问题，持续优化 |

---

### 1.2 审核架构

```
┌─────────────────────────────────────────────────────────────┐
│                    质量审核体系                              │
├─────────────────────────────────────────────────────────────┤
│  输入审核  │  过程审核  │  输出审核  │  结果审核            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    审核模型                                  │
├─────────────────────────────────────────────────────────────┤
│ 步骤1审核模型 │ 步骤2审核模型 │ ... │ 步骤6审核模型         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    审核工具                                  │
├─────────────────────────────────────────────────────────────┤
│ 检查清单 │ 评分标准 │ 审核报告 │ 改进建议                  │
└─────────────────────────────────────────────────────────────┘
```

---

### 1.3 审核维度

| 维度 | 说明 | 权重 |
|------|------|:----:|
| **完整性** | 内容是否完整，无遗漏 | 30% |
| **正确性** | 内容是否正确，无错误 | 30% |
| **规范性** | 格式是否规范，符合标准 | 20% |
| **一致性** | 内容是否一致，无冲突 | 20% |

## 二、步骤1：需求文档审核模型

### 2.1 输入审核

| 审核项 | 审核标准 | 通过条件 |
|--------|----------|----------|
| **需求来源** | 需求来源是否明确 | 有明确的客户/部门/会议 |
| **需求清晰度** | 需求描述是否清晰 | 无模糊、歧义表述 |
| **需求完整性** | 需求信息是否完整 | 包含背景、目标、范围 |

**输入审核清单**：

```
□ 需求来源明确（客户访谈/邮件/会议纪要）
□ 需求描述清晰，无模糊表述
□ 需求信息完整，包含背景、目标、范围
□ 需求方已确认需求内容
□ 需求优先级已明确
```

---

### 2.2 输出审核

#### 2.2.1 内容审核

| 审核项 | 审核标准 | 分值 | 通过条件 |
|--------|----------|:----:|----------|
| **需求背景** | 是否包含项目背景、业务现状、问题分析 | 20分 | 三个要素齐全 |
| **需求描述** | 是否包含功能需求、非功能需求、业务场景 | 30分 | 三个要素齐全 |
| **需求目标** | 是否包含业务目标、技术目标、量化指标 | 20分 | 三个要素齐全 |
| **约束条件** | 是否包含时间、成本、资源、技术约束 | 10分 | 四个要素齐全 |
| **验收标准** | 是否包含功能、性能、业务验收标准 | 20分 | 三个要素齐全 |

#### 2.2.2 格式审核

| 审核项 | 审核标准 | 通过条件 |
|--------|----------|----------|
| **文档格式** | 是否符合模板格式 | 按模板填写 |
| **表格格式** | 表格是否规范 | 列对齐、边框完整 |
| **编号规范** | 章节编号是否规范 | 按模板编号 |

#### 2.2.3 质量审核

| 审核项 | 审核标准 | 通过条件 |
|--------|----------|----------|
| **内容正确** | 内容是否正确，无错误 | 无事实错误、数据错误 |
| **逻辑清晰** | 逻辑是否清晰 | 因果关系明确 |
| **表述规范** | 表述是否规范 | 专业术语正确 |

---

**评分标准**：

| 得分范围 | 等级 | 说明 |
|:--------:|:----:|------|
| 90-100分 | A级 | 优秀，可直接进入下一步 |
| 80-89分 | B级 | 良好，小幅优化后进入下一步 |
| 70-79分 | C级 | 合格，需要优化后进入下一步 |
| <70分 | D级 | 不合格，需要重新编写 |

## 三、步骤2：思维导图审核模型

