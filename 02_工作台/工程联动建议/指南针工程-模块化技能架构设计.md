## 一、设计愿景

### 少帅的宏伟蓝图

```
┌─────────────────────────────────────────────────────────────┐
│                    指南针工程技能生态                        │
│         独立技能 → 技能调用 → 独立机器人 → 自动化工作        │
└─────────────────────────────────────────────────────────────┘
```

| 目标 | 说明 |
|------|------|
| **模块化** | 6个步骤 → 6个独立技能 |
| **可调用** | 技能之间可以相互调用 |
| **可复用** | 通用内容统一引用 |
| **可独立** | 每个技能可部署为独立机器人 |
| **可自动化** | 建立群主，自动化工作流 |

## 三、各技能详细设计

### 3.1 步骤1：需求文档技能 (compass-needdoc)

```yaml
name: compass-needdoc
description: |
  需求文档生成技能。
  输入：客户需求（口头/邮件/会议纪要）
  输出：需求文档（Word）
  
  触发场景：
  - 用户提到"需求文档"、"需求分析"
  - 用户提供需求素材，要求生成文档
  - 指南针工程步骤1
```

**目录结构**：
```
compass-needdoc/
├── SKILL.md                    # 技能说明
├── scripts/
│   └── generate_needdoc.py     # Word生成脚本
├── references/
│   ├── 需求文档模板.md          # 文档结构
│   └── 需求调研方法.md          # 方法论
└── assets/
    └── needdoc_template.docx   # Word模板
```

**调用关系**：
```
输入：用户需求素材
处理：结构化需求 → 填充模板 → 生成文档
输出：需求文档.docx
```

## 五、协调器技能 (compass-coordinator)

### 5.1 设计目标

```yaml
name: compass-coordinator
description: |
  指南针工程协调器。
  功能：流程编排、状态管理、自动化
  
  触发场景：
  - 用户要求"全流程"、"一键生成"
  - 用户要求"自动化工作流"
  - 多个技能需要协调执行
```

### 5.2 目录结构

```
compass-coordinator/
├── SKILL.md                    # 协调器说明
├── scripts/
│   ├── run_full_pipeline.py    # 全流程执行
│   ├── check_status.py         # 状态检查
│   └── auto_schedule.py        # 自动排程
├── references/
│   ├── 流程编排规则.md          # 步骤调用顺序
│   └── 状态管理规则.md          # 进度跟踪
└── config/
    └── pipeline.yaml           # 流程配置
```

### 5.3 流程编排

```yaml
# pipeline.yaml
pipeline:
  name: 指南针工程全流程
  steps:
    - name: needdoc
      skill: compass-needdoc
      input: 用户需求
      output: 需求文档.docx
      
    - name: mindmap
      skill: compass-mindmap
      input: 需求文档.docx
      output: 思维导图.html
      
    - name: solution
      skill: compass-solution
      input: 思维导图.html
      output: 方案举措.xlsx
      
    - name: ppt
      skill: compass-ppt
      input: 方案举措.xlsx
      output: 方案PPT.pptx
      
    - name: design
      skill: compass-design
      input: 方案PPT.pptx
      output: 详细设计.docx
      condition: 评审通过
      
    - name: project
      skill: compass-project
      input: 详细设计.docx
      output: 项目管控.xlsx
```

### 5.4 自动化工作流

```
┌─────────────────────────────────────────────────────────────┐
│                    自动化工作流                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
    用户输入需求 → compass-coordinator
                            ↓
    ┌─────────────────────────────────────────────┐
    │ 步骤1: compass-needdoc → 需求文档.docx      │
    │       ↓                                     │
    │ 步骤2: compass-mindmap → 思维导图.html      │
    │       ↓                                     │
    │ 步骤3: compass-solution → 方案举措.xlsx     │
    │       ↓                                     │
    │ 步骤4: compass-ppt → 方案PPT.pptx           │
    │       ↓                                     │
    │ 【人工评审点】                               │
    │       ↓                                     │
    │ 步骤5: compass-design → 详细设计.docx       │
    │       ↓                                     │
    │ 步骤6: compass-project → 项目管控.xlsx      │
    └─────────────────────────────────────────────┘
                            ↓
    输出全部文档 → 发送邮件/消息通知
```
