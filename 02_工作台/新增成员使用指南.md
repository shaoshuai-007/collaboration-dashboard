# 新增成员使用指南

## 一、快速使用（推荐）

### 方式1：交互式创建

```bash
python /root/.openclaw/workspace/scripts/add_new_member.py
```

按提示输入：
1. 选择角色类型（数字或自定义）
2. 输入Emoji
3. 选择名字
4. 选择核心技能
5. 输入角色描述
6. 确认创建

**时间**：约2分钟

---

### 方式2：命令行快速创建

```bash
python /root/.openclaw/workspace/scripts/add_new_member.py \
    --role "测试工程师" \
    --name "试剑" \
    --emoji "⚔️" \
    --skill "coding-agent" \
    --description "测试工程师，负责自动化测试、质量保障"
```

**时间**：约10秒

---

### 方式3：Shell脚本创建

```bash
./root/.openclaw/workspace/scripts/add_member.sh "测试工程师" "试剑" "⚔️" "coding-agent"
```

**时间**：约10秒

## 四、创建后检查清单

创建完成后，检查以下内容：

| 检查项 | 文件 | 说明 |
|--------|------|------|
| ✅ 技能定义 | SKILL.md | 功能描述是否完整 |
| ✅ 身份定义 | IDENTITY.md | 角色定位是否清晰 |
| ✅ 用户偏好 | USER.md | 少帅偏好是否准确 |
| ✅ 工具说明 | TOOLS.md | 工具使用是否清晰 |
| ✅ 灵魂风格 | SOUL.md | 价值观是否体现 |
| ✅ 参考资料 | references/README.md | 案例是否充实 |
| ✅ 脚本工具 | scripts/xxx_tool.py | 工具是否可用 |
| ✅ 模板资源 | assets/template.md | 模板是否规范 |

## 五、创建后必做事项

### 5.1 完善SKILL.md

编辑 `{技能路径}/SKILL.md`：
- 补充功能描述
- 补充触发场景
- 补充输出规范

### 5.2 补充参考资料

在 `{技能路径}/references/` 中添加：
- 真实案例
- 方法论文档
- 学习资源

### 5.3 开发工具脚本

在 `{技能路径}/scripts/` 中开发：
- 主工具脚本
- 辅助工具
- 测试脚本

### 5.4 配置师徒带教

为新成员指定导师：
- 技术类：织锦、工尺带教
- 分析类：采薇、筑台带教

## 八、注意事项

1. **避免重复**：创建前检查是否已存在相同角色
