## 第二部分：周计划管理SOP

### 📋 周计划管理流程

```
周初计划 → 日分解执行 → 周中检查 → 周末闭环 → 周总结归档
```

### 1. 周计划制定（周一）

**输入**：
- 周报Excel数据
- 上周遗留问题
- 本周重点任务

**输出**：
- 周计划看板（`周计划管理/2026-XX-第X周计划.md`）
- 任务清单（`周计划管理/任务清单.csv`）
- 日计划（`周计划管理/日计划/YYYY-MM-DD.md`）

**责任人**：南乔

**步骤**：
1. 提取周报数据，识别本周重点任务
2. 按条线分解任务，明确责任人
3. 评估风险，制定应对策略
4. 将任务分解到每日，形成日计划

### 2. 日计划执行（每日）

**输入**：
- 周计划任务分解
- 当日优先级

**输出**：
- 日工作日志
- 任务状态更新

**责任人**：各条线责任人

**步骤**：
1. 晨会确认当日重点（9:30 QQ提醒）
2. 执行任务，记录进展
3. 晚间更新状态，识别问题
4. 问题上报，风险预警

### 3. 周中检查（周三）

**输入**：
- 周一/二执行情况
- 风险问题清单

**输出**：
- 进度检查报告
- 风险应对措施

**责任人**：南乔

**步骤**：
1. 汇总各条线进度
2. 识别延期任务
3. 评估风险等级
4. 制定应对措施

### 4. 周末闭环（周五）

**输入**：
- 本周任务执行情况
- 遗留问题清单

**输出**：
- 周总结报告
- 下周计划建议

**责任人**：南乔

**步骤**：
1. 汇总本周完成情况
2. 统计关键指标（完成率、风险数）
3. 识别遗留问题
4. 形成下周计划建议

### 5. 周总结归档

**输入**：
- 周计划文档
- 执行日志

**输出**：
- 归档文件（`周计划管理/2026-XX-第X周计划.md`）
- 经验沉淀

**责任人**：南乔

### 📊 任务状态定义

| 状态 | 含义 | 标识 | 处理方式 |
|------|------|:----:|---------|
| 未开始 | 尚未启动 | ⚪ | 按计划执行 |
| 进行中 | 正在执行 | 🔵 | 每日跟踪 |
| 已完成 | 任务完成 | ✅ | 归档总结 |
| 有风险 | 存在阻塞 | 🔴 | 升级处理 |
| 已挂起 | 暂时搁置 | ⏸️ | 待条件恢复 |

### ⚠️ 风险预警机制

| 级别 | 触发条件 | 处理方式 | 上报对象 |
|:----:|---------|---------|---------|
| 🔴 高 | 延期超3天/阻塞严重 | 立即升级汇报 | 少帅 |
| 🟡 中 | 延期1-3天/有风险 | 每日跟踪 | 条线负责人 |
| 🟢 低 | 按计划进行 | 周报跟踪 | 南乔 |
