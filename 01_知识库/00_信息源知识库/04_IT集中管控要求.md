# IT集中建设管控要求

---

## 一、IT集中建设原则

### 1.1 总体原则

| 原则 | 内容 |
|------|------|
| **纵向整合中心能力** | 汇聚31省及总部分散的BSS/OSS能力中心，一点建设集中的业务中台 |
| **横向收敛门户入口** | 打造外部客户统一APP、内部角色4大APP及7大工作台与超级智能体入口 |
| **分阶段推进IT集中** | 2026年北方5省先行先试，用5-10年逐步推动全网IT大集中 |
| **兼顾集团统一和属地个性化** | 省内因需建设个性化应用，1+N模式 |

### 1.2 试点省份

**2026年北方5省先行先试**：
- 黑龙江
- 吉林
- 辽宁
- 山西
- 天津

## 二、IT集中期间管控要求

### 2.1 投资统筹管控

**原则**：先立后破，IT集中的过渡期保障必要的全网IT投资

**要求**：
- 加强统筹：全网IT投资基于统一的规划统筹管理
- 坚持无规划不立项：各省按规划批复项目进行立项

### 2.2 架构管控

**架构规范落地**：规划项目需符合数智化大平台架构要求，系统可定位、建设不重复

### 2.3 集中范围管控

| 类别 | 管控要求 |
|------|----------|
| **集中范围内系统** | - 不允许新建系统<br>- 不允许架构重构<br>- 不允许数据模型重大变更<br>- 新增核心功能模块必须向集团报批<br>- 更换厂商必须向集团报批<br>- 试点省正式上线前必须根据要求做好全网性需求支撑 |
| **其他系统** | 业务需求驱动，根据需要自行规划 |

## 三、管控流程

### 3.1 六步管控流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 01 规划  │───>│ 02 规划  │───>│ 03 规划  │
│   编制   │    │   评审   │    │   批复   │
└──────────┘    └──────────┘    └──────────┘
                                      │
                                      ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 06 规划  │<───│ 05 项目  │<───│ 04 项目  │
│   后评估 │    │   实施   │    │   立项   │
└──────────┘    └──────────┘    └──────────┘
```

#### 01 规划编制
- 省公司遵循IT集中架构和滚动规划要求
- 编制规划方案，规划次年IT项目
- IT项目规划遵循IT集中期间建设原则

#### 02 规划评审
- 云网运营部组织架构管理专家团队评审项目
- **重点项目打标**：涉及核心功能模块的重大变更的项目、涉及更换供应商的项目和新建系统

#### 03 规划批复
- 集团公司专项批复规划项目和投资
- 批复后的项目录入规划项目清单

#### 04 项目立项
- 以集团批复的规划清单为立项依据
- 立项项目必须与规划内项目建立映射
- 立项前需按照架构治理要求完成系统准生证、健康证申请

#### 05 项目实施
- 以下情况向集团报批：立项投资超出规划投资、系统在架构中定位较规划发生变化、系统核心功能较规划发生重大变化
- 实施过程中对打标项目的架构合规性进行巡检和抽查

#### 06 规划后评估
- 每年对上年度各省规划项目立项情况、规划投资完成情况、系统架构合规性进行评估
- 评估结果纳入下年度规划批复的投资方案考虑

## 四、投资需求管理原则

### 4.1 全网性需求
- 统一制定方案，集中系统承接
- 未集中省份同步建设，保持建设方案和核心功能与集中系统一致性

### 4.2 试点省公司
- 集中系统未全面上线前及新老系统并行期，保留未集中系统规划投资及待集中系统必要的配套改造、需求支撑、运营相关预算投入

### 4.3 其他省公司
- 集中完成前IT投资保持稳定，以满足省内和各类全网性需求

## 五、规划评审要点

### 5.1 评审红线

以下项目原则上评审不通过：
- 集约范围内系统架构重构
- 模型重大变更类项目

### 5.2 重点评审内容

| 评审维度 | 评审内容 |
|----------|----------|
| 架构合规性 | 是否符合数智化大平台架构要求 |
| 投资合理性 | 投资规模是否合理，是否与规划匹配 |
| 功能必要性 | 是否为业务刚需，是否可复用集中能力 |
| 技术先进性 | 是否采用AI原生、云原生等先进技术 |
| 风险可控性 | 是否有完善的风险预案 |

## 七、规划报告要求

### 7.1 省公司规划报告内容

**第一部分：数智化平台滚动规划--主报告PPT**
1. 25年规划执行情况分析
   - 项目和投资完成情况
   - 重点举措落地情况
   - 专业亮点工作成效等
2. 规划需求和驱动力分析
3. 规划总体思路
4. 建设重点举措
   - 举措内容条理清晰，能够和项目有效衔接
5. 规划项目及2026年投资结构分析

**第二部分：主报告脱敏版**
- 删除投资等敏感信息，用于专家评审

**第三部分：规划数据报表Excel**
- 注意表格填写规范性要求

### 7.2 专业公司规划报告内容

**第一部分：数智化平台滚动规划--主报告PPT**
1. 25年规划执行情况分析
2. 规划需求和驱动力分析
3. 规划总体思路
4. 建设重点举措及相关项目
5. 规划期对总部IT集中系统的主要需求
6. 26年项目及投资计划

**第二部分：主报告脱敏版**
- 删除投资等敏感信息

**第三部分：规划项目清单表**

## 八、规划执行要求

### 8.1 省公司规划要求

1. 系统评估25年规划执行情况（投资进度、项目进度、工作成效、存在问题）
2. 基于业务和技术双轮驱动，充分调研省内业务部门需求
3. 规划举措和项目计划应符合十五五数智化平台总体架构和IT集中架构管理要求
4. 拟定项目及建设内容应与规划关键举措相对应

### 8.2 专业公司规划要求

1. 系统评估25年规划执行情况
2. 规划关键举措应符合数智化平台总体架构要求，考虑对集团IT集中工作的承接
3. 规划应充分考虑对集团已有平台/数据的复用，避免重复建设
4. 系统梳理规划期对集团集中IT系统的业务需求
