> 创建时间：2026-03-11

## 一、核心框架：DIKW四层认知升级

```
数据(Data) → 信息(Information) → 知识(Knowledge) → 智慧(Wisdom)
   ↓              ↓                  ↓                  ↓
 无序事实       建立关联            形成认知           指导行动
```

---

## 二、四层提炼方法论

### 第一层：数据采集（无序事实）

**目标**：全面收集原始数据，不遗漏关键信息

**采集清单**：
- [ ] 文件基本信息（名称、页数、来源、时间）
- [ ] 项目名称、数量统计
- [ ] 关键干系人（姓名、角色、职责）
- [ ] 时间节点（规划周期、里程碑）
- [ ] 核心指标（关键数字、目标值）
- [ ] 技术关键词（大模型、数字员工、智能体等）

**输出物**：原始数据表（Excel/Markdown表格）

---

### 第二层：信息提炼（建立关联）

**目标**：赋予数据意义，发现隐藏关联

**关联分析方法**：

| 关联类型 | 提问方式 | 示例 |
|---------|---------|------|
| **技术关联** | 这个技术点与我们的能力匹配吗？ | 产销品数字员工 ↔ AI智能配案（大模型+知识图谱） |
| **业务关联** | 这个项目与当前商机相关吗？ | OAO营销中台 ↔ 电渠转型项目 |
| **人物关联** | 这个人是关键决策者吗？ | 万必波（P20/P21）→ 产销品+营业受理双项目 |
| **竞争关联** | 这个方向与竞品有差异化吗？ | 业务数字孪生 vs 科大讯飞AI外呼 |
| **风险关联** | 这个变化会影响现有项目吗？ | 电渠关停并转 → 需转型方案 |

**输出物**：关联发现表（发现点 + 意义 + 行动建议）

---

### 第三层：知识结构化（形成认知）

**目标**：将信息组织成可复用的认知体系

**知识结构化模板**：

#### 1. 全景图结构
```
总目标/主题
├── 方向一
│   ├── 子项目1
│   ├── 子项目2
│   └── 关键能力
├── 方向二
│   └── ...
└── 方向三
    └── ...
```

#### 2. 分类矩阵
| 分类维度 | 项目A | 项目B | 项目C |
|---------|-------|-------|-------|
| 技术点 | ... | ... | ... |
| 关联商机 | ... | ... | ... |
| 优先级 | ⭐⭐⭐ | ⭐⭐ | ⭐ |
| 负责人 | ... | ... | ... |

#### 3. 技术栈清单
| 技术栈 | 应用场景 | 成熟度 | 与我司能力匹配度 |
|--------|---------|:------:|:---------------:|
| ... | ... | 生产/试点/规划 | 高/中/低 |

#### 4. 竞争态势图
```
竞争对手定位
├── 竞品A → 核心优势 + 弱点
├── 竞品B → 核心优势 + 弱点
└── 我司 → 差异化定位 + 突破口
```

**输出物**：结构化知识文档（Markdown格式）

---

### 第四层：智慧转化（指导行动）

**目标**：将知识转化为可执行的行动指南

**智慧转化模板**：

#### 1. 优先级判断模型
```
商机价值 = 技术匹配度 × 干系人权力 × 预算可能性

评分标准：
- 技术匹配度：高(3) / 中(2) / 低(1)
- 干系人权力：高(3) / 中(2) / 低(1)
- 预算可能性：高(3) / 中(2) / 低(1)

总分 ≥ 7 → 🔴顶级商机
总分 5-6 → 🟡重点商机
总分 ≤ 4 → 🟢潜力商机
```

#### 2. 差异化竞争策略
```
【竞品定位】竞品 = X专家（优势 + 弱点）
【我司定位】我司 = Y专家（差异化方向）
【竞争锚点】一句话差异化话术
```

#### 3. 干系人维护策略
```
干系人姓名 → 关注点 → 沟通策略 → 下一步动作

示例：
万必波 → 技术成熟度 → 强调AI原生智能 → 安排技术交流
万海波 → 降本增效 → 强调效率提升 → 展示ROI案例
```

#### 4. 风险预警机制
```
风险项 → 影响评估 → 预警等级 → 应对预案
```

**输出物**：行动指南（优先级清单 + 策略建议 + 风险预案）

## 四、提炼原则

### 原则1：先广后深
- 第一遍：快速扫描，标记关键信息
- 第二遍：深入分析，建立关联
- 第三遍：结构输出，形成认知

### 原则2：以终为始
- 提炼前明确目标：支撑哪个商机？解决什么问题？
- 提炼中聚焦目标：只保留与目标相关的内容
- 提炼后验证目标：输出是否直接可用？

### 原则3：可复用优先
- 优先提炼可复用的知识（方法论、技术栈、话术）
- 次要提炼一次性信息（具体数字、人名）
- 建立知识索引，方便后续调用

### 原则4：行动导向
- 每个发现都要有行动建议
- 每个知识都要有应用场景
- 每个风险都要有应对预案

## 五、质量标准

### 高质量提炼的特征

| 维度 | 标准 |
|------|------|
| **完整性** | 不遗漏关键信息，数据采集全面 |
| **准确性** | 信息关联正确，无误导性解读 |
| **结构性** | 知识结构清晰，易于理解记忆 |
| **可用性** | 行动建议具体，可直接执行 |
| **时效性** | 标注信息来源时间，便于判断新鲜度 |

### 质量自检清单

- [ ] 数据层：是否采集了所有关键数据？
- [ ] 信息层：是否发现了3个以上有价值的关联？
- [ ] 知识层：是否形成了可复用的认知体系？
- [ ] 智慧层：是否输出了具体的行动指南？
- [ ] 入库：是否保存到知识库并建立索引？

## 六、应用场景

### 场景1：规划文档提炼
- 目标：发现商机、识别干系人、理解战略方向
- 重点：项目清单、技术路线、关键指标、时间节点
- 输出：商机优先级清单 + 干系人维护策略

### 场景2：竞品方案分析
- 目标：了解竞品能力、寻找差异化机会
- 重点：技术架构、核心功能、优势劣势、案例效果
- 输出：竞品对标分析 + 差异化话术

### 场景3：行业报告提炼
- 目标：把握行业趋势、寻找市场机会
- 重点：市场规模、增长趋势、技术演进、竞争格局
- 输出：行业洞察报告 + 商机建议

### 场景4：客户需求提炼
- 目标：理解客户痛点、匹配解决方案
- 重点：业务场景、痛点描述、期望目标、预算周期
- 输出：需求分析报告 + 解决方案建议

## 七、持续进化

### 方法论迭代机制
- 每次提炼后复盘：哪些步骤可以优化？
- 定期更新模板：根据新场景调整结构
- 积累优秀案例：形成最佳实践库

### 知识库维护机制
- 定期更新：标注信息时效性，过期信息标记
- 关联索引：建立知识间的关联链接
- 版本管理：保留历史版本，便于回溯
