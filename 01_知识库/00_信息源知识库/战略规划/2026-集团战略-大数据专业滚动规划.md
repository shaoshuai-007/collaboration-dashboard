# 2026-集团战略-大数据专业滚动规划

**来源**：数据发展中心
**发布日期**：2025年11月
**原文**：大数据专业26-28滚动规划指引

---

## 一、战略背景

承接集团"十五五"规划编制工作要求，深入落实"人工智能+"行动，推进内外部高质量数据集建设，考虑DT集约实施，服务企业数智化转型，并助力数据要素流通和价值释放。

---

## 二、核心战略方向

### 2.1 十五五大数据规划视图

**核心理念**：**智慧驱动 价值赋能**

| 方向 | 内涵 | 关键举措 |
|------|------|----------|
| **对内AI+** | 从数据汇聚、开发到智能体应用实现全流程AI | 高质量数据集为AI+应用和AI开发提供保障 |
| **沉淀知识体系** | 全面构建知识管理能力和知识体系 | 纳入科创体系 |
| **对外可信流通** | 打造可信数据空间，提供全场景流通服务 | 公共数据服务、产业数据服务、跨境数据服务 |

### 2.2 五大平台规划

| 平台 | 总体目标 | 省侧要求 |
|------|----------|----------|
| **数据智能中台** | 打造央企领先的新一代数据智能中台，强化"Data+AI"双引擎驱动 | 基于自研PaaS，持续做好大数据领域XC工作 |
| **智能体平台** | 构建集团统一建设、两级协同部署的企业级智能体平台 | 聚焦本地化、场景化业务需求，"应用百花齐放" |
| **数据开放共享平台** | 构建"知数用数"的前沿阵地，统筹提供数据供给 | 省分数据与集团数据融合计算，穿透至一线赋能生产 |
| **星海可信数据空间** | 围绕高质量数据集为外部生态客户提供全栈服务 | 配合集团推进两级架构建设 |
| **数据能力平台** | 持续改造升级"湖仓融合""3+1+N"架构体系 | 融合集团统一集约能力，优先使用集团能力 |

## 三、数智化应用规划

### 3.1 经分大模型

**总体目标**：基于真实业务指标数据的查询与增强分析，替代人工汇报的步骤

| 目标 | 说明 |
|------|------|
| 构建经分智能体 | 深度融合大语言模型、增强分析技术和垂直领域知识 |
| 智能化经营问答 | 基于大模型语义理解、逻辑推理、数据学习等能力 |
| 自动化经营报告 | 基于语义理解、表生文等能力自动生成经营报告 |
| 标准化经营分析 | 自动生成经营概况PPT |

### 3.2 三全能力体系

**总体目标**：打造以客户为中心的"**全市场、全客户、全业务**"的智慧经营体系

| 目标 | 说明 |
|------|------|
| 整合内外数据 | 提升小B空间入库规模 |
| 健全客户树覆盖 | 构建"统一建设、两级运营"的体系 |
| 丰富标签体系 | 强化小程序/公众号等识别能力，提升标签时效性 |

### 3.3 战新穿透一线

**总体目标**：落实数据开放，加强数据融通，建立营销闭环，推进重点战新产品数据穿透应用至一线

- **第一批5个产品**：云电脑、天翼视联、直连卫星、量子通信、AI视频彩铃
- **第二批7个产品**：AI通信助理、AI中屏、AI云盘、AI手机、安全大脑、智慧社区、数字乡村

### 3.4 产数业财融合

**总体目标**：通过完善EDA与各省业务数据接口，以业财深度融合为核心，实现标品、非标业务的收入确认与应收数据在产品实例级精准对应

### 3.5 智慧节能

**总体目标**：进一步构建与完善云网基础设施AI节能能力体系，**2028年末AI节电量达10.5亿度**

### 3.6 穿透式监管

**总体目标**：承接国资委穿透式监管要求，面向投资、产权、财务、会计等九个领域，迭代完善穿透式监管体系

### 3.7 高质量数据集

**总体目标**：持续建设高质量多模态数据集，扩大数据集规模，提升数据集多样性，拓展领域、行业覆盖

| 目标 | 说明 |
|------|------|
| 通用数据集 | 语义数据集、语音数据集（100种方言）、视觉数据集、多模态数据集 |
| 内部数据集 | 总部23个部门加快AI+应用落地，高质量数据集全量入湖 |
| 行业数据集 | 通过ICT项目和行业交流合作，积累行业数据集 |

## 四、数据治理与安全规划

### 4.1 企业级数据治理

**总体目标**：对标DCMM能力，推动数据治理体系的强化与完善，驱动数据资源化、资源资产化、资产价值化的持续跃迁

**核心举措**：
- 构建"集团-省"两级协同的数据治理新范式
- 打造全集团唯一、统一的高质量数据集目录
- 打造智能化数据治理平台，提升数据治理能力智能化

### 4.2 数据安全规划

**总体目标**：以人和数据为核心、以数据暴露面管理为重点，构建安全运营中心、安全数据中心和安全能力池三位一体的大数据安全运营防护体系

**重点举措**：
1. 建设大数据安全运营管控一体化平台
2. 集团-省两级多方共享互促，实现穿透式监管与协同联动
3. 聚集数据暴露面和数据流动，强化数安防护原子能力
4. 建设AI安全治理能力，护航智能业务

## 五、DT集中演进思路

### 5.1 总体目标

**能力重构**，打造湖仓一体、云原生的大数据处理架构，优化实时与跨域融合的数据模型；构建全国统一的数据智能中台基线版本，全面支撑"**Data + AI**"全栈数据能力

### 5.2 三阶段推进

| 阶段 | 内容 |
|------|------|
| ① 集团新建 | 构建集团级全国一体化数据智能中台 |
| ② 数据迁移 | 试点省IT系统集中，确保业务平稳过渡 |
| ③ 应用迁移 | 推进共性应用、个性化应用集约建设 |

### 5.3 省侧规划要求

- 在DT集中前完成冗余模型、低价值应用清理瘦身
- 梳理并推动组织、人员等必要的数据标准治理工作
- 省公司原则上不进行大规模扩容、大的架构调整
- 各省大数据平台XC改造按原计划推进
