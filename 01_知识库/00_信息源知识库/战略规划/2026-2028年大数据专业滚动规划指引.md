# 2026-2028年大数据专业滚动规划指引

**来源**：中国电信数据发展中心
**发布时间**：2025年11月
**入库时间**：2026-03-10
**文件类型**：集团战略文件（P0）

---

## 一、规划背景与原则

### 规划背景
- 承接集团"十五五"规划编制工作要求
- 深入落实"人工智能+"行动
- 推进内外部高质量数据集建设
- 考虑DT集约实施，服务企业数智化转型
- 助力数据要素流通和价值释放

### 规划原则

| 原则 | 内容 |
|------|------|
| 集团、省两级协同规划 | 集团协同31省统筹开展滚动规划制定，各省根据指引开展本地化建设 |
| DT集中规划布局 | 加强省数据域投资、规划、架构统筹 |
| 数据技术创新与投资建设紧密协同 | 加大对自研能力使用和投入生产 |
| 内外部需求协同考虑 | 规划、建设应充分满足内部业务需求，支持对外赋能 |
| 投资规划闭环管控 | 加强规划和实际立项匹配，保障项目合理稳定 |

## 二、核心规划内容

### 1. 企业级数据智能中台规划

**总体目标**：打造央企领先的新一代数据智能中台，强化"Data+AI"双引擎驱动

**三大目标**：

| 目标 | 关键内容 |
|------|----------|
| 数据智能中台持续演进 | AI深度嵌入"数据-模型-应用"一体化工具，数智双向闭环 |
| 数据+AI能力全面升级 | 增强跨域数据融合，构建知识融合体系，两级智能体注智生产系统 |
| 场景化数智能力全面开放 | 打造全集团统一的AI+能力开放门户 |

**省侧要求**：
- 基于自研PaaS，持续做好大数据领域XC工作
- 基于集团数据智能中台的训推两级能力，各省全面完成接入
- 统一沉淀高质量数据集，避免重复建设与数据孤岛
- 形成"数智双向增强闭环"

---

### 2. 企业级智能体平台规划

**总体目标**：构建集团统一建设、具备两级协同部署能力的企业级智能体平台

**三大目标**：

| 目标 | 关键内容 |
|------|----------|
| 提升智能化创新效率 | 覆盖智能体全生命周期的一站式开发与管理平台 |
| 强化规模化协同效能 | 两级协同的开发、部署与运营机制，构建"智能体广场" |
| 构建AI+应用智能体驾驶舱 | 统一评价机制和数据自动回流闭环 |

**省侧要求**：
- 遵循集团统一平台规范与技术标准
- **杜绝新的烟囱式建设**
- 省公司聚焦本地化、场景化业务需求，"应用百花齐放"
- 优秀应用贡献至集团"智能体广场"，形成"共建、共享、共营"生态

---

### 3. 数据开放共享平台规划

**总体目标**：构建集团"知数用数"的前沿阵地

**三大目标**：

| 目标 | 关键内容 |
|------|----------|
| 融合AI智能推荐 | AI快捷问数和精准推荐，"千人千面"数据推送 |
| 集团、省专两级协同 | 分级授权开放的知数、用数能力 |
| 基于智能体的开放共享生态 | 覆盖数据接入、开发、挖掘、服务封装的一站式服务 |

**核心能力**：
- 数据广场、模型广场、智能体广场、案例广场
- 一点承接数据需求："需求-方案-实施-评价"全流程闭环

---

### 4. 星海可信数据空间规划

**总体目标**：围绕高质量数据集为外部生态客户提供全栈服务

**三大目标**：

| 目标 | 关键内容 |
|------|----------|
| 面向多模态的可信数据空间 | 多模态计费结算体系，端到端可信交付 |
| 全网数据要素两级架构 | "核心+区域"互通的可信流通体系 |
| 高质量数据集流通服务 | 隐私计算、数据沙箱、区块链、连接器 |

**应用场景**：
- 能源行业、通信行业、工业制造
- 交通行业、医疗健康

**省侧要求**：
- 选择集团平台交付或省分节点交付模式
- 省分负责本省数据流通、产品加工

---

### 5. 数智化应用规划

#### 5.1 经分大模型

**目标**：经分大模型在省专公司应用全覆盖

**四大能力**：
1. **智能化经营问答**：分权分域支撑集团、省、市用户查询
2. **自动化经营报告**：语义理解+表生文自动生成
3. **标准化经营分析**：大模型点评分析+小模型图表生成
4. **经分智能体**：数据获取-分析洞察-可视化呈现

#### 5.2 三全能力体系建设

**目标**：打造"全市场、全客户、全业务"智慧经营体系

**三大目标**：
1. 整合内外数据，提升小B空间入库规模
2. 健全客户树覆盖，"统一建设、两级运营"
3. 丰富标签体系，提升标签时效性

#### 5.3 战新穿透一线

**目标**：推进重点战新产品数据穿透应用至一线

**战新产品清单**：

| 批次 | 产品 | 数量 |
|------|------|:----:|
| 第一批 | 云电脑、天翼视联、直连卫星、量子通信、AI视频彩铃 | 5个 |
| 第二批 | AI通信助理、AI中屏、AI云盘、AI手机、安全大脑、智慧社区、数字乡村 | 7个 |

**省侧要求**：
- 完成七大战新业务**194个接口**接入
- 完成至少**12个重点战新产品**必选指标穿透一线

#### 5.4 智慧节能

**目标**：**2028年末AI节电量达10.5亿度**

**四大目标**：
1. AI节能引擎升级：大小模型协同，"感知-分析-决策-控制-自优"
2. 扩大纳管规模：4G/5G双模站、自研小站、5G-A、BBU
3. 扩展节能新场景：基站多元节能、IDC机楼末端+水冷主机
4. 智能绿色运营：基于意图驱动、CoT、工具、编排技术

#### 5.5 高质量数据集

**三大目标**：

| 目标 | 内容 |
|------|------|
| 通用数据集 | 语义、语音（100种方言）、视觉、多模态数据集 |
| 内部数据集 | 总部23个部门AI+应用落地，**高质量数据集全量入湖** |
| 行业数据集 | 通过ICT项目和行业交流合作积累 |

---

### 6. 企业级数据治理规划

**总体目标**：对标DCMM能力，促进数据资源化、资源资产化、资产价值化

**三大目标**：
1. 构建"集团-省"两级协同的数据治理新范式
2. 打造全集团唯一、统一的高质量数据集目录
3. 打造智能化数据治理平台

**核心能力链**：盘、建、管、营、评

---

### 7. 全网数据运营与安全规划

#### 数据运营规划

**重点举措**：

| 举措 | 内容 |
|------|------|
| 智能服务台与端到端监控 | 集团L5级，A类省L4级，BC类省L3级 |
| AI+运营 | 省公司落地**不少于5个AI+运营智能体** |
| 集中运营 | 北方五省数据集中，建成两级运营模式 |
| AI运营 | "数据-模型-应用"一体化AI开发生态 |

#### 大数据安全规划

**三道安全防线**：
1. 事前安全基线
2. 事中态势感知
3. 事后全链路溯源

---

### 8. DT集中演进思路

**目标形态**：湖仓一体、云原生的大数据处理架构

**三大目标**：
1. 构建集团级全国一体化数据智能中台
2. 试点省DT集中目标形态过渡（3~6个月）
3. 强化"Data+AI"双引擎驱动

**省侧要求**：
- DT集中前完成冗余模型、低价值应用清理瘦身
- 原则上不进行大规模扩容、架构调整
- 大数据平台XC改造按原计划推进
