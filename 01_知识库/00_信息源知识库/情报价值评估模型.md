# 情报价值评估模型（Intelligence Value Assessment Model）

**创建时间**：2026-03-11
**创建人**：南乔
**版本**：v2.0（正式版）
**设计原则**：任何结论都要有论据支撑

---

## 🎯 模型目标

从海量信息中筛选出**高价值、高可靠、高可用**的情报，构建持续进化的知识体系。

---

## 📊 评估维度（五维模型）

### 维度1：准确性（Accuracy）- 权重25%

**存在理由**：错误情报导致错误决策，后果严重
**论据**：商务谈判场景占比40%，对准确性要求最高

| 指标 | 评分标准 | 分值 |
|------|---------|:----:|------|
| **来源权威性** | ⭐⭐⭐⭐⭐ 官方/政府/权威机构 | 5 |
| | ⭐⭐⭐⭐ 头部媒体/行业标杆 | 4 |
| | ⭐⭐⭐ 行业媒体/专业平台 | 3 |
| | ⭐⭐ 一般媒体/自媒体 | 2 |
| | ⭐ 不明来源/无验证 | 1 |
| **信息可验证性** | 有明确数据来源、可交叉验证 | +1 |
| | 有官方链接、可追溯 | +0.5 |
| | 无来源、无法验证 | 0 |
| **发布时间** | ⚠️ **必填项**，确保信息新鲜度 |  |
| | **近14天内** | +1 |
| | 14-30天 | +0.5 |
| | 30-60天 | 0 |
| | 超过60天 | -0.5 |
| | **无发布时间** | **-1（扣分项）** |
| **渠道评级** | ⚠️ **新增项**，参考渠道评级模型 |  |
| | A+级渠道（≥95） | +1 |
| | A级渠道（90-94） | +0.8 |
| | B+级渠道（85-89） | +0.5 |
| | B级渠道（80-84） | 0 |
| | C级渠道（60-79） | -0.3 |
| | D级渠道（<60） | **不采集** |

**准确性得分 = (来源分 + 验证分 + 时效分 + 渠道分) / 8 × 100**

**⚠️ 重要规则**：
1. 每条情报必须标注**发布时间**
2. **优先采集近14天内发布的情报**（IT行业更新快）
3. 超过60天的情报一般不采集，除非有特殊历史价值
4. 案例类情报可适当放宽时效要求（案例可长期参考）
5. **优先采集A+、A级渠道**，B+级渠道需验证，B级以下谨慎采集

---

### 维度2：价值度（Value）- 权重25%

**存在理由**：与业务无关的情报浪费资源
**论据**：商务谈判+客户拜访场景占比60%，需要高价值情报

| 指标 | 评分标准 | 分值 |
|------|---------|:----:|------|
| **业务关联度** | 直接相关（AI配案、存量经营、营销） | 5 |
| | 高度相关（湖北项目、客户痛点） | 4 |
| | 中度相关（行业趋势、技术动态） | 3 |
| | 低度相关（周边行业、参考案例） | 2 |
| | 无关 | 1 |
| **战略参考价值** | 可直接用于方案/投标 | 5 |
| | 可作为案例参考 | 4 |
| | 可作为趋势判断依据 | 3 |
| | 仅作信息了解 | 2 |
| | 无参考价值 | 1 |
| **商业机会** | 有明确商机线索 | 5 |
| | 潜在合作机会 | 4 |
| | 竞品动态 | 3 |
| | 行业参考 | 2 |
| | 无商业价值 | 1 |

**价值度得分 = (关联度×0.4 + 战略价值×0.3 + 商业机会×0.3) / 5 × 100**

---

### 维度3：可用性（Usability）- 权重30%⭐最高权重

**存在理由**：不能落地的情报只是信息噪音
**论据**：方案撰写场景占比30%，对可用性要求最高；商务谈判也需要可用案例

| 指标 | 评分标准 | 分值 |
|------|---------|:----:|------|
| **可复用性** | 可直接复用于湖北项目 | 5 |
| | 需少量改造后可复用 | 4 |
| | 可参考架构/方案 | 3 |
| | 仅可参考思路 | 2 |
| | 无法复用 | 1 |
| **可落地性** | 技术成熟、资源匹配 | 5 |
| | 技术成熟、需协调资源 | 4 |
| | 技术待验证 | 3 |
| | 概念阶段 | 2 |
| | 无法落地 | 1 |
| **数据完整度** | 有完整数据（效果、成本、周期） | 5 |
| | 有部分数据 | 4 |
| | 有定性描述 | 3 |
| | 仅有概念 | 2 |
| | 无实质内容 | 1 |

**可用性得分 = (可复用性×0.4 + 可落地性×0.3 + 数据完整度×0.3) / 5 × 100**

---

### 维度4：时效衰减（Decay）- 权重10%

**存在理由**：过时情报可能误导决策
**论据**：客户拜访场景占比20%，需要最新信息

**按情报类型调整衰减曲线**：

| 情报类型 | 7天内 | 7-30天 | 30-90天 | 90-180天 | 180天+ |
|---------|:-----:|:------:|:-------:|:--------:|:------:|
| **政策类** | 1.0 | 0.8 | 0.6 | 0.4 | 0.2 |
| **技术类** | 1.0 | 0.9 | 0.8 | 0.6 | 0.4 |
| **案例类** | 1.0 | 0.95 | 0.9 | 0.8 | **0.6** |
| **竞品类** | 1.0 | 0.7 | 0.4 | 0.2 | 0.1 |

**论据**：
- 政策类：政策变化快，时效衰减快
- 技术类：技术迭代周期长，时效衰减慢
- 案例类：案例可长期参考，时效衰减最慢
- 竞品类：竞品信息价值随时间快速下降

**时效衰减得分 = 衰减系数 × 100**

---

### 维度5：稀缺性（Scarcity）- 权重10%

**存在理由**：稀缺情报创造竞争优势
**论据**：竞品分析场景占比10%，对稀缺性要求最高

| 稀缺程度 | 评分标准 | 分值 |
|---------|---------|:----:|
| **独家/首发** | 首次公开、独家报道 | 5 |
| **稀缺** | 少见公开、行业内部信息 | 4 |
| **一般** | 常规公开信息 | 3 |
| **泛滥** | 大量重复信息 | 2 |
| **过期** | 已被广泛传播 | 1 |

**稀缺性得分 = 稀缺程度 / 5 × 100**

---

## 🧮 综合评分公式（最终版）

```
综合得分 = 准确性×25% + 价值度×25% + 可用性×30% + 时效衰减×10% + 稀缺性×10%
```

**权重确定论据**：

| 维度 | 权重 | 推导论据 |
|------|:----:|---------|
| 准确性 | 25% | 商务谈判40%×60% = 24% ≈ 25% |
| 价值度 | 25% | 商务谈判40%×40% + 客户拜访20%×50% = 26% ≈ 25% |
| **可用性** | **30%** | 方案撰写30%×80% + 商务谈判40%×20% = 32% ≈ **30%（最高）** |
| 时效衰减 | 10% | 客户拜访20%×50% = 10% |
| 稀缺性 | 10% | 竞品分析10%×80% + 商务谈判40%×5% = 10% |

## 📈 筛选标准

| 评级 | 得分范围 | 处理方式 |
|------|:--------:|---------|
| **S级（精华）** | ≥80 | 入库精华库 + QQ推送提醒 |
| **A级（优质）** | 70-79 | 入库精华库 |
| **B级（参考）** | 60-69 | 入库原始库 |
| **C级（备查）** | 50-59 | 入库原始库（标记低优先） |
| **D级（舍弃）** | <50 | 不入库 |

---

## 🔄 迭代机制

### 1. 入库流程

```
原始情报 → 五维评估 → 综合评分 → 分级入库
                ↓
          S/A级 → 精华知识库
          B/C级 → 原始情报库
          D级   → 舍弃
```

### 2. 定期更新

| 周期 | 操作 |
|------|------|
| **每周** | 新情报入库 + 评估 |
| **每月** | 精华库重评估 + 低价值条目降级 |
| **每季度** | 知识库清理 + 结构优化 |

### 3. 价值重评估触发条件

- 新情报入库时，关联旧情报重评估
- 用户主动查询时，记录使用频次
- 商务谈判中使用时，标记"实战验证"

## 💡 模型自进化

### 学习机制

1. **用户反馈学习**
   - 用户标记"有用/无用"，调整权重
   - 用户查询频次，调整稀缺性评分

2. **实战验证学习**
   - 方案中使用的情报，标记"实战验证"
   - 投标成功的案例，提升价值权重

3. **关联推荐学习**
   - 用户查询A时推荐B，记录点击率
   - 高点击率关联，建立知识图谱

### 权重动态调整

初始权重：
- 准确性：25%
- 价值度：25%
- 可用性：30%
- 时效衰减：10%
- 稀缺性：10%

根据用户反馈动态调整（每季度一次）

## 📌 核心创新点

1. **基于场景推导权重**：避免主观臆断
2. **可用性权重最高**：方案撰写是核心场景
3. **时效衰减按情报类型区分**：政策快、技术慢、案例最慢、竞品最快
4. **数据完整度作为可用性子维度**：支撑ROI测算
