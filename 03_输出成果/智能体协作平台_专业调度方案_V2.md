|------|------|----------|

## 三、RACI角色职责矩阵

### 3.1 RACI定义

| 角色 | 说明 | 在智能调度中的含义 |
|:----:|------|-------------------|
| **R (Responsible)** | 执行者，实际完成任务 | 主要发言、主导讨论的Agent |
| **A (Accountable)** | 负责者，对结果负责 | 最终审核、确认产出的Agent |
| **C (Consulted)** | 咨询者，提供专业意见 | 参与讨论、提供专业视角的Agent |
| **I (Informed)** | 知情者，需要了解结果 | 被动接收信息的Agent |

- **I** = 仅知情（被动接收）

### 3.3 调度规则

**调度优先级**：
1. **R角色**：必须调度，主导讨论
2. **A角色**：必须调度，最终审核
3. **C角色**：根据上下文复杂度决定
4. **I角色**：不参与讨论，仅接收结果

**复杂度判断**：
```python
def determine_participation(task_code, complexity):
    """
    根据任务复杂度决定参与角色
    
    complexity: low / medium / high
    """
    raci = RACI_MATRIX[task_code]
    
    # 必选：R和A
    participants = [agent for agent, role in raci.items() if role in ['R', 'A']]
    
    # 根据复杂度添加C角色
    if complexity == 'high':
        participants += [agent for agent, role in raci.items() if role == 'C']
    elif complexity == 'medium':
        participants += [agent for agent, role in raci.items() if role == 'C'][:2]
    
    return list(set(participants))
```

### 5.3 质量评分机制

```python
def quality_score(document, task_code):
    """
    对产出物进行质量评分
    
    Returns:
        score: 0-100分
        issues: 问题列表
        suggestions: 改进建议
    """
    checklist = QUALITY_CHECKLIST[task_code]
    score = 0
    issues = []
    
    for item in checklist:
        if check_item(document, item):
            score += item.weight
        else:
            issues.append(item.description)
    
    # 生成改进建议
    suggestions = generate_suggestions(issues)
    
    return {
        "score": score,
        "level": "A" if score >= 90 else "B" if score >= 80 else "C",
        "issues": issues,
        "suggestions": suggestions
    }
```

## 六、调度执行引擎

### 6.1 调度决策流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 意图识别                                            │
│  用户输入 → 关键词匹配 → 语义分析 → 置信度计算               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 置信度判断                                          │
│  置信度 ≥ 0.95 → 直接执行                                    │
│  置信度 < 0.95 → 人工确认                                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 复杂度评估                                          │
│  输入长度 + 关键词复杂度 + 上下文 = 复杂度等级               │
│  low / medium / high                                         │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: RACI调度                                            │
│  任务代码 + 复杂度 → RACI矩阵 → 参与Agent列表                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 执行监控                                            │
│  监控讨论进度 → 检查产出质量 → 触发下一阶段                  │
└─────────────────────────────────────────────────────────────┘
```
