## 一、工程概述

### 1.1 工程主题
**指南针工程**——多Agent智能协作平台

### 1.2 工程Slogan
> **「南有乔木，不可休思；九星连珠，共指迷津」**

寓意：九个Agent如北斗九星，协同指引方向，为决策导航。

### 1.3 工程理念

| 维度 | 内涵 |
|------|------|
| **协作** | 九位一体，各司其职，协同决策 |
| **智能** | 百度千帆赋能，真实LLM对话 |
| **专业** | 角色认知、立场分明、辩论机制 |
| **共识** | 质疑挑战、收敛决策、达成共识 |

## 四、问题与解决

### 4.1 遇到的问题

| 问题 | 描述 |
|------|------|
| UI专业度不足 | 评分仅69/100，缺少角色专属色、共识度面板 |
| 发送按钮无反应 | JS事件绑定问题 |
| Agent不完整 | 仅7个Agent，缺少呈彩、折桂 |
| 快捷场景不相关 | CRM/数据平台/AI客服不贴合电信业务 |
| 对话框闪动 | 动画效果影响体验 |

### 4.2 解决方案

| 问题 | 解决方案 | 结果 |
|------|----------|------|
| UI专业度 | V10全面重构，极简专业风 | ✅ 评分提升 |
| 发送按钮 | 修复handleKeyPress事件 | ✅ 正常响应 |
| Agent不完整 | V11补齐至9个 | ✅ 完整团队 |
| 快捷场景 | 改为电信行业场景 | ✅ 业务贴合 |
| 对话框闪动 | 移除pulse/slideIn动画 | ✅ 稳定显示 |

## 五、收获与总结

### 5.1 技术收获

| 领域 | 收获 |
|------|------|
| **多Agent架构** | 掌握协作调度、通信总线、角色规则设计模式 |
| **LLM集成** | 熟悉百度千帆API调用、System Prompt设计 |
| **角色认知** | 理解立场、风险偏好、专业边界对决策的影响 |
| **动态调度** | 实现基于上下文的智能发言者选择 |
| **辩论机制** | 设计质疑触发、立场冲突、共识收敛逻辑 |
| **UI设计** | 掌握极简专业风格、角色专属色、数据可视化 |

### 5.2 业务价值

| 价值点 | 说明 |
|--------|------|
| **决策支持** | 多视角专业分析，降低决策风险 |
| **知识沉淀** | 对话导出，形成决策知识库 |
| **效率提升** | 自动化协作讨论，节省会议时间 |
| **共识保障** | 辩论机制确保方案全面性 |

### 5.3 核心经验

> **「一个都不能少」**
> 
> 9个Agent各司其职，缺一不可。成本、质量、进度、资源全方位覆盖。

> **「立场决定视角」**
> 
> 筑台（成本优先）必然质疑织锦（质量优先）的高成本方案，玉衡（进度优先）必然预警工期风险。

> **「辩论催生共识」**
> 
> 质疑不是对抗，而是完善。通过立场冲突，方案更加全面稳健。
