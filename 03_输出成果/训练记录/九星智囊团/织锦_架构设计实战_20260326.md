## 一、架构设计目标

### 1.1 设计原则

- **高可用**：单点故障不影响整体服务
- **高并发**：支持万级并发访问
- **可扩展**：支持水平扩展
- **可维护**：清晰的模块边界

## 二、系统架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      网关层 (API Gateway)                   │
│                    Spring Cloud Gateway                     │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
        ▼                  ▼                  ▼
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│  用户服务     │  │  套餐服务     │  │  订单服务     │
│  (user-svc)  │  │ (package-svc) │  │ (order-svc)   │
└───────┬───────┘  └───────┬───────┘  └───────┬───────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
                    ┌──────▼──────┐
                    │  数据访问层  │
                    │  (MySQL)    │
                    └─────────────┘
```

## 四、接口设计

### 4.1 用户服务API

| 接口 | 方法 | 路径 | 说明 |
|------|:----:|------|------|
| 创建用户 | POST | /api/user | 创建新用户 |
| 查询用户 | GET | /api/user/{id} | 根据ID查询 |
| 更新用户 | PUT | /api/user/{id} | 更新用户信息 |
| 删除用户 | DELETE | /api/user/{id} | 删除用户 |

### 4.2 套餐服务API

| 接口 | 方法 | 路径 | 说明 |
|------|:----:|------|------|
| 套餐列表 | GET | /api/package | 获取套餐列表 |
| 套餐详情 | GET | /api/package/{id} | 获取套餐详情 |

### 4.3 订单服务API

| 接口 | 方法 | 路径 | 说明 |
|------|:----:|------|------|
| 创建订单 | POST | /api/order | 创建新订单 |
| 订单列表 | GET | /api/order | 获取订单列表 |
| 订单详情 | GET | /api/order/{id} | 获取订单详情 |

## 五、部署架构

### 5.1 容器化部署

```yaml
# docker-compose.yml
version: '3.8'
services:
  nacos:
    image: nacos/nacos-server:v2.2.0
    ports:
      - "8848:8848"
  
  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
  
  redis:
    image: redis:7.0
    ports:
      - "6379:6379"
```
