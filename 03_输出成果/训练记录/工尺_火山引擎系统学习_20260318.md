## 一、系统架构设计

### 1.1 产品整体架构

火山引擎 DataAgent 采用**分层架构设计**，从底层到顶层依次为：

```
┌─────────────────────────────────────────────────────────────┐
│                        行业场景层                            │
│         汽车 | 金融 | 大消费 | 互联网 | 其他                  │
├─────────────────────────────────────────────────────────────┤
│                        交互层                                │
│              GUI | REST API | SDK | LUI                     │
├─────────────────────────────────────────────────────────────┤
│                      智能体层                                │
│    智能分析Agent | 智能营销Agent | 数据工程Agent(规划) | ...  │
├─────────────────────────────────────────────────────────────┤
│                        认知层                                │
│   大模型 | 数据引擎 | 语义模型配置 | Multi-Agent协作          │
│   Prompt配置 | 长期记忆存储 | 业务知识配置                    │
├─────────────────────────────────────────────────────────────┤
│                        工具层                                │
│   DataWind | VeCDP | DataFinder | GMP | DataTester | ...    │
├─────────────────────────────────────────────────────────────┤
│                      数据应用基座                            │
│   豆包大模型 | EMR | LAS | ByteHouse | DataSail | Flink     │
├─────────────────────────────────────────────────────────────┤
│                        云基础                               │
│   豆包公有云Pro | 豆包私部Lite | DeepSeek | 其他开源模型      │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 智能分析Agent功能架构

**核心定位**：基于AI深度思考与大数据分析的专家顾问

**功能模块划分**：

| 层级 | 功能模块 | 说明 |
|------|----------|------|
| 多终端适配 | IM / H5嵌入 / 页面嵌出 | 飞书、钉钉、企业微信等多端支持 |
| 数据消费 | 多轮追问 / 深度研究 / 归因分析 / 智能推荐 / 个性化推送 | 核心分析能力 |
| 配置管理 | 语义模型配置 / 业务知识配置 / 推荐问题配置 / Prompt配置 | 定制化配置 |
| 能力基座 | 反问澄清 / 模糊问题拆解 / 指标小结配置 | 基础能力支撑 |
| 模型对接 | 豆包大模型 / DeepSeek R1/V3 / 更多大模型 | 多模型支持 |
| 数据连接 | 数据集 / 数据权限 / 建模处理 | 数据基础设施 |

**业务能力架构（REACT模式）**：

```
规划(P LAN)阶段 → 推理&行动(REACT)阶段
    ↓                      ↓
用户问题 ──→ 问题改写 ──→ 生成整体计划 ──→ 反问澄清
                                    ↓
                          ┌────────┼────────┐
                          ↓        ↓        ↓
                      Text2SQL  Text2Python  联网搜索
                          ↓        ↓        ↓
                      SQL执行  Python执行  知识库检索
                          ↓        ↓        ↓
                      SQL优化  Python优化  MCP开放调用
                          └────────┼────────┘
                                   ↓
                          问数取数 ──→ 归因诊断 ──→ 行动规划
                                   ↓
                          报告生成 ──→ 深度思考报告
```

### 1.3 智能营销Agent功能架构

**核心定位**：智能营销策略数字营销决策中枢

**功能模块划分**：

| Agent类型 | 核心能力 | 应用场景 |
|-----------|----------|----------|
| 智能营销策略Agent | 营销策略编排与效果预测、目标人群挖掘、智能通道择优、个性化营销时机、智能复盘分析 | 方案规划 |
| 智能会话助手 | 意向评分、实时画像、话术建议、商品推荐、客户建档、任务跟进 | 销售转化 |
| VeCDP Copilot | 智能标签提取、智能个体画像解读、智能圈群、智能人群画像分析 | 数据分析 |
| GMP AIGC创意助手 | 海报/落地页/卡片/营销文案/商品图/直播脚本生成 | 内容生产 |

**营销链路优化**：

```
方案规划 → 活动执行&监控 → 复盘分析
    ↓            ↓            ↓
营销目标拆解   活动人群圈选   会话数据分析
营销方案生成   任务智能编排   智能客群分析
               创意素材生成   VOC分析
                            效果复盘建议
```

### 1.4 系统集成方式

**多Agent协作架构**：

- **Multi-Agent协作**：多个Agent联动协作，覆盖复杂业务链条
- **MCP开放能力**：提供统一上下文、多Agent/模型协作、第三方tools接入
- **多模型智能路由**：发挥不同模型优势，满足不同场景个性需求

## 二、数据架构设计

### 2.1 数据采集与处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                      数据来源                                │
├──────────────────────┬──────────────────────────────────────┤
│   企业私域数据        │         公域数据                     │
│   (结构化+非结构化)   │     (结构化+非结构化)                 │
├──────────────────────┼──────────────────────────────────────┤
│ 订单/会员/标签属性    │  第三方可交易数据                     │
│ 行为数据/企微对话     │  公开文献/网络搜索                    │
│ 产品资料/评论/知识库  │  优质数据/内容清洗                    │
└──────────────────────┴──────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                   VeCDP数据引擎                              │
├─────────────────────────────────────────────────────────────┤
│ 数据清洗 → OneID构建 → 多主体档案 → 标签提取/加工           │
│ 向量化 → 行为事件 → 维度档案 → 特征工程 → 产品知识提取       │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                   GMP营销引擎                               │
├─────────────────────────────────────────────────────────────┤
│ 实时事件营销引擎 → 批量调度营销引擎 → 个性化营销引擎         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 结构化与非结构化数据融合

**数据类型分布**：
- 企业数据中**80%**以文件形式存在（非结构化和半结构化数据）
- **68%**的企业数据没有得到充分利用

**融合策略**：

| 数据类型 | 处理方式 | 应用场景 |
|----------|----------|----------|
| 结构化数据 | 直接接入数据集，语义模型配置 | 智能问数、指标分析 |
| 非结构化数据 | 知识库构建、向量化处理 | 深度研究、知识融合 |
| 半结构化数据 | 标签提取、特征工程 | 人群洞察、画像构建 |

**知识库建设**：
- 企业知识库：业务规则、历史数据、专家经验
- 行业知识库：行业标准、业务规则、垂直领域专业数据
- 交互指令数据：用户行为数据、对话记录

### 2.3 语义模型配置

**核心配置项**：

1. **业务知识配置**
   - 同义词配置
   - 映射关系配置
   - 常见问题FAQ

2. **指标资产配置**
   - 指标定义
   - 计算口径
   - 数据来源映射

3. **推荐问题配置**
   - 高频问题预设
   - 场景化问题模板

4. **Prompt配置**
   - 系统提示词
   - 用户提示词引导
   - hidden-thought推理引入

## 四、系统特性分析

### 4.1 24/7不间断运行

**实现机制**：

| 维度 | 传统员工 | DataAgent |
|------|----------|-----------|
| 工作耐力 | 日均8小时，情绪波动影响产出 | 全年无休，7×24小时在线 |
| 知识获取 | 500字/分钟，理解率70% | 每秒解析5份财报，跨文档关联 |
| 能力迭代 | 年培训2-3次，知识半衰期2年 | 数据驱动自动进化 |

**技术保障**：
- 云原生架构，高可用部署
- 容器化弹性伸缩
- 多区域容灾备份

### 4.2 幻觉压制与记忆机制

**幻觉压制策略**：

```
┌─────────────────────────────────────────────────────────────┐
│                     幻觉压制技术栈                           │
```

### 4.3 推理速度优化

**优化路径**：

1. **GPU性能提升**

2. **模型优化**
   - KV Cache优化

3. **工程优化**

   - 流式输出优化首token响应速度

## 五、系统设计启示

### 5.1 架构设计启示

1. **分层解耦**
   - 清晰的层次划分（云基础→数据基座→工具层→认知层→智能体层→交互层）
   - 每层可独立演进，降低系统耦合度

2. **多模型融合**
   - 大模型（通用能力）+ 小模型（特定任务优化）
   - 多模型智能路由，按需选择最优模型

3. **Agent能力分级**
   - L1：响应式执行
   - L2：理解式洞察
   - L3：建议式决策
   - L4：自主式决策

### 5.2 数据设计启示

1. **数据价值金字塔**
   - 原始互联网数据（数百TB）→ 清洗后高质量数据（约15%）
   - 企业专属数据价值最高，需重点建设

2. **语义模型是关键桥梁**
   - 连接业务语言与数据语言
   - 需要业务专家与技术专家共同构建

3. **知识库建设是核心竞争力**
   - 行业知识 + 企业知识 + 交互知识
   - 持续积累，形成壁垒

### 5.3 产品设计启示

1. **场景化落地**
   - 从具体业务场景切入（如电商GMV分析、金融投顾）
   - 验证价值后再扩展

2. **渐进式交付**
   - 业务调研 → 数据准备 → 智能体搭建 → 测试调优 → 推广渗透
   - 降低一次性交付风险

3. **人机协同**
   - AI不是替代人，而是增强人
   - 提供人工干预/纠偏机制
