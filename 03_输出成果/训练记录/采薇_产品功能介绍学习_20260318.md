## 一、业务概念定义

### 1.1 什么是商机？

**商机定义**：基于客户潜在需求挖掘形成的信息记录，具备明确业务需求，可预期执行时间，可量化价值和有效触达路径，可促成个人或企业的实际购买。

**有效商机标准**：
- 具备明确业务需求
- 可预期执行时间
- 可量化价值
- 有效触达路径
- 可促成实际购买

### 1.2 商机分类

从文档中梳理出以下分类维度：

| 分类维度 | 类别说明 |
|---------|---------|
| 按来源渠道 | 线下商机、线上商机、政企商机 |
| 按客户类型 | 公众客户商机、政企客户商机、异网客户商机 |
| 按数据完整度 | 原始数据、空间数据、商机数据 |
| 按处理状态 | 有效商机、需补充商机、无效商机 |

### 1.3 商机生命周期

```
商机采集 → 商机清洗 → 信息增强 → 商机分派 → 商机执行 → 商机转化 → 商机评估
```

**各阶段说明**：

| 阶段 | 核心工作 | 关键产出 |
|------|---------|---------|
| 商机采集 | 多渠道、多方式收集商机 | 原始商机数据 |
| 商机清洗 | AI清洗+规则清洗，去除无效数据 | 有效商机数据 |
| 信息增强 | 补充客户画像、企业信息 | 高级商机数据 |
| 商机分派 | 按规则自动/人工派单 | 分派工单 |
| 商机执行 | 跟进、营销、转化 | 执行记录 |
| 商机转化 | 成交、回访、闭环 | 订单、转化记录 |
| 商机评估 | 数据统计、效能分析 | 分析报表 |

### 1.4 空间数据定义

**空间数据**：对潜在客户物理存在的客观描述性信息，尚未明确需求成为实际购买者的个人或企业。

**空间数据要素**：
- **必要要素**：客户信息、物理空间
- **可选要素**：在用业务、企业信息

## 三、用户角色分析

### 3.1 用户角色清单

| 角色 | 角色定位 | 使用场景 | 核心诉求 |
|------|---------|---------|---------|
| 装维经理 | 一线触点人员 | 入户服务时收集商机 | 快速录入、CP协同、激励可见 |
| 厅店人员 | 一线营销人员 | 接收商机、跟进转化 | 自动派单、信息完整、便于跟进 |
| 支局长 | 基层管理者 | 组织营销活动、收集商机 | 统一调度、执行管控、评估激励 |
| 分公司/县公司管理员 | 中层管理者 | 区域商机运营管理 | 统计分析、通报督导、效能评估 |
| 省市商机管理人员 | 高层管理者 | 全省商机洞察决策 | 总览视图、趋势分析、决策支撑 |
| 商机运营官 | 平台配置人员 | 模板配置、规则管理 | 灵活配置、快速响应 |
| 全员（泛渠道） | 任意员工 | 随时发现商机随手录 | 低门槛、便捷操作 |

### 3.2 典型使用场景分析

#### 场景一：装维入户采集

**角色**：装维经理  
**痛点**：无合适工具，口头信息传递，利益分配难评估  
**解决方案**：手机快速录音记录商机，系统自动分享CP结对厅店，可实时跟进转化情况

#### 场景二：支局兵团作战

**角色**：支局长  
**痛点**：纸质登记不标准，人工跟进易遗忘，管理评估难  
**解决方案**：配置场景化采集模板，统一打印下发，拍照批量上传系统，统一调度执行

#### 场景三：全民随手录

**角色**：任意员工  
**痛点**：发现商机无处录入，录入门槛高  
**解决方案**：微信小程序轻量入口，支持语音/图片/视频多模态录入，AI自动解析

```

   ↓ 需求明确

   │
   ├─ 必要要素：客户信息、意向业务、时间信息、归属关系
   └─ 可选要素：物理空间、在用业务、渠道信息、企业信息
```

## 五、需求分析要点

### 5.1 业务需求分析

#### 核心业务目标

1. **统一商机认知**：制定全公司统一的商机标准和管理规范
2. **便捷采集工具**：助力形成采、用商机的常态化运营思维
3. **智能运营中台**：全渠道全形态商机统一精细运营管理

#### 关键业务痛点

| 痛点类别 | 具体问题 | 解决方案 |
|---------|---------|---------|
| 采集问题 | 商机缺乏统一标准、一线不用工具收集 | 统一要素定义、便捷采集工具 |
| 隐私问题 | 一线担心商机被他人看到 | 分级隐私管理、脱敏查看 |
| 管理问题 | 商机分散各处、场景化采集难 | 统一汇聚、灵活模板配置 |
| 协同问题 | 信息传递不及时、利益分配不透明 | 系统自动分享、执行留痕 |

### 5.2 功能需求分析

#### 功能优先级矩阵

| 优先级 | 功能项 | 业务价值 | 实现难度 |
|-------|-------|---------|---------|
| P0 | 商机页面录入 | 核心采集能力 | 中 |
| P0 | 商机隐私管理 | 打消录入顾虑 | 低 |
| P0 | 商机汇聚清洗 | 数据质量保障 | 高 |
| P1 | AI随手录 | 降低录入门槛 | 高 |
| P1 | 拍照批量录 | 适应一线习惯 | 中 |
| P1 | 商机分享协同 | 提升转化效率 | 低 |
| P2 | 模板灵活配置 | 支撑场景化采集 | 中 |
| P2 | 数据增强 | 补全客户画像 | 高 |
| P2 | 对外赋能接口 | 系统对接 | 中 |

### 5.3 非功能需求分析

| 维度 | 需求说明 |
|------|---------|
| 性能 | 支持大规模并发录入，批量导入效率高 |
| 安全 | 商机数据分级权限管控，敏感信息脱敏 |
| 可用性 | 微信小程序轻量入口，无需安装 |
| 兼容性 | 对接现有系统（承包助手、翼销售、客服V600等） |
| 可扩展 | 支持新增采集场景、新增数据源对接 |

### 5.4 数据需求分析

#### 数据质量要求

- **完整性**：关键要素必填，可选要素尽量补全
- **准确性**：地址标准化、格式校验、AI解析校对
- **唯一性**：同号码、同地址、同客户去重标记
- **时效性**：商机有效期管理、过期提醒

#### 数据安全要求

- 录入人看商机全貌，上级脱敏查看
- 分享授权机制，被分享人可见完整信息
- 敏感字段（姓名、联系方式、地址）分级管控

## 六、个人收获与改进方向

### 6.1 学习收获

#### 业务理解收获

1. **商机本质**：商机不是简单的客户信息，而是具备明确需求、可预期时间、可量化价值的完整信息记录
2. **数据演进**：从原始数据→空间数据→商机数据，是一个逐步明确、逐步增值的过程
3. **运营思维**：商机运营强调"采-用"闭环，不仅是采集，更重要的是转化执行

#### 产品设计收获

1. **用户体验优先**：六大特点设计（分级隐私、多形式采集、灵活元素、AI辅助、自动提醒、轻量随用）充分体现以用户为中心的设计理念
2. **场景驱动**：从具体场景出发（装维入户、兵团作战、全民随手录），确保功能实用、好用
3. **技术赋能**：AI大模型能力（OCR、ASR、NLP）的应用显著降低使用门槛

#### 需求分析收获

1. **痛点即需求**：从一线"不愿录、不会录、不敢录"的痛点出发，逐一定向解决
2. **角色视角**：不同角色（一线、管理、运营）诉求差异大，需分层设计功能权限
3. **数据标准先行**：统一的商机要素定义是一切功能设计的基础

### 6.2 需求分析改进方向

#### 分析方法改进

1. **深入一线调研**：文档中的痛点来源于调研，后续需强化一线访谈和实地观察
2. **量化分析**：建立商机转化率、采集效率等量化指标，支撑需求优先级决策
3. **场景化梳理**：形成标准化的场景梳理模板，确保不遗漏关键场景

#### 文档输出改进

1. **需求规格文档**：将学习成果转化为标准的需求规格文档，明确功能点、边界条件、异常处理
2. **原型设计**：配合UI/UX设计低保真原型，验证需求理解是否准确
3. **验收标准**：为每个功能点定义可测试的验收标准

### 6.3 后续学习计划

1. **深化业务理解**：学习电信行业基础知识，理解公众业务、政企业务、异网竞争格局
2. **技术方案学习**：了解AI大模型在商机处理中的具体技术实现
3. **竞品对标**：研究其他运营商或行业的商机管理平台设计方案
4. **持续迭代**：跟踪平台上线后的用户反馈，持续优化需求分析能力
