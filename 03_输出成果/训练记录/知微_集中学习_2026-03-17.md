## 一、电信核心指标体系

### 1. 收入类指标

#### ARPU（每用户平均收入）
```
公式: ARPU = 总收入 / 用户数
单位: 元/户/月
```

**计算示例**:
- 月度收入: 1000万元
- 在网用户: 50万户
- ARPU = 1000万 / 50万 = 20元/户/月

**分级标准**:
| 等级 | ARPU范围 | 用户占比 | 价值特征 |
|------|----------|----------|----------|
| 高价值 | >80元 | 10% | 高消费、高黏性、高转化潜力 |
| 中高价值 | 50-80元 | 20% | 消费稳定、套餐升级潜力大 |
| 中等价值 | 30-50元 | 30% | 主流用户、需精细运营 |
| 低价值 | <30元 | 40% | 保号用户、需价值提升 |

**应用场景**:
1. 用户价值分层
2. 套餐定价策略
3. 营销资源分配
4. 收入预测分析

---

#### MOU（每用户平均通话时长）
```
公式: MOU = 总通话时长(分钟) / 用户数
单位: 分钟/户/月
```

**行业基准**:
- 4G时代平均: 300-400分钟/月
- 5G时代下降趋势: 200-300分钟/月
- 语音业务萎缩，数据业务为主

**分析维度**:
- 主叫MOU vs 被叫MOU
- 网内MOU vs 网间MOU
- 工作日MOU vs 周末MOU
- 高峰时段MOU vs 低峰时段MOU

---

#### DOU（每用户平均流量使用）
```
公式: DOU = 总流量(MB/GB) / 用户数
单位: GB/户/月
```

**行业趋势**:
| 年份 | 平均DOU | 增长率 |
|------|---------|--------|
| 2020 | 8GB | - |
| 2021 | 12GB | 50% |
| 2022 | 16GB | 33% |
| 2023 | 20GB | 25% |
| 2024 | 25GB | 25% |

**应用场景**:
1. 流量包推荐
2. 套餐适配分析
3. 网络容量规划
4. 用户行为分析

### 2. 用户管理类指标

#### 用户留存率与流失率
```
留存率 = 当期末仍在网用户 / 期初用户 × 100%
流失率 = 当期流失用户 / 期初用户 × 100%
```

**计算示例**:
```
月初用户: 100,000户
月末仍在网: 95,000户
月末流失: 5,000户

留存率 = 95,000 / 100,000 = 95%
流失率 = 5,000 / 100,000 = 5%
```

**分类统计**:
| 流失类型 | 定义 | 预警信号 |
|----------|------|----------|
| 主动流失 | 用户主动销户 | 连续投诉、不满情绪 |
| 被动流失 | 欠费停机超期 | 余额不足、未充值 |
| 沉默流失 | 长期不使用 | 使用量骤降 |

**流失预警规则**:
```
高风险用户判定:
1. 话费余额 < 10元
2. 近30天通话时长下降 > 50%
3. 近30天流量使用下降 > 50%
4. 投诉次数 > 3次/月
5. 距上次充值 > 60天

满足任意2条 → 高风险用户
```

---

#### 用户增长率
```
公式: 用户增长率 = (新增用户 - 流失用户) / 期初用户 × 100%
```

**健康指标**:
- 增长率 > 0: 业务扩张期
- 增长率 = 0: 稳定期
- 增长率 < 0: 收缩期（需关注）

---

### 3. 业务运营类指标

#### 套餐渗透率与转化率
```
渗透率 = 某套餐订购用户数 / 总用户数 × 100%
转化率 = 营销活动转化用户数 / 触达用户数 × 100%
```

**应用场景**:
- 新套餐推广效果评估
- 营销活动ROI计算
- 产品优化方向确定

---

#### 接通率与投诉率
```
接通率 = 接通次数 / 呼叫次数 × 100%
投诉率 = 投诉用户数 / 总用户数 × 100%
```

**质量标准**:
| 指标 | 目标值 | 预警值 |
|------|--------|--------|
| 网络接通率 | >98% | <95% |
| 服务投诉率 | <0.1% | >0.3% |
| 故障处理及时率 | >95% | <90% |

### 方法二：RFM模型

**模型定义**:
```
R (Recency) 最近一次消费时间
F (Frequency) 消费频率
M (Monetary) 消费金额
```

**电信场景适配**:
| RFM维度 | 电信指标 | 计算方式 |
|---------|----------|----------|
| R | 最近充值天数 | 距上次充值的日期差 |
| F | 充值频次 | 月均充值次数 |
| M | 充值金额 | 月均充值金额 |

**用户分层矩阵**:
```
              F高（高频充值）
                   │
    ┌──────────────┼──────────────┐
    │              │              │
    │  重要价值客户 │  重要发展客户 │
    │  (R高F高M高) │  (R高F低M高) │
    │              │              │
R高 │──────────────┼──────────────│ R低
    │              │              │
    │  重要保持客户 │  重要挽留客户 │
    │  (R低F高M高) │  (R低F低M高) │
    │              │              │
    └──────────────┼──────────────┘
                   │
              F低（低频充值）
```

**8类用户细分**:
| 用户类型 | R | F | M | 特征 | 运营策略 |
|----------|---|---|---|------|----------|
| 重要价值客户 | 高 | 高 | 高 | 核心用户 | VIP服务、高端套餐 |
| 重要发展客户 | 高 | 低 | 高 | 高价值新客 | 提升使用频次 |
| 重要保持客户 | 低 | 高 | 高 | 有流失风险 | 关怀挽留、优惠活动 |
| 重要挽留客户 | 低 | 低 | 高 | 高价值流失风险 | 重点挽留、专属优惠 |
| 一般价值客户 | 高 | 高 | 低 | 中等价值 | 套餐升级引导 |
| 一般发展客户 | 高 | 低 | 低 | 普通新客 | 激活使用 |
| 一般保持客户 | 低 | 高 | 低 | 活跃低价值 | 保持活跃、价值提升 |
| 低价值客户 | 低 | 低 | 低 | 沉睡用户 | 唤醒或放弃 |

**Python实现代码**:
```python
import pandas as pd
import numpy as np

# 计算RFM值
def calculate_rfm(df, reference_date):
    """
    计算用户RFM值
    
    参数:
    - df: 充值记录数据框
    - reference_date: 参考日期（通常是分析截止日期）
    
    返回:
    - RFM数据框
    """
    rfm = df.groupby('user_id').agg({
        'recharge_date': lambda x: (reference_date - x.max()).days,  # R
        'recharge_id': 'count',  # F
        'recharge_amount': 'sum'  # M
    })
    
    rfm.columns = ['R', 'F', 'M']
    
    # 分箱打分（5分制）
    rfm['R_score'] = pd.qcut(rfm['R'], 5, labels=[5,4,3,2,1], duplicates='drop')
    rfm['F_score'] = pd.qcut(rfm['F'], 5, labels=[1,2,3,4,5], duplicates='drop')
    rfm['M_score'] = pd.qcut(rfm['M'], 5, labels=[1,2,3,4,5], duplicates='drop')
    
    # 计算RFM总分
    rfm['RFM_score'] = rfm['R_score'].astype(str) + rfm['F_score'].astype(str) + rfm['M_score'].astype(str)
    
    return rfm
```

**关键特征设计**:
```python
# 流失预测特征工程示例

def create_churn_features(df):
    """创建流失预测特征"""
    
    features = pd.DataFrame()
    features['user_id'] = df['user_id'].unique()
    
    # === 使用量特征 ===
    features['avg_mou_3m'] = df.groupby('user_id')['mou'].transform(lambda x: x.rolling(3).mean())
    features['mou_decline_rate'] = df.groupby('user_id')['mou'].transform(lambda x: x.pct_change())
    
    features['avg_dou_3m'] = df.groupby('user_id')['dou'].transform(lambda x: x.rolling(3).mean())
    features['dou_decline_rate'] = df.groupby('user_id')['dou'].transform(lambda x: x.pct_change())
    
    # === 消费特征 ===
    features['arpu'] = df.groupby('user_id')['arpu'].transform('mean')
    features['balance'] = df['balance']
    features['days_since_recharge'] = df['days_since_last_recharge']
    
    # === 互动特征 ===
    features['complaint_count'] = df.groupby('user_id')['complaint_flag'].transform('sum')
    features['customer_service_calls'] = df.groupby('user_id')['cs_call_count'].transform('sum')
    features['days_since_login'] = df['days_since_last_login']
    
    # === 套餐特征 ===
    features['plan_tenure'] = df['current_plan_months']
    features['days_to_plan_expiry'] = df['plan_expiry_days']
    
    # === 组合特征 ===
    features['value_usage_ratio'] = features['arpu'] / (features['avg_mou_3m'] + 1)
    features['risk_score'] = (
        (features['balance'] < 10).astype(int) * 2 +
        (features['mou_decline_rate'] < -0.5).astype(int) * 2 +
        (features['complaint_count'] > 3).astype(int) * 3
    )
    
    return features
```

**模型评估指标**:
```
混淆矩阵:
                    预测
                 流失    非流失
实际  流失      TP       FN
      非流失    FP       TN

关键指标:
- 准确率(Accuracy) = (TP + TN) / (TP + TN + FP + FN)
- 精确率(Precision) = TP / (TP + FP)
- 召回率(Recall) = TP / (TP + FN)
- F1-score = 2 * Precision * Recall / (Precision + Recall)
- AUC: ROC曲线下面积，衡量模型区分能力
```

**流失预警规则引擎**:
```python
def churn_risk_alert(user_features):
    """流失风险预警"""
    risk_score = 0
    alerts = []
    
    # 规则1: 余额不足
    if user_features['balance'] < 10:
        risk_score += 2
        alerts.append("余额不足10元")
    
    # 规则2: 使用量下降
    if user_features['mou_decline_rate'] < -0.5:
        risk_score += 2
        alerts.append("通话时长下降超50%")
    
    if user_features['dou_decline_rate'] < -0.5:
        risk_score += 2
        alerts.append("流量使用下降超50%")
    
    # 规则3: 投诉频繁
    if user_features['complaint_count'] > 3:
        risk_score += 3
        alerts.append("月投诉超3次")
    
    # 规则4: 长期未充值
    if user_features['days_since_recharge'] > 60:
        risk_score += 2
        alerts.append("超60天未充值")
    
    # 规则5: 套餐即将到期
    if user_features['days_to_plan_expiry'] < 7:
        risk_score += 1
        alerts.append("套餐即将到期")
    
    # 风险等级判定
    if risk_score >= 5:
        risk_level = "高风险"
    elif risk_score >= 3:
        risk_level = "中风险"
    else:
        risk_level = "低风险"
    
    return {
        'risk_score': risk_score,
        'risk_level': risk_level,
        'alerts': alerts
    }
```

## 三、数据可视化规范

### 图表设计原则

**核心原则（来自Edward Tufte）**:
1. **数据墨水比最大化**: 每一笔墨水都应该传递数据信息
2. **避免图表垃圾**: 去除不必要的装饰元素
3. **真实性原则**: 面积/长度应与数据成比例
4. **清晰性原则**: 标签、图例、标题要清晰

**图表选择指南**:
```
数据关系类型 → 推荐图表

比较类:
├── 少量类别 → 柱状图
├── 多类别对比 → 分组柱状图
├── 时间趋势 → 折线图
└── 排名展示 → 条形图

分布类:
├── 单变量分布 → 直方图
├── 多变量分布 → 箱线图
├── 地理分布 → 地图
└── 密度分布 → 密度图

构成类:
├── 占比构成 → 饼图（少用）
├── 堆叠构成 → 堆叠柱状图
├── 时间构成变化 → 堆叠面积图
└── 层级构成 → 树图/旭日图

关联类:
├── 两变量相关 → 散点图
├── 多变量相关 → 气泡图
├── 矩阵关系 → 热力图
└── 流向关系 → 桑基图
```

**颜色使用规范**:
```
配色原则:
1. 分类数据: 使用对比色
2. 顺序数据: 使用渐变色
```

```python
churn_users = df[df['churn_flag'] == 1]
```

**Step 2: 流失风险因素分析**
```python
from scipy import stats

# 计算各特征与流失的相关性
print("=== 流失相关性分析 ===\n")
correlations = {}
for feature in numeric_features:
    corr, p_value = stats.pointbiserialr(df['churn_flag'], df[feature])
    correlations[feature] = {'correlation': corr, 'p_value': p_value}
    
# 按相关性排序
sorted_corr = sorted(correlations.items(), key=lambda x: abs(x[1]['correlation']), reverse=True)
for feature, values in sorted_corr:
    sig = '***' if values['p_value'] < 0.001 else '**' if values['p_value'] < 0.01 else '*' if values['p_value'] < 0.05 else ''
    print(f"{feature}: r={values['correlation']:.3f}{sig}")
```

```python
def calculate_churn_risk(row):

        risk_score += 2

        risk_score += 2
    if row['dou'] < 5:
        risk_score += 1
    
    # 在网时长风险
    if row['tenure_months'] < 6:
        risk_score += 1
    
    # 投诉风险
    if row['complaint_count'] > 2:
        risk_score += 2
    
    return risk_score

df['churn_risk_score'] = df.apply(calculate_churn_risk, axis=1)

# 风险等级分类
def classify_risk(score):

        return '高风险'
    elif score >= 3:
        return '中风险'

df['risk_level'] = df['churn_risk_score'].apply(classify_risk)
```

```python
def handle_missing_values(df):
    """处理缺失值"""
    
    # 数值型: 用中位数填充
    numeric_cols = df.select_dtypes(include=[np.number]).columns
    for col in numeric_cols:
        if df[col].isnull().sum() > 0:
            df[col].fillna(df[col].median(), inplace=True)
    
    # 分类型: 用众数填充
    categorical_cols = df.select_dtypes(include=['object']).columns
    for col in categorical_cols:
        if df[col].isnull().sum() > 0:
            df[col].fillna(df[col].mode()[0], inplace=True)
    
    return df
```

```python
def detect_outliers_iqr(df, column):
    """使用IQR方法检测异常值"""
    Q1 = df[column].quantile(0.25)
    Q3 = df[column].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR
    
    outliers = df[(df[column] < lower_bound) | (df[column] > upper_bound)]
    return outliers, lower_bound, upper_bound

def handle_outliers(df, column, method='clip'):
    """处理异常值"""
    outliers, lower, upper = detect_outliers_iqr(df, column)
    
    if method == 'clip':
        # 截断法
        df[column] = df[column].clip(lower, upper)
    elif method == 'remove':
        # 删除法
        df = df[(df[column] >= lower) & (df[column] <= upper)]
    
    return df

# 处理ARPU异常值
```

```python
def create_features(users, usage, billing, interaction, churn):
    """构建流失预测特征"""
    
    # 使用量特征
    usage_agg = usage.groupby('user_id').agg({
        'mou': ['mean', 'std', 'min', 'max'],
        'dou': ['mean', 'std', 'min', 'max'],
        'sms_count': 'mean'
    }).reset_index()
    
    # 扁平化列名
    usage_agg.columns = ['user_id'] + [f'{col[0]}_{col[1]}' for col in usage_agg.columns[1:]]
    
    # 消费特征
    billing_agg = billing.groupby('user_id').agg({
        'arpu': ['mean', 'std'],
        'balance': ['mean', 'min'],
        'recharge_amount': ['sum', 'mean']
    }).reset_index()
    billing_agg.columns = ['user_id'] + [f'{col[0]}_{col[1]}' for col in billing_agg.columns[1:]]
    
    # 合并所有特征
    features = users.merge(usage_agg, on='user_id')
    features = features.merge(billing_agg, on='user_id')
    features = features.merge(interaction, on='user_id')
    features = features.merge(churn, on='user_id')
    
    # 创建衍生特征
    features['value_usage_ratio'] = features['arpu_mean'] / (features['mou_mean'] + 1)
    features['balance_recharge_ratio'] = features['balance_mean'] / (features['recharge_amount_mean'] + 1)
    
    # 使用趋势特征（简化：使用标准差表示波动）
    features['mou_volatility'] = features['mou_std'] / (features['mou_mean'] + 1)
    features['dou_volatility'] = features['dou_std'] / (features['dou_mean'] + 1)
    
    return features

# 构建特征
features = create_features(users, usage, billing, interaction, churn)
```

```python
features['gender_encoded'] = le.fit_transform(features['gender'])
features['city_tier_encoded'] = le.fit_transform(features['city_tier'])

# 添加编码后的特征
feature_cols.extend(['gender_encoded', 'city_tier_encoded'])
X = features[feature_cols].fillna(0)

# 特征选择
selector = SelectKBest(score_func=f_classif, k=10)
X_selected = selector.fit_transform(X, y)

# 查看选中的特征
selected_features = [feature_cols[i] for i in selector.get_support(indices=True)]
print("\n=== 选中的TOP10特征 ===")
for i, feat in enumerate(selected_features):
    score = selector.scores_[selector.get_support()][i]
    print(f"{i+1}. {feat}: {score:.2f}")
```
