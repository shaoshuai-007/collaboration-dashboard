### 二、多轮讨论执行器

#### 改造前
```python
# 单轮循环
for agent in agents:
    response = generate(agent, task)
```

#### 改造后
```python
# 多轮循环 + 质疑检测
for round in range(max_rounds):
    for agent in participants:
        response = generate(agent, task, round, context)
        if detect_challenge(response):
            challenge_count += 1
    if check_consensus():
        break
```

#### 关键创新
- **动态轮次**：基于复杂度、参与者数量计算上限
- **质疑检测**：自动识别"不同意"、"不可行"等12种质疑词
- **共识判断**：同意人数>半数即达成共识
- **上下文传递**：每轮引用前序讨论内容

### 四、多产出物工作流

#### 改造前
```python
# 无工作流，直接执行
for task in tasks:
    execute(task)
```

#### 改造后
```python
# 拓扑排序 + 上下文传递
ordered = topological_sort(outputs)  # 按依赖排序
for output in ordered:
    context = get_dependencies(output)  # 获取前置产出物
    result = execute(output, context)   # 执行并传递上下文
```

#### 关键创新
- **依赖定义**：SRS→SAD→DDD、SRS+ SAD→PP、PP→CE
- **拓扑排序**：自动按依赖关系排序
- **上下文传递**：后面的产出物引用前面的内容
