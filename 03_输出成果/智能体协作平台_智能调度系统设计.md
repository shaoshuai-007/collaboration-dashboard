## 一、设计背景

### 1.1 当前问题

| 问题 | 影响 | 严重程度 |
|------|------|:--------:|
| 9个Agent全员发言 | 信息过载，讨论焦点分散 | 高 |
| 无意图识别 | 无法理解用户真实需求 | 高 |
| 调度策略缺失 | 所有人对所有事发言，效率低 | 高 |
| 输出非结构化 | 讨论结果难以直接使用 | 高 |

### 1.2 设计目标

| 目标 | 说明 | 衡量标准 |
|------|------|----------|
| **精准调度** | 根据意图选择相关Agent参与 | 讨论人数减少50%，聚焦度提升 |
| **自然理解** | 识别用户语义，理解真实意图 | 意图识别准确率≥90% |
| **分层协作** | 聚焦讨论+全员辩论+共识输出 | 讨论效率提升，风险覆盖完整 |
| **结构化输出** | 将讨论转化为可交付文档 | 支持Word/Excel导出 |

## 二、核心架构

### 2.1 三层智能架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Layer 1: 意图识别层                       │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  用户输入 → 语义分析 → 意图分类 → 任务类型判断            │ │
│  │                                                         │ │
│  │  示例：                                                  │ │
│  │  "帮我写需求文档" → 需求分析类任务                        │ │
│  │  "评估技术方案风险" → 风险评估类任务                      │ │
│  │  "项目成本估算" → 成本分析类任务                          │ │
│  └─────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                   Layer 2: Agent调度层（南乔总控）             │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  意图 → Agent角色匹配 → 选择参与人员 → 流程编排           │ │
│  │                                                         │ │
│  │  职责：                                                  │ │
│  │  1. 意图识别：理解用户真正想要什么                        │ │
│  │  2. Agent调度：选择最合适的角色参与                       │ │
│  │  3. 流程控制：控制讨论节奏，防止跑题                      │ │
│  │  4. 共识收敛：引导达成一致意见                           │ │
│  │  5. 输出整合：将讨论结果结构化输出                        │ │
│  └─────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                     Layer 3: 协作讨论层                       │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  三阶段协作流程：                                        │ │
│  │                                                         │ │
│  │  阶段一：聚焦讨论（精准调度）                             │ │
│  │  → 相关Agent专业深入讨论                                 │ │
│  │  → 输出方案初稿                                         │ │
│  │                                                         │ │
│  │  阶段二：风险辩论（全员参与）                             │ │
│  │  → 全员多角度挑战质疑                                    │ │
│  │  → 充分暴露风险漏洞                                     │ │
│  │                                                         │ │
│  │  阶段三：共识输出（南乔整合）                             │ │
│  │  → 引导共识达成                                         │ │
│  │  → 结构化文档输出                                       │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心洞察

**分层处理，而非全盘精准调度**

| 阶段 | 参与方式 | 目的 | 适用场景 |
|------|----------|------|----------|
| **聚焦讨论** | 精准调度 | 专业深入 | 需求分析、技术选型、成本估算 |
| **风险辩论** | 全员参与 | 充分暴露 | 方案评估、风险识别、共识达成 |
| **共识输出** | 南乔整合 | 结构化 | 文档生成、方案汇总 |

**设计原则**：
- 精准调度保证效率
- 全员辩论保证风险覆盖
- 结构化输出保证可用性

### 4.3 调度策略

**精准调度策略**：
```python
def select_agents(intent: str, context: dict) -> list:
    """
    根据意图选择参与Agent
    
    Args:
        intent: 意图类型
        context: 上下文信息（预算、时间、复杂度等）
    
    Returns:
        参与Agent列表
    """
    mapping = {
        "REQ": {
            "lead": "BA",
            "required": ["TA", "UX"],
            "optional": ["PM", "PD"]
        },
        "TEC": {
            "lead": "TA",
            "required": ["RA"],
            "optional": ["BA", "PM"]
        },
        # ... 其他映射
    }
    
    agents = [mapping[intent]["lead"]] + mapping[intent]["required"]
    
    # 根据上下文添加可选Agent
    if context.get("complexity") == "high":
        agents.extend(mapping[intent]["optional"])
    
    return agents
```

## 五、三阶段协作流程

### 5.1 阶段一：聚焦讨论（精准调度）

**触发条件**：用户输入意图识别完成

**参与方式**：精准调度相关Agent

**流程**：
```
1. 南乔识别意图
2. 南乔调度相关Agent
3. 主导Agent开场，明确讨论目标
4. 各Agent按专业角度发言
5. 输出方案初稿
```

**示例对话**：
```
用户：帮我分析智能客服系统的需求

南乔：
"收到！这是智能客服系统需求分析任务。
已邀请需求分析师、技术架构师、用户体验师参与。
请需求分析师先从业务角度分析客户需求..."

[需求分析师] 发言...
[技术架构师] 发言...
[用户体验师] 发言...

南乔：
"阶段一讨论完成，输出需求初稿。
现在进入阶段二：全员风险辩论。"
```

### 5.2 阶段二：风险辩论（全员参与）

**触发条件**：方案初稿输出完成

**参与方式**：全员参与，多角度挑战

**流程**：
```
1. 南乔宣布方案初稿
2. 全员轮流发言挑战
3. 主导Agent回应质疑
4. 风险暴露并记录
5. 方案修订完善
```

**辩论规则**：
- 风险评估师必须提出至少2个风险点
- 技术架构师必须质疑可行性
- 成本分析师必须质疑成本合理性
- 其他Agent按专业角度补充

**示例对话**：
```
南乔：
"方案初稿已完成，进入全员辩论阶段。
请各位从专业角度挑战这个方案。"

[风险评估师]："这个方案有个风险..."
[技术架构师]："技术上有个问题..."
[成本分析师]："成本方面有个疑虑..."
[测试工程师]："质量保障方面..."

南乔：
"辩论完成，共暴露5个风险点。
请主导Agent回应并修订方案。"
```

### 5.3 阶段三：共识输出（南乔整合）

**触发条件**：辩论完成，方案修订完成

**参与方式**：南乔主导，全员确认

**流程**：
```
1. 南乔整合讨论结果
2. 提炼核心共识点
3. 生成结构化文档
4. 全员确认无异议
5. 输出最终文档
```

**输出格式**：
- Word文档（需求规格说明书）
- Excel表格（成本估算表、风险清单）
- Markdown文档（讨论纪要）

## 九、风险与应对

## 十、总结

### 核心价值

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 效率 | 9人讨论，信息过载 | 3-5人聚焦，高效 |
| 质量 | 泛泛而谈 | 专业深入+风险充分暴露 |
| 输出 | 对话流 | 结构化文档 |
| 体验 | 喧闹混乱 | 清晰有序 |

### 设计理念

> 「兼听则明，偏信则暗」

- **精准调度**：让讨论聚焦，专业深入
- **全员辩论**：让风险暴露，决策稳健
- **结构化输出**：让结果可用，价值落地
