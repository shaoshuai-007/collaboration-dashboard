# 代码自动同步方案

## 三种自动化方式

### 方式一：Cron定时同步（推荐）

**优点**：简单、稳定、资源占用低

**配置步骤**：

```bash
# 编辑crontab
crontab -e

# 添加定时任务（每5分钟同步一次）
*/5 * * * * /root/.openclaw/workspace/scripts/auto_sync.sh

# 或者每小时同步一次
0 * * * * /root/.openclaw/workspace/scripts/auto_sync.sh
```

---

### 方式二：文件监控服务

**优点**：实时同步、精确检测变更

**启动方式**：

```bash
# 前台运行
python3 /root/.openclaw/workspace/scripts/auto_sync_service.py

# 后台运行
nohup python3 /root/.openclaw/workspace/scripts/auto_sync_service.py > /tmp/sync.log 2>&1 &
```

**配置参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| sync_interval | 300s | 同步间隔（5分钟） |
| watch_dirs | 03_输出成果, skills | 监控目录 |
| file_extensions | .py, .md, .html等 | 监控文件类型 |

---

### 方式三：手动同步命令

**优点**：灵活、可控

**使用方式**：

```bash
# 同步所有变更
bash /root/.openclaw/workspace/scripts/auto_sync.sh

# 同步并指定提交信息
bash /root/.openclaw/workspace/scripts/auto_sync.sh "feat: 新增功能模块"
```

---

## 工作流程

```
代码修改
    ↓
检测变更（文件hash对比）
    ↓
复制文件到同步目录
    ↓
Git commit + push
    ↓
GitHub仓库更新
```

---

## 使用建议

| 场景 | 推荐方式 |
|------|---------|
| 日常开发 | Cron定时同步 |
| 重要提交 | 手动同步（可指定提交信息） |
| 实时协作 | 文件监控服务 |

---

## 快速开始

```bash
# 1. 给脚本执行权限
chmod +x /root/.openclaw/workspace/scripts/auto_sync.sh

# 2. 设置Cron定时任务
(crontab -l 2>/dev/null; echo "*/5 * * * * /root/.openclaw/workspace/scripts/auto_sync.sh") | crontab -

# 3. 验证
crontab -l
```

---

## 监控与日志

**查看同步日志**：
```bash
# Cron日志
grep CRON /var/log/syslog

# 同步脚本输出
tail -f /tmp/sync.log
```

**检查同步状态**：
```bash
# 查看GitHub最新提交
curl -s "https://api.github.com/repos/shaoshuai-007/collaboration-dashboard/commits" | head -20
```
