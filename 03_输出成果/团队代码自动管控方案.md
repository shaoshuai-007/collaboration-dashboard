# 团队代码自动管控方案

> 九星智囊团 - 2026-03-17

---

## 一、管控架构

| 环节 | 负责人 | 频率 | 机制 |
|------|:------:|:----:|------|
| 代码变更监控 | 📐 工尺 | 实时 | 监控文件变更 |
| 自动提交 | ⚖️ 玉衡 | 每小时 | 定时任务 |
| 自动推送 | ⚖️ 玉衡 | 每小时 | Git push |
| 变更日志 | 📚 折桂 | 每次提交 | 记录变更内容 |

---

## 二、管控流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   📐 工尺   │ ──▶ │   ⚖️ 玉衡   │ ──▶ │   📚 折桂   │
│  监控变更   │     │  提交推送   │     │  记录日志   │
└─────────────┘     └─────────────┘     └─────────────┘
      ↓                    ↓                    ↓
   检测变更            git commit           JSON日志
   分析类型            git push             MD日志
   返回列表            返回状态             持久化
```

## 四、定时任务

| 任务名称 | Cron ID | 触发时间 | 说明 |
|:--------||:--------|:--------:|------|
| 团队代码管控 | `8f862da9` | 每小时整点 | 执行完整管控周期 |

## 五、使用方式

### 自动执行（推荐）
定时任务每小时自动执行，无需人工干预。

### 手动触发
```bash
python3 scripts/team_code_control.py --once
```

### 守护模式
```bash
python3 scripts/team_code_control.py --interval 30
```

## 六、日志示例

### JSON日志
```json
{
  "时间": "2026-03-17 23:23:08",
  "变更数量": 2,
  "提交状态": "成功",
  "变更详情": [
    {"status": "M", "file": "scripts/xxx.py", "type": "修改"},
    {"status": "??", "file": "scripts/new.py", "type": "新增"}
  ]
}
```

### Markdown日志
```markdown
## 23:23:08 变更记录

- [修改] scripts/xxx.py
- [新增] scripts/new.py
```

## 七、责任矩阵

| Agent | 职责 | 产出 |
|-------|------|------|
| 📐 工尺 | 监控代码变更 | 变更列表 |
| ⚖️ 玉衡 | 提交和推送 | Git提交 |

## 八、异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| Git命令超时 | 60秒超时自动终止 |
| 推送失败 | 记录失败状态，下次重试 |
| 网络问题 | 使用GIT_SSL_NO_VERIFY绕过 |
