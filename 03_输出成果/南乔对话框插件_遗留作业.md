# 南乔对话框插件 - 遗留作业

> 提出时间：2026-03-14 凌晨
> 提出人：少帅
> 创新点：根据语气、语境动态生成对话框格式

---

## 一、需求描述

### 核心功能
根据对话的**语气**和**语境**，动态生成不同风格的对话框格式，渲染成美观的图片。

### 目标用户
- 南乔自己使用
- 支持手机和PC端

### 创新价值
- 对话格式智能化、个性化
- 提升用户体验
- AI自我进化的体现

---

## 二、功能设计

### 2.1 语气识别

| 语气类型 | 特征 | 推荐格式 |
|----------|------|----------|
| 正式 | 汇报、总结、计划 | 引用框、左侧色条 |
| 轻松 | 闲聊、开玩笑 | 极简单行、标签云 |
| 紧急 | 提醒、警告 | 深色主题、双栏 |
| 温暖 | 关心、鼓励 | 卡片版、引用框 |
| 专业 | 分析、方案 | 时间轴、双栏布局 |

### 2.2 语境识别

| 语境类型 | 触发条件 | 推荐格式 |
|----------|----------|----------|
| 项目汇报 | 包含"项目"、"任务"、"进度" | 时间轴 |
| 问题分析 | 包含"问题"、"风险"、"差距" | 引用框 |
| 每日总结 | 包含"今日"、"成果"、"明日" | 双栏布局 |
| 快速回复 | 内容简短（<50字） | 极简单行 |
| 详细分析 | 内容较长（>200字） | 卡片版 |

### 2.3 格式模板

| 样式 | 适用场景 | 特点 |
|:----:|----------|------|
| 1 | 日常对话 | 简洁气泡 |
| 2 | 重点突出 | 卡片版 |
| 3 | 快速汇报 | 精简版 |
| 4 | 正式报告 | 左侧色条 |
| 5 | 夜间模式 | 深色主题 |
| 6 | 对比展示 | 双栏布局 |
| 7 | 进度汇报 | 时间轴 |
| 8 | 一句话总结 | 极简单行 |
| 9 | 关键词展示 | 标签云 |
| 10 | 正式引用 | 引用框 |

## 三、技术方案

### 3.1 核心流程

```
输入文本 → 语气识别 → 语境识别 → 选择模板 → 填充内容 → 渲染图片 → 发送
```

### 3.2 技术栈

| 模块 | 技术选型 |
|------|----------|
| 语气识别 | LLM API / 规则匹配 |
| 格式渲染 | Pillow (Python) / Canvas (Node.js) |
| 图片生成 | 服务端渲染 |
| 多端支持 | HTTP API + Webhook |

### 3.3 插件架构

```
nanqiao-dialog-plugin/
├── analyzer/          # 语气语境分析
│   ├── tone.py        # 语气识别
│   └── context.py     # 语境识别
├── templates/         # 格式模板
│   ├── style_1.png    # 模板图片
│   └── ...
├── renderer/          # 渲染引擎
│   ├── pillow_render.py
│   └── canvas_render.js
├── api/               # 接口层
│   ├── server.py      # Flask/FastAPI
│   └── webhook.py     # Webhook回调
└── config/            # 配置
    └── settings.yaml
```

## 四、接口设计

### 4.1 生成对话框图片

**请求**：
```json
POST /api/dialog/generate
{
  "text": "今日成果：一期交付完成",
  "tone": "formal",      // 可选，不传则自动识别
  "context": "report",   // 可选，不传则自动识别
  "style": 10            // 可选，不传则自动选择
}
```

**响应**：
```json
{
  "status": "success",
  "image_url": "/output/dialog_20260314_001.png",
  "style_used": 10,
  "tone_detected": "formal",
  "context_detected": "report"
}
```

### 4.2 Webhook回调

```json
POST /webhook/dialog
{
  "session_id": "xxx",
  "message": "生成的图片路径",
  "image_url": "/output/dialog_xxx.png"
}
```
