## 1. 概述

### 1.1 项目背景

多智能体协同平台V15.1已实现多Agent协同讨论功能，但讨论结束后只能输出会议纪要，无法生成用户需要的可交付产物。V15.2第二期的核心目标是实现**从"讨论共识"到"实际产出"的转化**。

### 1.2 目标用户

| 用户角色 | 职责 | 使用场景 |
|---------|------|---------|
| 项目经理 | 项目规划与管控 | 需要快速生成项目计划、甘特图、风险登记册 |
| 产品经理 | 需求管理与产品规划 | 需要输出需求规格说明书、需求清单 |
| 技术负责人 | 架构设计与技术决策 | 需要输出架构设计文档、接口文档 |
| 售前工程师 | 方案输出与客户对接 | 需要快速生成方案PPT、报价单 |
| 部门领导 | 决策与汇报 | 需要查看精简的汇报材料 |

### 1.3 核心价值

- **效率提升**：从讨论到产物的输出时间从数小时缩短到数分钟
- **质量保证**：产物格式规范、内容完整、逻辑一致
- **一致性保障**：多产物之间内容自动对齐，避免矛盾
- **灵活性**：支持多种产物类型和导出格式

### 1.4 范围说明

**本期范围**：
- 产物生成引擎
- 产物模板库
- 质量检查模块
- 产物导出功能

**下期规划**：
- 自定义模板上传
- 多人协作编辑
- 知识沉淀自动化

### 3.2 功能详细说明

#### 3.2.1 产物生成引擎

##### F-001：产物类型自动识别

**功能描述**：
讨论结束后，系统自动分析讨论内容，识别用户需要生成的产物类型。

**业务规则**：
| 规则ID | 规则描述 |
|:------:|---------|
| BR-001 | 识别关键词触发产物类型（如"需求文档"→需求规格说明书） |
| BR-002 | 识别场景触发产物类型（如"项目立项"→需求文档+架构设计+项目计划） |
| BR-003 | 识别讨论主题触发产物类型（如技术讨论→技术方案） |
| BR-004 | 支持用户手动指定产物类型覆盖自动识别 |

**输入**：讨论会话内容  
**输出**：产物类型列表（带置信度）

**接口设计**：
```
POST /api/v1/agent/product/identify
Request: { sessionId: string, content: string }
Response: { 
  products: [
    { type: "REQUIREMENT_DOC", confidence: 0.85 },
    { type: "ARCHITECTURE_DOC", confidence: 0.72 }
  ]
}
```

---

##### F-002：Agent调度引擎

**功能描述**：
根据产物类型，调度对应的专业Agent执行产物生成。

**产物-Agent映射规则**：
| 产物类型 | 负责Agent | Agent能力要求 |
|---------|:--------:|--------------|
| 需求规格说明书 | 采薇 | 需求分析、用户故事编写 |
| 架构设计文档 | 织锦 | 架构设计、技术选型 |
| 详细设计文档 | 工尺 | 详细设计、接口设计 |
| 方案PPT | 呈彩 | 内容组织、演示设计 |
| 项目计划 | 玉衡 | 项目管理、进度规划 |
| 报价单 | 筑台 | 成本估算、报价编制 |
| 分析报告 | 知微 | 数据分析、报告撰写 |

**输入**：产物类型、讨论上下文  
**输出**：生成任务ID

**接口设计**：
```
POST /api/v1/agent/product/generate
Request: { 
  productType: string, 
  context: object,
  templateId?: string
}
Response: { 
  taskId: string,
  estimatedTime: number 
}
```

##### F-003：多产物工作流

**功能描述**：
支持一次性生成多个产物，自动处理产物之间的依赖关系和上下文传递。

**依赖关系定义**：
```
产物依赖图：
需求规格说明书 → 架构设计文档 → 详细设计文档
                ↓
            方案PPT
                ↓
            项目计划 → 报价单
```

**执行规则**：
| 规则ID | 规则描述 |
|:------:|---------|
| BR-005 | 按依赖关系拓扑排序执行顺序 |
| BR-006 | 上游产物生成后，自动提取关键信息传递给下游 |
| BR-007 | 支持并行执行无依赖关系的产物 |
| BR-008 | 任一产物生成失败，可单独重试 |

**接口设计**：
```
POST /api/v1/agent/product/workflow
Request: { 
  productTypes: string[],
  sessionId: string
}
Response: { 
  workflowId: string,
  steps: [
    { productType: string, status: "pending"|"running"|"completed"|"failed", dependsOn: string[] }
  ]
}
```

#### 3.2.3 质量检查模块

##### F-005：完整性检查

**功能描述**：
检查产物是否包含所有必填章节和内容。

**检查规则**：
| 检查项ID | 检查项名称 | 适用产物 | 检查规则 |
|:--------:|----------|---------|---------|
| C-001 | 必填章节完整性 | 文档类 | 所有标记为必填的章节必须存在 |
| C-002 | 章节内容非空 | 所有 | 必填章节内容不能为空 |
| C-003 | 表格完整性 | 表格类 | 表格必须包含所有定义的列 |
| C-004 | 幻灯片完整性 | PPT类 | 必须包含封面和结束页 |

**输出格式**：
```json
{
  "checkType": "COMPLETENESS",
  "productType": "REQUIREMENT_DOC",
  "result": "PASS" | "FAIL",
  "issues": [
    {
      "type": "MISSING_SECTION",
      "severity": "ERROR",
      "section": "3.2 功能清单",
      "message": "必填章节缺失"
    }
  ],
  "score": 85
}
```

---

##### F-006：规范性检查

**功能描述**：
检查产物格式是否符合规范要求。

**检查规则**：
| 检查项ID | 检查项名称 | 检查规则 |
|:--------:|----------|---------|
| C-005 | 标题层级规范 | 标题层级不超过4级，层级顺序正确 |
| C-006 | 表格格式规范 | 表格有表头、有边框、对齐方式一致 |
| C-007 | 图表规范 | 图表有标题、有来源说明 |
| C-008 | 引用规范 | 引用内容有标注来源 |

---

##### F-007：一致性检查

**功能描述**：
检查多个产物之间内容是否一致。

**检查规则**：
| 检查项ID | 检查项名称 | 检查规则 |
|:--------:|----------|---------|
| C-009 | 项目名称一致性 | 所有产物中项目名称一致 |
| C-010 | 功能范围一致性 | 需求文档功能范围与架构设计一致 |
| C-011 | 时间节点一致性 | 项目计划时间节点与其他产物一致 |
| C-012 | 成本数据一致性 | 报价单成本与方案内容一致 |

**输出格式**：
```json
{
  "checkType": "CONSISTENCY",
  "products": ["REQUIREMENT_DOC", "ARCHITECTURE_DOC"],
  "result": "PASS" | "FAIL",
  "issues": [
    {
      "type": "INCONSISTENT_DATA",
      "severity": "WARNING",
      "source": "REQUIREMENT_DOC",
      "target": "ARCHITECTURE_DOC",
      "field": "功能范围",
      "message": "需求文档包含'用户管理'模块，架构设计中未体现"
    }
  ],
  "score": 72
}
```

#### 3.2.4 产物导出功能

##### F-009/F-010/F-011/F-015：格式导出

**功能描述**：
支持将生成的产物导出为多种格式。

**支持格式**：
| 格式 | 扩展名 | 适用产物类型 | 优先级 |
|-----|:------:|-------------|:------:|
| Word | .docx | 文档类 | Must |
| Excel | .xlsx | 表格类 | Must |
| PowerPoint | .pptx | 演示类 | Must |
| Markdown | .md | 文档类 | Should |

**技术实现**：
- Word：使用 python-docx 库
- Excel：使用 openpyxl 库
- PowerPoint：使用 python-pptx 库
- Markdown：自定义渲染引擎

**接口设计**：
```
POST /api/v1/product/export
Request: { 
  productId: string,
  format: "docx" | "xlsx" | "pptx" | "md"
}
Response: { 
  downloadUrl: string,
  expiresAt: string
}
```

## 附录A：验收标准清单

### AC-001：产物类型识别验收标准

```
Given: 讨论会话已结束
When: 系统分析讨论内容
Then: 
  - 输出产物类型列表
  - 每个产物类型带有置信度分数
  - 准确率 ≥ 80%
```

### AC-002：Agent调度验收标准

```
Given: 已识别产物类型
When: 系统调度对应Agent
Then:
  - 正确的Agent被调度
  - 生成任务成功创建
  - 返回任务ID和预计时间
```

### AC-003：多产物工作流验收标准

```
Given: 用户请求生成多个产物
When: 系统执行工作流
Then:
  - 按依赖顺序执行
  - 上下文正确传递
  - 所有产物生成成功
  - 产物之间内容一致
```

### AC-005：模板应用验收标准

```
Given: 产物正在生成
When: 应用标准化模板
Then:
  - 输出包含所有标准章节
  - 格式符合规范
  - 可直接使用或微调
```

### AC-008：完整性检查验收标准

```
Given: 产物已生成
When: 执行完整性检查
Then:
  - 识别所有缺失项
  - 输出缺失项列表
  - 给出补充建议
```

### AC-010：一致性检查验收标准

```
Given: 多个产物已生成
When: 执行一致性检查
Then:
  - 识别所有不一致项
  - 定位不一致的位置
  - 给出修正建议
```
