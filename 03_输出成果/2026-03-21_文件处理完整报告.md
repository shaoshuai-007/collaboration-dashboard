## 三、对"AI+客户群洞察"项目的启示

### 3.1 现状业务流程

```
政策盘点 → 标签筛选 → CPCP分群 → 场景输出 → 一线执行
    ↓           ↓           ↓           ↓           ↓
  场景定义   6000+标签   SQL规则     目标用户    话术推荐
```

### 3.2 核心痛点

| 痛点 | 现状 | AI解决方案 |
|------|------|-----------|
| 标签选择难 | 6000+标签，同质化严重 | 自然语言→标签推荐 |
| 分群门槛高 | 3人半年完成场景配餐 | AI辅助分群规则生成 |
| 规则维护难 | SQL-like表达式复杂 | 自然语言→规则转换 |
| 场景复制难 | 依赖个别专家经验 | 场景知识库沉淀 |

### 3.3 AI核心功能设计

**1. 自然语言→标签推荐**

```
用户输入："找出流量需求高的用户"
    ↓
AI分析：流量需求度 = 高
    ↓
标签推荐：
- 上月套餐流量饱和度≥80%
- 近三月月均流量使用量>10GB
- 近三月有流量套外溢出费用
```

**2. 自然语言→分群规则生成**

```
用户输入："找出适合推荐FTTR的宽带用户"
    ↓
AI生成分群规则：
bas_dcol1499 IN ['光猫类型1', '光猫类型2']
AND bas_dcol291 < 1000
AND bas_dcol841 = '未办理全屋WIFI'
```
