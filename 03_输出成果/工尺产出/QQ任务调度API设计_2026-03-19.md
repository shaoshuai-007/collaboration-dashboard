## 一、API概述

| 项目 | 值 |
|------|-----|


## 二、核心API设计

### 2.1 任务创建API

**POST /tasks**

接收南乔解析后的任务，创建调度任务。

#### 请求参数

```json
{
  "task_id": "task_20260319_001",
  "task_type": "code_generation",
  "priority": "normal",
  "source": {
    "platform": "qq",
    "user_id": "12345678",
    "group_id": "87654321",
    "message_id": "msg_abc123"
  },
  "payload": {
    "description": "帮我写一个Python爬虫",
    "context": {
      "language": "python",
      "requirements": ["requests", "beautifulsoup4"]
    }
  },
  "callback_url": "http://nanqiao:8080/callback",
  "created_at": "2026-03-19T10:47:00+08:00"
}
```

#### 参数说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | string | ✓ | 任务唯一标识 |
| task_type | string | ✓ | 任务类型：code_generation/research/analysis/custom |
| priority | string | | 优先级：urgent/high/normal/low |
| source | object | ✓ | 任务来源信息 |
| payload | object | ✓ | 任务负载（南乔解析结果） |
| callback_url | string | | 完成回调地址 |

#### 响应示例

```json
{
  "code": 200,
  "message": "任务创建成功",
  "data": {
    "task_id": "task_20260319_001",
    "status": "pending",
    "assigned_agent": null,
    "queue_position": 3,
    "estimated_start": "2026-03-19T10:50:00+08:00",
    "created_at": "2026-03-19T10:47:00+08:00"
  }
}
```

#### OpenAPI 3.0 定义

```yaml
openapi: 3.0.3
info:
  title: QQ任务调度API
  version: 1.0.0
paths:
  /tasks:
    post:
      summary: 创建任务
      operationId: createTask
      tags:
        - Task
      security:
        - BearerAuth: []
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/TaskCreateRequest'
      responses:
        '200':
          description: 任务创建成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/TaskCreateResponse'
        '400':
          description: 请求参数错误
        '401':
          description: 认证失败
        '503':
          description: 服务不可用
```

#### 方案B：WebSocket

**WS /ws/tasks/{task_id}/logs**

#### 连接示例

```javascript
const ws = new WebSocket('ws://localhost:8080/ws/tasks/task_20260319_001/logs');

ws.onopen = () => {
  ws.send(JSON.stringify({
    action: 'subscribe',
    task_id: 'task_20260319_001',
    token: 'Bearer xxx'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(data);
};
```

## 三、实时推送设计

### 3.1 推送架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   南乔(QQ)   │────▶│  调度中心    │────▶│   Agent池   │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
                   ┌─────────────┐
                   │ 消息队列    │
                   │ (Redis Pub) │
                   └─────────────┘
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
     ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
     │ SSE连接池   │ │ WS连接池    │ │ 日志存储    │
     └─────────────┘ └─────────────┘ └─────────────┘
            │             │
            ▼             ▼
     ┌───────────────────────────┐
     │   前端可视化面板           │
     └───────────────────────────┘
```

### 3.2 推送事件类型

| 事件类型 | 触发条件 | 推送内容 |
|---------|---------|---------|
| task.created | 任务创建 | 任务基本信息 |
| task.assigned | 任务分配Agent | Agent信息 |
| task.progress | 进度更新 | 进度百分比+阶段 |
| task.log | 日志输出 | 日志内容 |
| task.completed | 任务完成 | 产出引用 |
| task.failed | 任务失败 | 错误信息 |
| agent.status | Agent状态变更 | 新状态 |

### 3.3 推送示例

```json
// 任务进度推送
{
  "event": "task.progress",
  "timestamp": "2026-03-19T10:49:00+08:00",
  "data": {
    "task_id": "task_20260319_001",
    "progress": 75,
    "stage": "execution",
    "message": "代码验证中"
  }
}

// 任务完成推送
{
  "event": "task.completed",
  "timestamp": "2026-03-19T10:50:00+08:00",
  "data": {
    "task_id": "task_20260319_001",
    "status": "completed",
    "output_ref": "output_001",
    "execution_time_ms": 180000,
    "callback_sent": true
  }
}
```

```bash
curl -N http://localhost:8080/api/v1/tasks/task_001/logs/stream \
```

### JavaScript 示例

```javascript
// SSE日志流
const eventSource = new EventSource(
  '/api/v1/tasks/task_001/logs/stream',
  { headers: { 'Authorization': 'Bearer YOUR_TOKEN' } }
);

eventSource.addEventListener('log', (event) => {
  const data = JSON.parse(event.data);
  console.log(`[${data.level}] ${data.agent}: ${data.message}`);
});

eventSource.addEventListener('progress', (event) => {
  const data = JSON.parse(event.data);
  updateProgressBar(data.progress);
});

eventSource.addEventListener('completed', (event) => {
  const data = JSON.parse(event.data);
  fetchOutput(data.output_ref);
  eventSource.close();
});
```

## 七、总结

本API设计覆盖任务调度全流程：

1. **任务创建** - 接收南乔解析后的任务，返回队列位置
2. **状态查询** - 实时获取任务执行状态和进度
3. **Agent状态** - 监控Agent池健康状况
4. **实时日志** - SSE/WebSocket双方案，按需选择
5. **产出获取** - 支持元数据查询和文件下载

建议前端可视化面板采用 **SSE方案**，实现简单、兼容性好、支持断线重连。
