## 1. RESTful API设计规范

### 1.1 资源命名规范

#### 规范说明

资源命名是RESTful API设计的核心，好的命名能让API直观易懂。

| 规则 | 说明 |
|------|------|
| 使用名词 | 资源名应为名词，表示业务实体 |
| 使用复数 | 集合资源使用复数形式 |
| 使用小写 | 全部小写字母 |
| 使用连字符 | 多个单词用连字符`-`连接 |
| 避免动词 | 动作通过HTTP方法表达 |
| 层级清晰 | 用路径表达资源层级关系 |

#### 示例

**✅ 正确示例：**

```
GET    /users                    # 获取用户列表
GET    /users/{id}               # 获取单个用户
GET    /users/{id}/orders        # 获取用户的订单列表
POST   /users                    # 创建用户
GET    /order-items              # 获取订单项列表（多词用连字符）
GET    /products/{productId}/reviews  # 获取产品的评论
```

**❌ 错误示例：**

```
GET    /getUsers                 # ❌ 不应在URL中使用动词
GET    /user                     # ❌ 集合应使用复数
GET    /Users                    # ❌ 应使用小写
GET    /order_items              # ❌ 应使用连字符而非下划线
POST   /createUser               # ❌ 动作由HTTP方法表达
```

#### 注意事项

- **版本控制**：在URL中包含版本号，如 `/api/v1/users`
- **避免过深层级**：资源层级不超过3层，复杂查询使用查询参数
- **避免文件扩展名**：不要在URL中使用 `.json`、`.xml` 等扩展名
- **特殊情况**：非CRUD操作（如登录、导出）可作为子资源或使用动词，如 `/users/{id}/activate`

### 1.2 HTTP方法使用

#### 规范说明

HTTP方法定义了对资源的操作类型，应严格遵循其语义。

| 方法 | 语义 | 幂等性 | 安全性 | 使用场景 |
|------|------|--------|--------|----------|
| GET | 获取资源 | ✅ 幂等 | ✅ 安全 | 查询、读取 |
| POST | 创建资源 | ❌ 非幂等 | ❌ 不安全 | 新增、复杂查询 |
| PUT | 全量更新 | ✅ 幂等 | ❌ 不安全 | 整体替换 |
| PATCH | 部分更新 | ❌ 非幂等 | ❌ 不安全 | 局部修改 |
| DELETE | 删除资源 | ✅ 幂等 | ❌ 不安全 | 删除 |

> **幂等性**：多次相同请求产生相同结果
> **安全性**：请求不会改变服务器状态

#### 示例

```
# 用户资源操作示例

# 创建用户（POST）
POST /api/v1/users
Content-Type: application/json

{
  "name": "张三",
  "email": "zhangsan@example.com"
}

# 获取用户列表（GET）
GET /api/v1/users?page=1&size=20

# 获取单个用户（GET）
GET /api/v1/users/12345

# 全量更新用户（PUT）- 需提供完整资源
PUT /api/v1/users/12345
Content-Type: application/json

{
  "name": "李四",
  "email": "lisi@example.com",
  "age": 25,
  "status": "active"
}

# 部分更新用户（PATCH）- 只提供修改字段
PATCH /api/v1/users/12345
Content-Type: application/json

{
  "name": "王五"
}

# 删除用户（DELETE）
DELETE /api/v1/users/12345
```

#### 注意事项

- **POST vs PUT**：POST用于创建（服务端生成ID），PUT用于更新（客户端提供ID）
- **PUT vs PATCH**：PUT替换整个资源，PATCH只修改部分字段
- **批量操作**：批量删除使用 `DELETE /users?id=1,2,3` 或 POST到专用端点
- **查询参数**：复杂查询条件使用POST + 请求体，避免URL过长
- **禁止滥用GET**：GET请求不应有副作用，不应修改数据

### 1.3 状态码定义

#### 规范说明

HTTP状态码是API响应的重要组成部分，准确使用能帮助客户端正确处理响应。

| 状态码 | 类别 | 含义 | 常用场景 |
|--------|------|------|----------|
| **2xx** | 成功 | 请求成功处理 | |
| 200 | OK | 请求成功 | GET/PUT/PATCH/DELETE成功 |
| 201 | Created | 资源创建成功 | POST创建成功 |
| 202 | Accepted | 请求已接受，异步处理 | 长时间任务 |
| 204 | No Content | 成功但无返回内容 | DELETE成功 |
| **3xx** | 重定向 | 需要进一步操作 | |
| 301 | Moved Permanently | 资源永久迁移 | API版本升级 |
| 302 | Found | 资源临时迁移 | 临时重定向 |
| **4xx** | 客户端错误 | 客户端请求有问题 | |
| 400 | Bad Request | 请求参数错误 | 参数校验失败 |
| 401 | Unauthorized | 未认证 | 未登录或token过期 |
| 403 | Forbidden | 已认证但无权限 | 权限不足 |
| 404 | Not Found | 资源不存在 | 资源已删除或不存在 |
| 405 | Method Not Allowed | 方法不允许 | POST端点用GET访问 |
| 409 | Conflict | 资源冲突 | 重复创建、版本冲突 |
| 422 | Unprocessable Entity | 语义错误 | 业务规则校验失败 |
| 429 | Too Many Requests | 请求过于频繁 | 限流触发 |
| **5xx** | 服务器错误 | 服务端处理失败 | |
| 500 | Internal Server Error | 服务器内部错误 | 代码异常 |
| 502 | Bad Gateway | 网关错误 | 上游服务不可用 |
| 503 | Service Unavailable | 服务不可用 | 维护或过载 |
| 504 | Gateway Timeout | 网关超时 | 上游服务超时 |

#### 示例

**成功响应：**

```json
// 200 OK - GET成功
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 12345,
    "name": "张三",
    "email": "zhangsan@example.com"
  }
}

// 201 Created - POST创建成功
// 响应头应包含 Location: /api/v1/users/12345
{
  "code": 201,
  "message": "创建成功",
  "data": {
    "id": 12345,
    "name": "张三"
  }
}

// 204 No Content - DELETE成功
// 响应体为空
```

**错误响应：**

```json
// 400 Bad Request - 参数错误
{
  "code": 400,
  "message": "参数校验失败",
  "errors": [
    {
      "field": "email",
      "message": "邮箱格式不正确"
    },
    {
      "field": "age",
      "message": "年龄必须大于0"
    }
  ]
}

// 401 Unauthorized - 未认证
{
  "code": 401,
  "message": "请先登录",
  "error": "TOKEN_EXPIRED"
}

// 403 Forbidden - 权限不足
{
  "code": 403,
  "message": "无权访问该资源",
  "error": "PERMISSION_DENIED"
}

// 404 Not Found - 资源不存在
{
  "code": 404,
  "message": "用户不存在",
  "error": "USER_NOT_FOUND"
}

// 422 Unprocessable Entity - 业务规则错误
{
  "code": 422,
  "message": "库存不足，无法下单",
  "error": "INSUFFICIENT_STOCK",
  "data": {
    "productId": "P001",
    "requested": 10,
    "available": 5
  }
}
```

#### 注意事项

- **统一响应格式**：成功和错误都应使用统一的JSON格式
- **错误码设计**：业务错误码使用字符串（如 `USER_NOT_FOUND`），便于国际化和理解
- **区分401和403**：401是"你是谁"，403是"你不能做这个"
- **避免暴露细节**：500错误不应返回堆栈信息给客户端
- **包含帮助信息**：错误响应应包含如何解决问题的提示或文档链接

## 2. 接口文档规范

### 2.1 OpenAPI规范

#### 规范说明

OpenAPI（原Swagger）是API文档的行业标准，使用YAML或JSON格式描述API。

**核心结构：**

```yaml
openapi: 3.0.3
info:           # API基本信息
servers:        # 服务器地址
paths:          # API路径定义
components:     # 可复用组件
  schemas:      # 数据模型
  parameters:   # 公共参数
  responses:    # 公共响应
  securitySchemes: # 安全方案
```

#### 示例

**完整的OpenAPI 3.0文档示例：**

```yaml
openapi: 3.0.3
info:
  title: 用户管理API
  description: 九星智囊团用户管理系统接口文档
  version: 1.0.0
  contact:
    name: API支持
    email: api@jiuxing.com

servers:
  - url: https://api.jiuxing.com/v1
    description: 生产环境
  - url: https://api-staging.jiuxing.com/v1
    description: 测试环境

tags:
  - name: users
    description: 用户管理
  - name: orders
    description: 订单管理

paths:
  /users:
    get:
      tags:
        - users
      summary: 获取用户列表
      description: 支持分页和筛选条件查询
      operationId: listUsers
      parameters:
        - $ref: '#/components/parameters/PageParam'
        - $ref: '#/components/parameters/SizeParam'
        - name: status
          in: query
          description: 用户状态筛选
          schema:
            type: string
            enum: [active, inactive, banned]
      responses:
        '200':
          description: 成功返回用户列表
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/UserListResponse'
        '401':
          $ref: '#/components/responses/Unauthorized'
        '500':
          $ref: '#/components/responses/InternalError'
      security:
        - bearerAuth: []

    post:
      tags:
        - users
      summary: 创建用户
      description: 创建新用户账号
      operationId: createUser
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateUserRequest'
      responses:
        '201':
          description: 用户创建成功
          headers:
            Location:
              description: 新用户资源地址
              schema:
                type: string
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/UserResponse'
        '400':
          $ref: '#/components/responses/BadRequest'
        '409':
          description: 用户已存在
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
      security:
        - bearerAuth: []

  /users/{userId}:
    get:
      tags:
        - users
      summary: 获取用户详情
      operationId: getUser
      parameters:
        - $ref: '#/components/parameters/UserIdParam'
      responses:
        '200':
          description: 成功返回用户详情
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/UserResponse'
        '404':
          $ref: '#/components/responses/NotFound'

    put:
      tags:
        - users
      summary: 更新用户信息
      operationId: updateUser
      parameters:
        - $ref: '#/components/parameters/UserIdParam'
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/UpdateUserRequest'
      responses:
        '200':
          description: 更新成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/UserResponse'

    delete:
      tags:
        - users
      summary: 删除用户
      operationId: deleteUser
      parameters:
        - $ref: '#/components/parameters/UserIdParam'
      responses:
        '204':
          description: 删除成功
        '404':
          $ref: '#/components/responses/NotFound'

components:
  schemas:
    User:
      type: object
      required:
        - id
        - name
        - email
      properties:
        id:
          type: integer
          format: int64
          description: 用户ID
          example: 12345
        name:
          type: string
          description: 用户名
          maxLength: 50
          example: 张三
        email:
          type: string
          format: email
          description: 邮箱地址
          example: zhangsan@example.com
        status:
          type: string
          enum: [active, inactive, banned]
          default: active
          description: 用户状态
        createdAt:
          type: string
          format: date-time
          description: 创建时间
        updatedAt:
          type: string
          format: date-time
          description: 更新时间

    CreateUserRequest:
      type: object
      required:
        - name
        - email
        - password
      properties:
        name:
          type: string
          minLength: 2
          maxLength: 50
        email:
          type: string
          format: email
        password:
          type: string
          format: password
          minLength: 8

    UpdateUserRequest:
      type: object
      properties:
        name:
          type: string
          minLength: 2
          maxLength: 50
        email:
          type: string
          format: email
        status:
          type: string
          enum: [active, inactive, banned]

    UserResponse:
      type: object
      properties:
        code:
          type: integer
          example: 200
        message:
          type: string
          example: success
        data:
          $ref: '#/components/schemas/User'

    UserListResponse:
      type: object
      properties:
        code:
          type: integer
          example: 200
        message:
          type: string
        data:
          type: object
          properties:
            items:
              type: array
              items:
                $ref: '#/components/schemas/User'
            total:
              type: integer
              description: 总记录数
            page:
              type: integer
              description: 当前页码
            size:
              type: integer
              description: 每页大小

    ErrorResponse:
      type: object
      properties:
        code:
          type: integer
        message:
          type: string
        error:
          type: string
          description: 错误代码
        errors:
          type: array
          items:
            type: object
            properties:
              field:
                type: string
              message:
                type: string

  parameters:
    UserIdParam:
      name: userId
      in: path
      required: true
      description: 用户ID
      schema:
        type: integer
        format: int64

    PageParam:
      name: page
      in: query
      description: 页码
      schema:
        type: integer
        minimum: 1
        default: 1

    SizeParam:
      name: size
      in: query
      description: 每页大小
      schema:
        type: integer
        minimum: 1
        maximum: 100
        default: 20

  responses:
    BadRequest:
      description: 请求参数错误
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            code: 400
            message: 参数校验失败
            errors:
              - field: email
                message: 邮箱格式不正确

    Unauthorized:
      description: 未授权访问
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            code: 401
            message: 请先登录

    NotFound:
      description: 资源不存在
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            code: 404
            message: 资源不存在

    InternalError:
      description: 服务器内部错误
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            code: 500
            message: 服务器内部错误，请稍后重试

  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
      description: JWT认证令牌
```

#### 注意事项

- **版本控制**：使用OpenAPI 3.0.x版本，兼容性好
- **描述完整**：每个字段都应有description
- **示例丰富**：使用example提供示例值
- **复用组件**：将通用定义放在components中复用
- **安全声明**：明确标注需要认证的接口
- **工具支持**：可使用Swagger Editor、Redoc等工具渲染文档

### 2.2 接口描述模板

#### 规范说明

每个接口文档应包含以下核心要素：

| 要素 | 必填 | 说明 |
|------|------|------|
| 接口名称 | ✅ | 简洁明了的接口名称 |
| 接口描述 | ✅ | 详细说明接口功能 |
| 请求方式 | ✅ | GET/POST/PUT/PATCH/DELETE |
| 请求路径 | ✅ | 完整的URL路径 |
| 请求参数 | ✅ | Query/Path/Header/Body参数 |
| 请求示例 | ✅ | 完整的请求示例 |
| 响应参数 | ✅ | 返回字段说明 |
| 响应示例 | ✅ | 成功和失败的响应示例 |
| 错误码 | ⬜ | 该接口特有的错误码 |
| 注意事项 | ⬜ | 使用说明和限制 |

#### 示例

```markdown
## 创建用户

### 基本信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 创建用户 |
| 请求方式 | POST |
| 请求路径 | /api/v1/users |
| 接口描述 | 创建新用户账号，成功后返回用户信息 |

### 请求参数

#### 请求头 (Header)

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer {token} |
| Content-Type | string | 是 | application/json |

#### 请求体 (Body)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| name | string | 是 | 用户名，2-50字符 | 张三 |
| email | string | 是 | 邮箱地址 | zhangsan@example.com |
| password | string | 是 | 密码，至少8位 | ******** |
| phone | string | 否 | 手机号码 | 13800138000 |

### 请求示例

```bash
curl -X POST 'https://api.jiuxing.com/v1/users' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "张三",
    "email": "zhangsan@example.com",
    "password": "SecurePass123!",
    "phone": "13800138000"
  }'
```

### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | integer | 状态码 |
| message | string | 提示信息 |
| data | object | 返回数据 |
| data.id | integer | 用户ID |
| data.name | string | 用户名 |
| data.email | string | 邮箱地址 |
| data.status | string | 用户状态 |
| data.createdAt | string | 创建时间 |

### 响应示例

**成功响应 (201)：**

```json
{
  "code": 201,
  "message": "创建成功",
  "data": {
    "id": 12345,
    "name": "张三",
    "email": "zhangsan@example.com",
    "status": "active",
    "createdAt": "2026-03-19T09:30:00Z"
  }
}
```

**失败响应 (400)：**

```json
{
  "code": 400,
  "message": "参数校验失败",
  "errors": [
    {
      "field": "email",
      "message": "邮箱格式不正确"
    }
  ]
}
```

### 错误码说明

| 错误码 | HTTP状态码 | 说明 |
|--------|------------|------|
| INVALID_EMAIL | 400 | 邮箱格式不正确 |
| EMAIL_EXISTS | 409 | 邮箱已被注册 |
| WEAK_PASSWORD | 400 | 密码强度不足 |

### 注意事项

1. 密码需包含大小写字母和数字，长度至少8位
2. 邮箱地址在系统中必须唯一
3. 创建成功后，响应头Location字段包含用户资源地址
```

#### 注意事项

- **结构清晰**：使用表格和代码块提高可读性
- **示例完整**：提供可直接复制使用的示例
- **包含认证**：说明需要的认证方式
- **错误齐全**：列出所有可能的错误情况
- **保持更新**：接口变更时同步更新文档

### 2.3 示例代码格式

#### 规范说明

示例代码是接口文档的重要组成部分，应提供多种语言/工具的调用示例。

**示例代码要求：**

1. 完整可运行：包含完整的请求代码
2. 参数真实：使用有意义的参数值
3. 注释清晰：关键步骤添加注释
4. 格式统一：使用统一的代码风格

#### 示例

**cURL 示例：**

```bash
# 获取用户列表
curl -X GET 'https://api.jiuxing.com/v1/users?page=1&size=20' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \

import requests

**Java (OkHttp) 示例：**

```java
import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;

public class UserApiClient {
    private final OkHttpClient client;
    private final ObjectMapper mapper;
    private final String baseUrl = "https://api.jiuxing.com/v1";
    private final String accessToken;

    public UserApiClient(String accessToken) {
        this.client = new OkHttpClient();
        this.mapper = new ObjectMapper();
        this.accessToken = accessToken;
    }

    // 获取用户列表
    public String getUsers(int page, int size) throws Exception {
        HttpUrl url = HttpUrl.parse(baseUrl + "/users")
            .newBuilder()
            .addQueryParameter("page", String.valueOf(page))
            .addQueryParameter("size", String.valueOf(size))
            .build();

        Request request = new Request.Builder()
            .url(url)
            .addHeader("Authorization", "Bearer " + accessToken)
            .addHeader("Accept", "application/json")
            .build();

        try (Response response = client.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new RuntimeException("请求失败: " + response.code());
            }
            return response.body().string();
        }
    }

    // 创建用户
    public String createUser(String name, String email, String password) throws Exception {
        String json = mapper.writeValueAsString(Map.of(
            "name", name,
            "email", email,
            "password", password
        ));

        RequestBody body = RequestBody.create(
            json, 
            MediaType.parse("application/json")
        );

        Request request = new Request.Builder()
            .url(baseUrl + "/users")
            .addHeader("Authorization", "Bearer " + accessToken)
            .post(body)
            .build();

        try (Response response = client.newCall(request).execute()) {
            if (response.code() == 409) {
                throw new RuntimeException("邮箱已被注册");
            }
            if (!response.isSuccessful()) {
                throw new RuntimeException("创建失败: " + response.code());
            }
            return response.body().string();
        }
    }
}
