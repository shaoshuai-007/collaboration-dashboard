## 1. 设计概述

### 1.1 背景与目标

多智能体协同平台V15.1第一期已完成协作框架搭建，实现了Agent间通信、任务调度、冲突处理等核心能力。第二期的核心目标是实现**协同产出**，即多个Agent协作生成标准化产物并支持全生命周期管理。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **RESTful规范** | 遵循REST架构风格，资源语义清晰 |
| **电信行业兼容** | 参考TM Forum API规范，便于与电信系统集成 |
| **标准化产物** | 产物格式标准化，支持多种输出格式 |
| **可追溯性** | 产物生成过程可追溯，支持审计 |
| **安全性** | 符合电信行业安全合规要求 |

### 1.3 参考标准

| 标准体系 | 标准名称 | 应用场景 |
|----------|----------|----------|
| TM Forum | TMF620 Product Catalog | 产物目录管理 |
| TM Forum | TMF622 Product Ordering | 产物订单管理 |
| TM Forum | TMF641 Process Management | 流程管理 |
| 3GPP | TS 32.150 | 电信管理接口规范 |
| ISO/IEC | 27001 | 信息安全管理体系 |

### 1.4 接口版本

```
/api/v2/artifacts    # 产物资源（第二期新增）
/api/v2/versions     # 版本管理（第二期新增）
/api/v2/exports      # 导出服务（第二期新增）
```

## 2. 产物生成接口

### 2.1 接口概述

产物生成接口支持多Agent协作生成标准化产物，包括需求文档、架构设计、方案PPT、接口设计、测试用例等。

### 2.2 创建产物任务

#### 基本信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/v2/artifacts/tasks` |
| 功能描述 | 创建产物生成任务，支持多Agent协作 |
| 权限要求 | 需要认证，角色：admin, leader, agent |

#### 请求参数

**请求头 (Header)**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| Authorization | string | 是 | Bearer {jwt_token} |
| Content-Type | string | 是 | application/json |
| X-Request-ID | string | 是 | 请求追踪ID，UUID格式 |
| X-Session-ID | string | 否 | 协作会话ID |

**请求体 (Body)**

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|:----:|------|------|
| artifactType | string | 是 | 产物类型 | REQUIREMENT_DOC |
| projectCode | string | 是 | 项目编码 | HBDX-AI-001 |
| collaborationMode | string | 是 | 协作模式 | SEQUENTIAL, PARALLEL, REVIEW |
| participants | array | 是 | 参与Agent列表 | ["caiwei", "zhijin"] |
| inputContext | object | 是 | 输入上下文 | 见示例 |
| outputConfig | object | 否 | 输出配置 | 格式、模板等 |
| qualityStandard | string | 否 | 质量标准等级 | A, B, C |
| deadline | datetime | 否 | 截止时间 | 2026-03-20T18:00:00+08:00 |

**产物类型枚举 (artifactType)**

| 枚举值 | 说明 | 主责Agent |
|--------|------|-----------|
| REQUIREMENT_DOC | 需求文档 | 采薇 |
| ARCHITECTURE_DESIGN | 架构设计 | 织锦 |
| SOLUTION_PPT | 方案PPT | 呈彩 |
| API_DESIGN | 接口设计 | 工尺 |
| DATABASE_DESIGN | 数据库设计 | 工尺 |
| TEST_CASE | 测试用例 | 天工 |
| CODE_IMPLEMENT | 代码实现 | 天工 |
| DATA_ANALYSIS | 数据分析报告 | 知微 |
| PROJECT_PLAN | 项目计划 | 玉衡 |
| SALES_PROPOSAL | 售前方案 | 筑台 |

**协作模式枚举 (collaborationMode)**

| 枚举值 | 说明 | 适用场景 |
|--------|------|----------|
| SEQUENTIAL | 顺序执行 | 有明确依赖关系的任务链 |
| PARALLEL | 并行执行 | 无依赖关系的独立任务 |
| REVIEW | 评审模式 | 需要多轮评审迭代 |
| DISCUSSION | 讨论模式 | 需要充分讨论达成共识 |

#### 请求示例

```json
{
    "artifactType": "REQUIREMENT_DOC",
    "projectCode": "HBDX-AI-001",
    "projectName": "湖北电信AI智能配案系统",
    "collaborationMode": "SEQUENTIAL",
    "participants": [
        {
            "agentId": "caiwei",
            "role": "LEAD",
            "tasks": ["需求调研", "需求文档编写"]
        },
        {
            "agentId": "zhiwei",
            "role": "SUPPORT",
            "tasks": ["数据支撑分析"]
        },
        {
            "agentId": "zhegui",
            "role": "SUPPORT",
            "tasks": ["知识检索支撑"]
        }
    ],
    "inputContext": {
        "description": "为湖北电信电渠开发AI智能配案系统，实现用户画像、智能推荐、精准营销",
        "requirements": [
            "支持用户画像构建",
            "支持套餐智能匹配",
            "支持推荐效果跟踪"
        ],
        "constraints": [
            "需与现有CRM系统集成",
            "响应时间<500ms",
            "支持每日100万次调用"
        ],
        "referenceDocs": [
            "/docs/HBDX-AI-001-需求说明书.docx"
        ]
    },
    "outputConfig": {
        "format": "MARKDOWN",
        "template": "TELECOM_REQUIREMENT_V2",
        "language": "zh-CN"
    },
    "qualityStandard": "A",
    "deadline": "2026-03-20T18:00:00+08:00"
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | integer | 状态码 |
| message | string | 提示信息 |
| data.taskId | string | 任务ID |
| data.status | string | 任务状态 |
| data.estimatedDuration | integer | 预估耗时（秒） |
| data.participantStatus | array | 各参与者状态 |

## 3. 产物存储接口

### 3.1 接口概述

产物存储接口提供产物的持久化存储、检索、分类管理能力，支持电信行业的文档管理规范。

### 3.2 存储产物

#### 基本信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/v2/artifacts` |
| 功能描述 | 存储产物到产物库 |
| 权限要求 | 需要认证 |

#### 请求参数

**请求体 (Body)**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| taskId | string | 是 | 关联的任务ID |
| artifactType | string | 是 | 产物类型 |
| projectCode | string | 是 | 项目编码 |
| name | string | 是 | 产物名称 |
| description | string | 否 | 产物描述 |
| content | object | 是 | 产物内容 |
| format | string | 是 | 格式：MARKDOWN, DOCX, PDF, JSON |
| tags | array | 否 | 标签列表 |
| classification | object | 否 | 分类信息（电信行业分类） |
| retentionPolicy | string | 否 | 保留策略 |

**电信行业分类体系 (classification)**

```json
{
    "classification": {
        "domain": "TELECOM",           // 领域：电信
        "businessLine": "BSS",         // 业务线：BSS/OSS/MSS
        "category": "REQUIREMENT",     // 类别：需求/设计/开发/测试
        "sensitivity": "INTERNAL",     // 敏感级别：PUBLIC/INTERNAL/CONFIDENTIAL/SECRET
        "compliance": ["ISO27001", "电信行业安全规范"]
    }
}
```

#### 请求示例

```json
{
    "taskId": "task_2026031910080001",
    "artifactType": "REQUIREMENT_DOC",
    "projectCode": "HBDX-AI-001",
    "name": "湖北电信AI智能配案系统需求文档_V1.0",
    "description": "系统需求规格说明书，包含用户画像、智能推荐、精准营销三大模块",
    "content": {
        "raw": "# 需求规格说明书...",
        "structured": {
            "sections": [
                {"id": "sec_001", "title": "项目概述", "content": "..."},
                {"id": "sec_002", "title": "用户画像构建", "content": "..."}
            ]
        }
    },
    "format": "MARKDOWN",
    "tags": ["需求文档", "AI配案", "湖北电信"],
    "classification": {
        "domain": "TELECOM",
        "businessLine": "BSS",
        "category": "REQUIREMENT",
        "sensitivity": "INTERNAL"
    },
    "retentionPolicy": "PROJECT_LIFECYCLE"
}
```

#### 响应示例

```json
{
    "code": 201,
    "message": "产物存储成功",
    "data": {
        "artifactId": "art_2026031910300001",
        "taskId": "task_2026031910080001",
        "version": 1,
        "versionId": "ver_2026031910300001_v1",
        "storageLocation": "/artifacts/HBDX-AI-001/REQUIREMENT_DOC/",
```

## 4. 产物版本管理接口

### 4.1 接口概述

版本管理接口提供产物全版本生命周期管理，支持版本追溯、差异对比、版本回退等功能。

### 4.2 创建新版本

#### 基本信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/v2/artifacts/{artifactId}/versions` |
| 功能描述 | 基于现有产物创建新版本 |
| 权限要求 | 需要认证，需有编辑权限 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| baseVersion | integer | 是 | 基础版本号 |
| changeType | string | 是 | 变更类型：MAJOR, MINOR, PATCH |
| changeDescription | string | 是 | 变更说明 |
| content | object | 是 | 新版本内容 |
| reviewers | array | 否 | 评审人列表 |

#### 请求示例

```json
{
    "baseVersion": 1,
    "changeType": "MINOR",
    "changeDescription": "新增用户画像实时更新功能需求",
    "content": {
        "raw": "# 需求规格说明书 V1.1...",
        "changedSections": ["sec_002", "sec_003"]
    },
    "reviewers": ["nanqiao", "zhijin"]
}
```

#### 响应示例

```json
{
    "code": 201,
    "message": "版本创建成功",
```

## 5. 产物导出接口

### 5.1 接口概述

导出接口支持将产物转换为多种格式输出，包括文档格式（Word、PDF、HTML）、数据格式（JSON、XML、YAML）、电信行业专用格式等。

### 5.2 支持的导出格式

| 格式 | 说明 | 适用产物类型 |
|------|------|--------------|
| DOCX | Microsoft Word文档 | 全部 |
| PDF | PDF文档 | 全部 |
| HTML | 网页格式 | 全部 |
| MARKDOWN | Markdown格式 | 全部 |
| JSON | JSON数据格式 | 结构化数据 |
| XML | XML格式 | 接口定义、数据交换 |
| YAML | YAML格式 | 配置文件、API定义 |
| OPENAPI | OpenAPI 3.0规范 | 接口设计 |
| SQL | SQL脚本 | 数据库设计 |
| TMF_JSON | TM Forum JSON格式 | 电信行业产物 |

### 5.3 创建导出任务

#### 基本信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `POST /api/v2/exports` |
| 功能描述 | 创建产物导出任务 |
| 权限要求 | 需要认证 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| artifactIds | array | 是 | 产物ID列表 |

| version | integer | 否 | 指定版本，默认最新 |
