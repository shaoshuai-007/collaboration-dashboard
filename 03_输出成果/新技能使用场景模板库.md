## 一、🐙 github 使用模板

### 模板1：查看代码仓库状态

```
【场景】查看项目代码仓库状态

【命令模板】
gh repo list
gh issue list --repo [owner/repo]
gh pr list --repo [owner/repo]
gh run list --repo [owner/repo] --limit 5

【参数说明】
- owner/repo：仓库所有者/仓库名
- --limit：显示数量限制

【示例】
少帅：查看智能协作平台的代码仓库状态
南乔执行：
gh repo list | grep collaboration
gh issue list --repo openclaw/collaboration-dashboard
gh pr list --repo openclaw/collaboration-dashboard
gh run list --repo openclaw/collaboration-dashboard --limit 5
```

### 模板2：创建新仓库

```
【场景】为新项目创建GitHub仓库

【命令模板】
gh repo create [repo-name] --public --description "[描述]"
git remote add origin https://github.com/[owner]/[repo-name].git
git push -u origin main

【参数说明】
- repo-name：仓库名称
- --public/--private：公开/私有
- --description：仓库描述

【示例】
少帅：为九星监控系统创建GitHub仓库
南乔执行：
gh repo create nine-stars-monitor --public --description "九星智囊团监控系统"
cd /root/.openclaw/workspace/02_工作台/九星监控
git init
git add .
git commit -m "Initial commit: 九星监控系统"
git remote add origin https://github.com/openclaw/nine-stars-monitor.git
git push -u origin main
```

### 模板3：Issue管理

```
【场景】创建和追踪Issue

【命令模板】
gh issue create --repo [owner/repo] --title "[标题]" --body "[内容]"

gh issue view [issue-number] --repo [owner/repo]

【参数说明】
- --title：Issue标题
- --body：Issue内容
- --state：状态（open/closed）

【示例】

## 预期效果
效率提升10倍以上"
```

### 模板4：PR审查

```
【场景】审查Pull Request

【命令模板】
gh pr list --repo [owner/repo] --state open
gh pr view [pr-number] --repo [owner/repo]
gh pr diff [pr-number] --repo [owner/repo]
gh pr review [pr-number] --repo [owner/repo] --approve

【参数说明】
- --state：PR状态
- --approve：批准
- --request-changes：请求修改

【示例】
少帅：审查最近的PR

```

## 三、🧾 summarize 使用模板

### 模板1：情报快速处理

```
【场景】快速摘要情报内容

【命令模板】
summarize "[URL]" --length short --model google/gemini-3-flash-preview

【参数说明】
- --length：摘要长度（short/medium/long）
- --model：使用的模型

【示例】
少帅：摘要这篇电信行业新闻
南乔执行：
summarize "https://news.ctyun.cn/xxx" --length short --model google/gemini-3-flash-preview

输出：
- 核心要点
- 关键数据
- 趋势判断"
```

### 模板2：会议纪要生成

```
【场景】从录音到会议纪要

【命令模板】
summarize "[录音URL]" --youtube auto --extract-only
summarize "[转录文件]" --length medium

【示例】
少帅：生成这次会议的纪要
南乔执行：
# 步骤1：转录音频
summarize "https://meeting-recording.mp3" --youtube auto --extract-only > transcript.txt

# 步骤2：生成摘要
summarize "transcript.txt" --length medium

输出：
- 会议议题
- 关键决策
- 待办事项
- 责任人"
```

## 四、🔄 组合使用模板

### 模板1：需求到代码全流程

```
【场景】从需求文档到代码原型

【步骤】
1. summarize：摘要需求文档
2. coding-agent：生成代码原型
3. github：创建仓库并提交

【示例】
少帅：智能客服系统需求到原型

南乔执行：
# 步骤1：摘要需求
summarize "智能客服需求.docx" --length short > requirement_summary.md

# 步骤2：生成原型
pi "基于以下需求摘要，生成智能客服系统原型代码：

$(cat requirement_summary.md)

技术栈：Python + Flask
输出：完整的Web应用"

# 步骤3：创建仓库
gh repo create smart-customer-service --public
git init && git add . && git commit -m "Initial commit"
git push -u origin main"
```
