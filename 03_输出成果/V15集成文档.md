## 🎯 核心改进

### 改进前（V14）

| 问题 | 描述 |
|------|------|
| 意图识别简单 | 只识别任务类型，不识别项目、产出物、复杂度 |
| 单轮讨论 | 每个Agent发言一次就结束，没有真正互动 |
| 无会议纪要 | 只有简单的总结输出，没有完整纪要 |
| 无工作流 | 多产出物没有依赖排序，上下文无法传递 |

### 改进后（V15）

| 模块 | 功能 | 价值 |
|------|------|------|
| 南乔意图分析器 | 识别项目名称、产出物、复杂度、参与者 | 理解用户真正想要什么 |
| 多轮讨论执行器 | 支持多轮循环、质疑检测、共识判断 | Agent真正互动讨论 |
| 会议纪要生成器 | 生成完整纪要（8个章节） | 自动产出专业文档 |
| 多产出物工作流 | 依赖排序、上下文传递 | 保证文档一致性 |

## 📊 使用示例

### 示例1：单一产出物

**输入**：
```
做一个智能客服系统的需求分析
```

**流程**：
```
1. 意图分析 → 项目：智能客服，产出物：需求规格说明书
2. 拓扑排序 → 步骤1：需求规格说明书（无依赖）
3. 执行讨论 → 采薇、玉衡讨论1轮
4. 生成纪要 → 会议纪要.md
```

---

### 示例2：多产出物

**输入**：
```
智能客服系统的需求文档和架构设计
```

**流程**：
```
1. 意图分析 → 项目：智能客服，产出物：需求+架构
2. 拓扑排序 → 
   - 步骤1：需求规格说明书（无依赖）
   - 步骤2：系统架构设计文档（依赖：需求）
3. 执行讨论 → 
   - 步骤1：采薇、玉衡讨论 → 需求文档
   - 步骤2：织锦、工尺讨论（引用需求）→ 架构文档
4. 生成纪要 → 统一会议纪要.md
```

---

### 示例3：全流程

**输入**：
```
给出完整的可落地方案
```

**流程**：
```
1. 意图分析 → 产出物：需求+架构+详细设计+项目计划+成本估算
2. 拓扑排序 → 5个步骤按依赖排序
3. 分步执行 → 每步引用前序产出物
4. 生成纪要 → 包含5个产出物的完整纪要
```
