# 智能体协作平台 - 创新记录

## 2026-03-14 多意图识别与语义增强

### 一、问题背景

**原始问题**：
1. 多意图识别硬编码，只支持2个任务
2. 识别精准度低，口语化表达无法识别
3. 缺乏语义理解，只能匹配关键词

### 二、解决方案

#### 创新1：动态任务类型系统

**设计思路**：
- 从硬编码 → TASK_TYPES字典动态生成
- 支持扩展，新增任务只需添加配置

**代码架构**：
```python
TASK_TYPES = {
    'REQ-01': TaskType(
        code='REQ-01',
        name='需求收集',
        keywords=['需求收集', '调研', '访谈', ...],
        confidence_base=0.90
    ),
    # ... 共20个任务类型
}
```

**效果**：
- 支持20种任务类型
- 可任意组合，无数量限制
- 扩展性：新增任务只需添加配置项

---

#### 创新2：四层语义识别架构

**设计思路**：
- 分层递进，精准度递减，覆盖度递增
- 每层独立，可插拔

**架构图**：
```
用户输入
    ↓
第一层：精准关键词匹配（190个关键词，置信度90-98%）
    ↓ 未命中
第二层：同义词匹配（59个口语同义词，置信度85%）
    ↓ 未命中
第三层：语义规则分析（正则规则，置信度75-85%）
    ↓ 未命中
第四层：上下文推理（历史对话，置信度70%）
```

**关键代码**：
```python
def recognize(self, user_input: str, context: str = "") -> Tuple[str, float, str]:
    # 第一层：精准关键词
    task_code, confidence = self._keyword_match(user_input)
    if confidence >= 0.90:
        return task_code, confidence, "keyword_match"

    # 第二层：同义词
    task_code, confidence = self._synonym_match(user_input)
    if confidence >= 0.80:
        return task_code, confidence, "synonym_match"

    # 第三层：语义规则
    task_code, confidence = self._semantic_analysis(user_input)
    if confidence >= 0.75:
        return task_code, confidence, "semantic_analysis"

    # 第四层：上下文推理
    if context:
        task_code, confidence = self._context_inference(user_input, context)
        if confidence >= 0.70:
            return task_code, confidence, "context_inference"

    # 默认
    return 'REQ-02', 0.60, "default"
```

**效果**：
| 输入 | 识别结果 | 方法 |
|------|----------|------|
| "做一个报价单" | 成本估算 | 同义词匹配 |
| "帮我排期" | 项目计划 | 同义词匹配 |
| "多少钱" | 成本估算 | 同义词匹配 |

#### 创新3：反向关键词扩展

**设计思路**：
- 用户可能说"设计架构"，但关键词是"架构设计"
- 增加反向表达，提高覆盖率

**示例**：
```python
keywords=[
    '架构设计',    # 标准表达
    '设计架构',    # 反向表达
    '系统设计',    # 近义表达
    '设计系统',    # 反向近义
]
```

**效果**：
- 关键词从100个 → 190个
- 覆盖率提升约40%

### 四、可复用模式

**模式1：分层识别架构**
- 适用场景：多层级意图识别
- 关键点：每层独立，置信度递减

**模式2：同义词词典**
- 适用场景：口语化表达理解
- 关键点：持续积累，人工维护

**模式3：反向关键词扩展**
- 适用场景：提高关键词覆盖率
- 关键点：穷举用户可能的表达方式
