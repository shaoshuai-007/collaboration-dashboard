## 四、文件上传流程

### 4.1 上传流程图

```
步骤1: 生成数据文件(.DAT.gz)
    ↓
步骤2: 生成VAL校验文件
    ↓
步骤3: 生成CHECK清单文件
    ↓
步骤4: 上传数据文件(.TMP后缀)
    ↓
步骤5: 上传VAL文件(.TMP后缀)
    ↓
步骤6: 上传CHECK文件(.TMP后缀)
    ↓
步骤7: 去掉.TMP后缀（表示上传完成）
    ↓
步骤8: 等待回执文件
    ↓
步骤9: 下载回执文件(.RPT/.ERR)
```

### 4.2 上传顺序

**重要规则**：CHECK文件应在该省、该接口所有数据文件和校验文件上传完成后**最后上传**，表示该接口本批文件已生成。

## 五、文件命名规范

### 5.1 命名格式

```
{编码}_{数据来源}_{数据来源系统}_{接口编码}_{文件处理时间}_{文件数据时间}_{文件上传周期}_{文件上传批次}_{文件序列号}.{后缀}
```

### 5.2 字段详细说明

| 序号 | 字段名称 | 字节数 | 说明 | 示例 |
|:----:|----------|:------:|------|------|
| 1 | 编码 | 3-5 | 固定填"10800" | 10800 |
| 2 | 数据来源 | 2-10 | 单位编码（见主数据） | SHCHB |
| 3 | 数据来源系统 | 2-10 | 系统名称，4位英文大写 | AIPB |
| 4 | 接口编码 | 2-20 | AIP_MDL_TOKEN_INFO | AIP_MDL_TOKEN_INFO |
| 5 | 文件处理时间 | 12 | 生成日期YYYYMMDD | 20260318 |
| 6 | 文件数据时间 | 12 | 账期YYYYMMDD（实际Token消耗日期） | 20260317 |
| 7 | 文件上传周期 | - | 固定"D"（日） | D |
| 8 | 文件上传批次 | 2 | 首次"00"，重传依次+1 | 00 |
| 9 | 文件序列号 | 4 | 从0001开始 | 0001 |
| 10 | 后缀 | 4-8 | DAT.gz/VAL/CHECK/RPT/ERR/TMP | DAT.gz |

### 5.3 命名示例

**首次上传**：
```
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.DAT.gz
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.VAL
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.CHECK
```

**第一次重传**：
```
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_01_0001.DAT.gz
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_01_0001.VAL
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_01_0001.CHECK
```

### 5.4 TMP后缀规则

**重要**：所有文件在未上传完成时都应以".TMP"结尾，上传完成后再去掉".TMP"后缀。

```
上传中: 10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.DAT.gz.TMP
上传完成: 10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.DAT.gz
```

## 七、VAL校验文件格式

### 7.1 文件用途

VAL文件用于描述每一个传输周期内每个接口单元的每个数据文件需要校验的信息和数据处理接收方进行登记的信息。

### 7.2 字段详细说明

| 序号 | 信息内容 | 数据类型 | 说明 |
|:----:|----------|----------|------|
| 1 | 数据文件名称 | Varchar | 完整的.DAT.gz文件名 |
| 2 | 文件记录数 | Int | 数据文件包含的记录数（**压缩前记录数**） |
| 3 | 文件大小（字节数） | Int | 数据文件的物理存储大小，单位B（**压缩后的文件大小**） |
| 4 | MD5字符 | Varchar | 上传数据文件内容的MD5校验码（**压缩后文件的MD5**） |
| 5 | 数据日期 | Varchar | 抽取周期为日，格式为YYYYMMDD |
| 6 | 上传周期 | Varchar | 日：D |
| 7 | 接口数据文件的生成时间 | Char(14) | YYYYMMDDHH24MISS |

### 7.3 格式规范

- 字段分隔符：0x05
- 记录分隔符：0x0D0A（回车换行）
- 编码：UTF-8

### 7.4 示例

```
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.DAT.gz[0x05]1000[0x05]51200[0x05]a1b2c3d4e5f6...[0x05]20260317[0x05]D[0x05]20260318080000[0x0D0A]
```

## 八、CHECK文件格式

### 8.1 文件用途

CHECK文件用于描述该接口在本次操作的动作内所传送的文件列表。

### 8.2 字段详细说明

| 序号 | 信息内容 | 数据类型 | 说明 |
|:----:|----------|----------|------|
| 1 | 待核查的接口数据文件名称 | Varchar | 数据文件或VAL文件的完整名称 |
| 2 | 上传周期 | Varchar | 日：D |

### 8.3 格式规范

- 文件命名：没有文件序列号概念，默认"0001"
- 字段分隔符：0x05
- 记录分隔符：0x0D0A

### 8.4 示例

```
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.DAT.gz[0x05]D[0x0D0A]
10800_SHCHB_AIPB_AIP_MDL_TOKEN_INFO_20260318_20260317_D_00_0001.VAL[0x05]D[0x0D0A]
```

## 九、文件补传/重传规则

### 9.1 时间窗定义

| 时间窗 | 说明 |
|--------|------|
| 正常上传时间窗 | 要求源数据提供方接口文件准备完毕的时间点 |
| 迟传时间窗 | 无法在正常时间窗内准备完毕，需提前2小时申请 |
| 重传时间窗 | 迟传时间窗外重传，需提出申请 |

### 9.2 重传申请流程

| 场景 | 申请要求 |
|------|----------|
| 迟传 | 提前2小时提出申请，经数据中心运维人员批准 |
| 重传 | 准备好数据文件后提出申请，经数据中心运维人员批准 |

### 9.3 重传文件命名规则

**准实时文件重传**（如适用）：
- 文件缺失异常：对应小时数据，重传号从01开始
- 数据质量异常：对应小时数据，重传号做+1处理

**批量数据重传**：
- 文件缺失异常：生成缺失账期数据，重传号从01开始
- 数据质量异常：生成缺失账期数据，重传号做+1处理

### 9.4 数据封存时间点

**规则**：经过数据中心运维人员批准后，数据接收方将封存上月数据，不再接受迟传/重传数据。

## 十、回执文件格式

### 10.1 CHECK回执文件

**成功回执**：后缀为.RPT
**失败回执**：后缀为.ERR

**字段说明**：

| 序号 | 信息内容 | 数据类型 | 说明 |
|:----:|----------|----------|------|
| 1 | 待核查的接口数据文件名称 | Varchar | 校验结果代码为"10099"时，该字段为CHECK文件名称 |
| 2 | 核查结果代码 | Varchar(5) | 见下表错误码 |
| 3 | 0x0D0A | - | 回车换行符 |

**CHECK回执错误码**：

| 代码 | 说明 |
|:----:|------|
| 00 | 核查成功 |
| 10001 | CHECK文件无法打开 |
| 10002 | 接口数据文件不存在 |
| 10003 | 数据文件名异常 |
| 10004 | CHECK文件列表中无此数据文件 |
| 10005 | CHECK文件上传超时 |
| 10099 | 数据接口在规定时间内无CHECK文件 |

### 10.2 VAL回执文件

**成功回执**：后缀为.RPT
**失败回执**：后缀为.ERR

**字段说明**：

| 序号 | 文件信息内容 | 数据类型 | 说明 |
|:----:|-------------|----------|------|
| 1 | 接口数据文件名称 | Varchar | - |
| 2 | 处理时间 | Char(14) | YYYYMMDDHH24MISS |
| 3 | 校验结果代码 | Varchar(5) | 见下表错误码 |
| 4 | 0x0D0A | - | 回车换行符 |

**VAL回执错误码**：

| 代码 | 说明 |
|:----:|------|
| 00 | 校验成功 |
| 20001 | 数据文件无法打开 |
| 20002 | 文件记录中字段个数异常 |
| 20003 | 文件记录中字段顺序异常 |
| 20004 | 数据为空 |
| 20005 | 文件大小不符 |
| 20006 | 文件记录数不符 |
| 20007 | 文件MD5校验码不符 |
| 20008 | VAL文件不存在 |
| 20009 | 数据文件不存在 |
