**文档版本**：V3.0 综合深度版
**生成时间**：2026年3月18日
**分析团队**：九星智囊团（采薇+织锦+工尺+玉衡+南乔）

---

## 一、项目背景

### 1.1 业务背景

随着AI技术的快速发展，中国电信集团各单位已部署大量AI应用和大模型，包括：
- **总部部门**：23个部门（市场部、云网运营部、审计部等）
- **专业公司**：26家公司（天翼云、中电信人工智能、天翼安全等）
- **省公司**：31个省公司（湖北、广东、江苏等）
- **直属机构**：6个机构（研究院、人工智能研究院等）

### 1.2 问题现状

| 问题类别 | 具体问题 | 影响 |
|----------|----------|------|
| 数据分散 | Token消耗数据分散在各系统，缺乏统一采集 | 无法全局掌控 |
| 成本失控 | 无法实现全集团AI应用效果的统一评估和成本管控 | 成本难以控制 |
| 效果评估 | 缺乏统一的评估指标和数据支撑 | 效果难以衡量 |
| 决策支撑 | 无法为AI战略决策提供数据支撑 | 决策缺乏依据 |

### 1.3 项目目标

**核心目标**：构建集团级Token采集系统，实现全集团AI应用Token消耗数据的统一采集、统一存储、统一分析、统一展示。

## 三、核心能力需求

### 3.1 数据采集能力

#### 3.1.1 SFTP文件接收服务

| 能力项 | 详细说明 |
|--------|----------|
| **服务部署** | 部署SFTP服务器，监听端口12222 |
| **账号管理** | 为56个单位分配独立账号密码 |
| **目录管理** | 创建/data（上传目录）、/rpt（回执目录） |
| **权限控制** | /data写权限、/rpt读权限 |
| **文件监控** | 实时监控文件上传状态 |
| **网络配置** | CN2-1124网络映射，IP: 10.141.208.176 |

**关键约束**：
- 上传时间窗：每天上午8点前完成
- 文件命名：严格遵循10段命名规范
- 文件格式：UTF-8编码，0x05分隔符

#### 3.1.2 文件解析能力

| 能力项 | 详细说明 |
|--------|----------|
| **gzip解压** | 支持.DAT.gz文件解压 |
| **编码识别** | UTF-8编码自动识别 |
| **分隔符解析** | 0x05字段分隔、0x0D0A记录分隔 |
| **字段提取** | 21个字段的自动提取和类型转换 |
| **数据校验** | 主键唯一性、字段完整性、数据有效性 |

**关键约束**：
- 单文件压缩前不超过1GB（允许10MB浮动）
- 拆分文件按行拆分，同一行必须在同一文件
- 最后文件可小于1GB

#### 3.1.3 文件校验能力

| 校验类型 | 校验内容 | 错误码 |
|----------|----------|:------:|
| CHECK文件校验 | 文件列表完整性 | 10001-10099 |
| VAL文件校验 | 记录数、文件大小、MD5 | 20001-20009 |
| 数据文件校验 | 字段个数、字段顺序、数据为空 | 20002-20004 |

**CHECK校验错误码**：

| 错误码 | 说明 | 处理建议 |
|:------:|------|----------|
| 10001 | CHECK文件无法打开 | 检查文件格式和权限 |
| 10002 | 接口数据文件不存在 | 检查文件是否上传 |
| 10003 | 数据文件名异常 | 检查命名规范 |
| 10004 | CHECK文件列表中无此数据文件 | 检查CHECK文件内容 |
| 10005 | CHECK文件上传超时 | 检查网络连接 |
| 10099 | 规定时间内无CHECK文件 | 检查上传流程 |

**VAL校验错误码**：

| 错误码 | 说明 | 处理建议 |
|:------:|------|----------|
| 20001 | 数据文件无法打开 | 检查文件格式和权限 |
| 20002 | 字段个数异常 | 检查数据格式 |
| 20003 | 字段顺序异常 | 检查字段顺序 |
| 20004 | 数据为空 | 检查数据生成逻辑 |
| 20005 | 文件大小不符 | 检查压缩方式 |
| 20006 | 文件记录数不符 | 检查记录数统计 |
| 20007 | MD5校验码不符 | 重新生成VAL文件 |
| 20008 | VAL文件不存在 | 检查上传流程 |
| 20009 | 数据文件不存在 | 检查文件是否上传 |

#### 3.1.4 回执生成能力

| 能力项 | 详细说明 |
|--------|----------|
| **成功回执** | 生成.RPT文件，包含处理时间 |
| **失败回执** | 生成.ERR文件，包含错误码 |
| **回执格式** | 文件名+处理时间+错误码+换行符 |
| **回执推送** | 自动推送至/rpt目录 |

#### 3.2.1 数据清洗能力

**编码规则校验**：

```
模型编码规则：
格式：模型部署单位标识_模型品牌_模型参数量_序号
示例：YWYYB_QWEN_72_000001
约束：全部大写字母和数字，序号6位

应用编码规则：
格式：AI应用业务归属单位标识_AI应用图谱编码_序号
示例：YWYYB_ZNYY0001_000001
约束：AI应用图谱编码需对应三级分类
```
