# 采薇技能 - 输入输出质量保障方案

## 一、输入约束（必填信息）

### 1.1 核心必填信息（5项）

| 序号 | 信息项 | 说明 | 提问话术 |
|------|--------|------|---------|
| 1 | 项目名称 | 需求所属项目 | "请问这个需求是哪个项目的？" |
| 2 | 需求背景 | 为什么有这个需求 | "请问这个需求的背景是什么？要解决什么问题？" |
| 3 | 核心功能 | 主要要做什么 | "请问核心功能需求有哪些？系统要实现什么？" |
| 4 | 目标用户 | 谁来使用 | "请问这个系统是给谁用的？目标用户是谁？" |
| 5 | 业务目标 | 要达成什么效果 | "请问这个需求的业务目标是什么？期望达成什么效果？" |

### 1.2 次要信息（可选但有更好）

- 性能要求（响应时间、并发量等）
- 安全要求（数据安全、权限控制等）
- 集成要求（对接哪些系统）
- 时间约束（上线时间、里程碑）
- 预算约束

### 1.3 输入检查机制

```python
class InputValidator:
    """输入验证器"""

    REQUIRED_FIELDS = {
        'project_name': '项目名称',
        'background': '需求背景',
        'core_functions': '核心功能',
        'target_users': '目标用户',
        'business_goals': '业务目标'
    }

    @staticmethod
    def validate(requirements_text):
        """验证输入是否包含必要信息"""
        missing = []

        for field, name in InputValidator.REQUIRED_FIELDS.items():
            if not InputValidator._check_field(requirements_text, field):
                missing.append(name)

        return {
            'valid': len(missing) == 0,
            'missing_fields': missing,
            'questions': InputValidator._generate_questions(missing)
        }

    @staticmethod
    def _check_field(text, field):
        """检查文本是否包含该字段相关信息"""
        # 根据不同字段设计检查逻辑
        patterns = {
            'project_name': [r'项目', r'系统', r'平台'],
            'background': [r'背景', r'现状', r'问题', r'痛点'],
            'core_functions': [r'功能', r'需求', r'实现', r'支持'],
            'target_users': [r'用户', r'客户', r'使用'],
            'business_goals': [r'目标', r'效果', r'提升', r'降低', r'增加']
        }

        import re
        for pattern in patterns.get(field, []):
            if re.search(pattern, text):
                return True
        return False

    @staticmethod
    def _generate_questions(missing_fields):
        """生成提问话术"""
        questions_map = {
            '项目名称': '请问这个需求是哪个项目的？',
            '需求背景': '请问这个需求的背景是什么？要解决什么问题？',
            '核心功能': '请问核心功能需求有哪些？系统要实现什么？',
            '目标用户': '请问这个系统是给谁用的？目标用户是谁？',
            '业务目标': '请问这个需求的业务目标是什么？期望达成什么效果？'
        }

        return [questions_map.get(f, f'请提供{f}信息') for f in missing_fields]
```

## 二、输出约束（必含元素）

### 2.1 文档必含章节（8项）

| 序号 | 章节 | 必含元素 | 检查点 |
|------|------|---------|--------|
| 1 | 需求背景 | 项目背景、业务现状、问题分析 | ✅ 三项齐全 |
| 2 | 需求描述 | 功能需求列表、非功能需求 | ✅ 至少3条功能需求 |
| 3 | 需求目标 | 业务目标、技术目标、量化指标 | ✅ 目标可量化 |
| 4 | 多维度分析 | 8个维度分析结果 | ✅ 全部维度覆盖 |
| 5 | 验收标准 | 功能验收、性能验收、业务验收 | ✅ 标准可测试 |
| 6 | 风险识别 | 风险点、应对建议 | ✅ 至少识别1个风险 |
| 7 | 优先级划分 | MoSCoW分类 | ✅ 明确优先级 |
| 8 | 质量报告 | 质量评分、改进建议 | ✅ 评分≥80分 |

## 三、完整工作流程

### 3.1 输入检查流程

```
开始
  ↓
检查输入信息完整性
  ↓
是否包含5项核心信息？
  ├── 是 → 进入生成流程
  └── 否 → 生成提问清单
           ↓
         返回给用户
           ↓
         用户补充信息
           ↓
         重新检查
```

### 3.2 输出检查流程

```
生成文档
  ↓
运行输出验证器
  ↓
检查8个必含章节
  ↓
质量评分是否≥80分？
  ├── 是 → 输出文档
  └── 否 → 识别缺失项
           ↓
         重新生成缺失部分
           ↓
         再次验证
           ↓
         最多重试3次
```

## 五、质量评分标准

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 完整性 | 30% | 8个章节全部包含 |
| 正确性 | 20% | 内容正确无误 |
| 可行性 | 20% | 技术方案可行 |
| 优先级 | 15% | MoSCoW分类明确 |
| 可追溯 | 15% | 需求有编号、有来源 |

**等级标准**：
- A级：90-100分（优秀）
- B级：80-89分（良好）
- C级：70-79分（合格）
- D级：<70分（不合格，需重新生成）
