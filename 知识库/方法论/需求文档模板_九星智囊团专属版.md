### 第1章 简介

#### 1.1 建设背景

**写作要求**：一段话描述业务背景，涵盖客户原始业务要求、集团派发单要求等信息

**模板**：
```
本项目源于【客户名称】【部门】的【业务需求】。
【集团/省分】于【时间】下发《【文件名称】》，要求【具体要求】。
当前存在【痛点问题】，亟需通过【解决方案】实现【目标】。
```

**示例**：
```
本项目源于湖北电信营销运营部的智能配餐需求。
集团于2023年下发《中国电信〔2023〕194号工作指引》，要求各省建设AI智能配案能力。
当前武汉3人半年完成场景配餐，分群门槛高、标签选择难，亟需通过AI智能分群系统实现降本增效。
```

#### 1.2 建设目的

**写作要求**：
1. 解决问题：简单描述需求要解决什么问题
2. 效果预估：具像化可衡量的目标

**模板**：
```
【解决问题】
1. 问题一：xxx
2. 问题二：xxx

【效果预估】
1. 指标一：提升xxx%，从x提升到y
2. 指标二：降低xxx%，从x降低到y
3. 效率提升：xxx倍
```

**示例**：
```
【解决问题】
1. 分群门槛高：武汉3人半年完成场景配餐，需懂政策+标签分群能力
2. 标签选择难：6000+标签，多而杂，同质化严重

【效果预估】
1. 分群效率：提升5倍，从半年缩短到1个月
2. 标签推荐准确率：≥80%
3. 客户群有效率：从4.1%提升到15%+
```

### 第2章 业务需求

#### 2.1 业务场景

**写作要求**：从业务角度出发，要实现的业务/用户场景（非系统功能）

**方法**：使用**5W1H法**分析
- Who（谁）
- When（什么时间）
- Where（什么地点）
- What（什么事情）
- Why（为什么）
- How（如何达成）

**模板**：
```
【场景名称】
- Who: 【角色】
- When: 【触发时间】
- Where: 【地点/渠道】
- What: 【做什么事情】
- Why: 【为什么做】
- How: 【如何达成】
```

**示例**：
```
【场景1：自然语言分群】
- Who: 营销策划人员
- When: 需要创建客户分群时
- Where: AI智能配案系统
- What: 用自然语言描述目标客户特征
- Why: 降低分群门槛，无需掌握标签编码
- How: 输入"高流量5G用户"，系统自动生成分群条件

【场景2：一键同步CPCP】
- Who: 营销策划人员
- When: 分群条件确认后
- Where: AI智能配案系统
- What: 一键同步分群到CPCP系统
- Why: 实现快速触达客户
- How: 点击"同步CPCP"，自动生成人群ID
```

#### 2.2 业务流程

**写作要求**：从业务角度出发，描述要实现的业务场景流程

**侧重点**：不同角色的操作场景，与业务方确认流程

**模板**：
```
【流程图】
（插入流程图）

【流程描述】
1. 【角色A】执行【步骤1】
2. 【角色B】执行【步骤2】
3. ...
```

#### 2.3 现状分析（改造需求必选）

**写作要求**：
1. 业务现状：目前的界面、业务流程、逻辑等
2. 目前问题：可用对比图呈现"现状vs目标"

**模板**：
```
【业务现状】
- 现状描述：xxx
- 当前流程：xxx

【存在问题】
| 问题 | 影响 | 严重程度 |
|-----|------|:-------:|
| 问题1 | 影响1 | 高/中/低 |
| 问题2 | 影响2 | 高/中/低 |

【改进目标】
- 目标描述：xxx
- 目标流程：xxx
```

## 🎯 模板使用说明

### 写作原则

1. **完整性**：每个章节都要填写，无内容可标注"不适用"
2. **正确性**：内容正确，无错误
3. **可行性**：需求可实现
4. **可追溯性**：需求有编号，可追溯

### 质量标准

| 维度 | 标准 | 分值 |
|------|------|:----:|
| 完整性 | 背景完整、需求完整、验收标准完整 | 30分 |
| 正确性 | 业务正确、技术正确 | 20分 |
| 可行性 | 技术可行、资源可行、业务可行 | 20分 |
| 优先级 | MoSCoW分类明确、依赖关系清晰 | 15分 |
| 可追溯性 | 需求有编号、变更可追溯 | 15分 |

**质量等级**：
- A级：90-100分
- B级：80-89分
- C级：70-79分
- D级：<70分
