## 一、命名规范

### 1.1 表命名

- 使用 **小写字母** 和 **下划线**
- 使用 **单数形式**（user 而非 users）
- 使用 **业务前缀** 区分模块

```sql
-- ✅ 正确
t_user           -- 用户表
t_user_profile   -- 用户画像表
t_order          -- 订单表
t_order_item     -- 订单明细表

-- ❌ 错误
Users            -- 不要大写
tblUser          -- 不要驼峰
user_infos       -- 不要复数
```

### 1.2 字段命名

- 使用 **小写字母** 和 **下划线**
- 主键统一使用 `id`
- 外键使用 `表名_id`

```sql
-- ✅ 正确
CREATE TABLE t_user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_name       VARCHAR(50) NOT NULL,
    phone_number    VARCHAR(20),
    package_id      BIGINT,          -- 外键
    create_time     DATETIME,
    update_time     DATETIME
);

-- ❌ 错误
CREATE TABLE t_user (
    userID          BIGINT PRIMARY KEY,  -- 不要驼峰
    UserName        VARCHAR(50),          -- 不要大写
    pkg_id          BIGINT               -- 缩写不清晰
);
```

## 二、主键设计

### 2.1 主键类型

| 类型 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 自增ID | 单机、小规模 | 简单、高效 | 分布式不友好 |
| UUID | 分布式系统 | 全局唯一 | 占用空间大、索引效率低 |
| 雪花ID | 分布式系统 | 全局唯一、有序 | 需要维护机器ID |

### 2.2 推荐方案

```sql
-- 单机环境：自增ID
CREATE TABLE t_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    ...
);

-- 分布式环境：雪花ID
CREATE TABLE t_user (
    id BIGINT PRIMARY KEY COMMENT '雪花ID',
    ...
);
```

## 三、字段设计

### 3.1 数据类型选择

| 数据类型 | 适用场景 | 示例 |
|----------|----------|------|
| TINYINT | 状态、标志 | gender, status |
| SMALLINT | 小范围数值 | age, count |
| INT | 普通数值 | quantity, score |
| BIGINT | 大数值、ID | id, user_id |
| DECIMAL | 金额 | price, amount |
| VARCHAR | 变长字符串 | name, phone |
| TEXT | 长文本 | description |
| DATETIME | 日期时间 | create_time |
| TIMESTAMP | 时间戳 | update_time |

### 3.2 字段约束

```sql
CREATE TABLE t_user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    user_name       VARCHAR(50) NOT NULL COMMENT '用户名',
    phone           VARCHAR(20) NOT NULL COMMENT '手机号',
    email           VARCHAR(100) COMMENT '邮箱',
    gender          TINYINT DEFAULT 0 COMMENT '性别：0未知 1男 2女',
    status          TINYINT DEFAULT 1 COMMENT '状态：0禁用 1启用',
    value_score     INT DEFAULT 0 COMMENT '价值分数',
    create_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted         TINYINT DEFAULT 0 COMMENT '逻辑删除：0未删除 1已删除',

    UNIQUE KEY uk_phone (phone),
    KEY idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 四、索引设计

### 4.1 索引类型

| 索引类型 | 说明 | 适用场景 |
|----------|------|----------|
| 主键索引 | 唯一标识记录 | 主键 |
| 唯一索引 | 字段值唯一 | 手机号、邮箱 |
| 普通索引 | 加速查询 | 常用查询字段 |
| 组合索引 | 多字段组合 | 多条件查询 |
| 全文索引 | 文本搜索 | 文章内容 |

### 4.2 索引原则

1. **最左前缀原则**：组合索引从左到右匹配

```sql
-- 组合索引
KEY idx_user_package_status (user_id, package_id, status)

-- ✅ 能用到索引
WHERE user_id = 1001
WHERE user_id = 1001 AND package_id = 5
WHERE user_id = 1001 AND package_id = 5 AND status = 1

-- ❌ 用不到索引
WHERE package_id = 5
WHERE status = 1
```

2. **避免过度索引**：索引不是越多越好

```sql
-- ✅ 正确：选择性高的字段建索引
KEY idx_phone (phone)
KEY idx_user_id (user_id)

-- ❌ 错误：选择性低的字段不建索引
-- gender只有3个值，不适合建索引
```

3. **覆盖索引**：查询字段都在索引中

```sql
-- 组合索引
KEY idx_user_package (user_id, package_id)

-- ✅ 使用覆盖索引，不需要回表
SELECT user_id, package_id FROM t_order WHERE user_id = 1001
```

## 五、表设计规范

### 5.1 必备字段

每张表都应该包含以下字段：

```sql
CREATE TABLE t_example (
    id              BIGINT PRIMARY KEY COMMENT '主键ID',
    -- 业务字段...

    create_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    create_by       VARCHAR(50) COMMENT '创建人',
    update_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    update_by       VARCHAR(50) COMMENT '更新人',
    deleted         TINYINT DEFAULT 0 COMMENT '逻辑删除：0未删除 1已删除',
    version         INT DEFAULT 0 COMMENT '乐观锁版本号'
);
```

### 5.2 字段注释

- 所有表和字段必须有注释
- 枚举字段必须说明枚举值含义

```sql
-- ✅ 正确
CREATE TABLE t_order (
    id              BIGINT PRIMARY KEY COMMENT '订单ID',
    order_no        VARCHAR(32) NOT NULL COMMENT '订单编号',
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    order_type      TINYINT NOT NULL COMMENT '订单类型：1新开 2续费 3变更',
    status          TINYINT NOT NULL COMMENT '状态：0待支付 1已支付 2已取消 3已退款',
    total_amount    DECIMAL(10,2) NOT NULL COMMENT '订单金额（元）',

    UNIQUE KEY uk_order_no (order_no),
    KEY idx_user_id (user_id)
) COMMENT='订单表';
```

## 六、SQL编写规范

### 6.1 SELECT规范

```sql
-- ✅ 正确：指定字段
SELECT id, user_name, phone FROM t_user WHERE id = 1001;

-- ❌ 错误：使用 *
SELECT * FROM t_user;
```

### 6.2 WHERE规范

```sql
-- ✅ 正确
WHERE create_time >= '2024-01-01'
  AND status = 1
  AND deleted = 0

-- ❌ 错误：索引列使用函数
WHERE DATE(create_time) = '2024-01-01'

-- ❌ 错误：索引列使用计算
WHERE id + 1 = 1002

-- ❌ 错误：隐式类型转换
WHERE phone = 13800138000  -- phone是VARCHAR，应该用字符串
```

### 6.3 JOIN规范

```sql
-- ✅ 正确：小表驱动大表
SELECT u.user_name, o.order_no
FROM t_user u
INNER JOIN t_order o ON u.id = o.user_id
WHERE u.id = 1001;

-- ✅ 正确：指定连接类型
FROM t_user u
LEFT JOIN t_order o ON u.id = o.user_id

-- ❌ 错误：避免子查询
SELECT * FROM t_user WHERE id IN (SELECT user_id FROM t_order);
```

## 七、分库分表

### 7.1 分表策略

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| 按时间分表 | 日志、流水 | t_order_202401, t_order_202402 |
| 按ID取模 | 用户、订单 | t_order_0, t_order_1, ... t_order_15 |
| 按地域分表 | 地域相关 | t_user_beijing, t_user_shanghai |

### 7.2 分表数量

- 取模分表：通常 **2的N次方**（如16、32、64）
- 时间分表：按月或按季度

```sql
-- 按user_id取模16分表
CREATE TABLE t_order_0 ( ... );
CREATE TABLE t_order_1 ( ... );
...
CREATE TABLE t_order_15 ( ... );

-- 路由规则
table_index = user_id % 16
```

## 八、性能优化

### 8.1 查询优化

```sql
-- ✅ 使用EXPLAIN分析
EXPLAIN SELECT * FROM t_user WHERE phone = '13800138000';

-- ✅ 使用LIMIT
SELECT * FROM t_order WHERE user_id = 1001 LIMIT 100;

-- ✅ 批量插入
INSERT INTO t_user (user_name, phone) VALUES
    ('张三', '13800138001'),
    ('李四', '13800138002'),
    ('王五', '13800138003');
```

### 8.2 避免慢查询

```sql
-- ❌ 避免 LIKE 前缀匹配
WHERE name LIKE '%张%'  -- 全表扫描

-- ✅ 使用后缀匹配（可用索引）
WHERE name LIKE '张%'

-- ❌ 避免 OR
WHERE status = 1 OR status = 2

-- ✅ 使用 IN
WHERE status IN (1, 2)
```

## 九、安全规范

### 9.1 防止SQL注入

```python
# ✅ 正确：使用参数化查询
cursor.execute("SELECT * FROM t_user WHERE phone = %s", (phone,))

# ❌ 错误：字符串拼接
cursor.execute(f"SELECT * FROM t_user WHERE phone = '{phone}'")
```

### 9.2 敏感数据加密

```sql
-- ✅ 加密存储
INSERT INTO t_user (phone, password) VALUES (
    AES_ENCRYPT('13800138000', 'secret_key'),
    MD5('password123')
);

-- ✅ 解密查询
SELECT AES_DECRYPT(phone, 'secret_key') FROM t_user;
```
