## 一、建设背景

### 1.1 集团文件要求

《市场空间洞察工作指引1.0》提出：
- 坚持以客户为中心，精准洞察客户需求
- 构建市场空间洞察能力

### 1.2 集团系统构建要求

| 场景 | 要求 |
|------|------|
| 公众网格场景 | 客户、产品、渠道、物理空间和资源入库；本网客户多维度洞察 |
| 存量经营场景 | 业务使用、消费行为和终端使用特征入库；融合空间洞察，客户质态跟踪 |

### 1.3 现状问题

| 问题 | 现状 |
|------|------|
| **客户群冗余** | 共29690个客户群，有效仅1218个，占比**4.1%** |
| **人工取数周期长** | 效率低 |
| **缺乏可视化展示** | 指标复用率低，沟通成本高 |
| **缺乏统一管理** | 无法高效集约赋能各级触点/平台 |

## 二、建设目标

### 2.1 总体目标

```
统一数据底座 + 统一管理客户群 + 统一对外开放
= 一站式客户群洞察分析平台
```

### 2.2 两大能力目标

| 目标 | 内容 |
|------|------|
| **统一客户群管理能力** | 建立客户群原子能力，高效集约化赋能其他系统 |
| **可视化洞察分析能力** | 基于大数据标签全方位、快速刻画客户分群特征 |

### 2.3 六大业务举措

1. 客户群共建共享
2. 客户群拍照
3. 客户群对外能力开放
4. 客户群洞察维度指标管理
5. 客户群洞察维度指标开发
6. 客户群可视化多维洞察分析

---

## 三、核心能力

### 3.1 客户群统一管理

| 能力 | 说明 |
|------|------|
| **3类客群纳管** | 用户级、客户级、异网级 |
| **业务分类清晰** | 提供统一的业务分类口径 |
| **分群维度丰富** | 按分群目标、分群用途打标 |
| **群生命周期管理** | 新建、编辑、下线全流程管理 |

### 3.2 群分布可视化呈现

| 功能 | 说明 |
|------|------|
| **群规模趋势** | 15天、30天、60天变化趋势 |
| **地域分布** | 省-市-区县层级下钻 |
| **网格分布** | 四级网格和五级网格分布可视化 |

### 3.3 群空间可视化洞察

**多维度空间洞察**：分析分群用户在其他业务上的渗透率和剩余空间

**支持业务维度**：
- 智能音响、云盘黄金会员、云回看、千兆
- 小合约、全屋Wi-Fi、橙分期、FTTR、5G

### 3.4 群特征可视化分析

| 特征类型 | 内容 |
|----------|------|
| **业务特征** | 套餐分布、移动终端分布、套餐流量分布、网龄分布 |
| **画像特征** | 年龄分布、城乡分布、性别、白领用户占比、K12家长占比、婴幼儿家长占比、有车一族占比 |
| **极值计算** | 自动计算最大值、最小值、中位数 |

### 3.5 可视化群洞察一点配置

```
流程：
新建/选择分群 → 选择洞察指标 → 一键生成可视化洞察
```

### 3.6 标杆客户群示范

| 分类方式 | 内容 |
|----------|------|
| **业务分群** | 融连接、融终端、融应用 |
| **生命周期分群** | 潜客期、入网期、共生期、异动期、离网期 |
