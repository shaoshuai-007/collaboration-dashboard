## 一、需求背景

### 1.1 业务现状

| 项目 | 现状 |
|------|------|
| **人员配置** | 武汉3人负责，半年完成场景配餐 |
| **能力要求** | 需熟悉政策 + 懂标签分群 |
| **标签体系** | 6000+标签，同质化严重 |
| **响应效率** | 市场部每次需找武汉帮忙分析 |

### 1.2 核心痛点

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| **分群门槛高** | 需将业务特征转化为标签组合 | 依赖专家，难以复制 |
| **标签选择难** | 6000+标签，命名晦涩（如`bas_dcol1499`） | 运营人员"大海捞针" |
| **同质化严重** | 多个标签含义相似 | 选择困难，容易选错 |
| **响应效率低** | 市场部需找武汉帮忙 | 全省无法自主运营 |
| **场景维护难** | 专班结束后无人维护 | 知识流失 |

## 三、全省画像标签体系分析

### 3.1 用户特征画像定义（7大维度）

| 画像维度 | 定义 | 输出值 | 核心标签 |
|----------|------|:------:|----------|
| **流量需求度** | 评估用户对移动流量的潜在需求程度 | 高/中/低 | 流量饱和度、使用量、溢出费用 |
| **语音需求度** | 评估用户对语音通话的潜在需求 | 高/中/低 | 语音饱和度、通话时长、溢出费用 |
| **合约接受度** | 评估用户对签订合约的接受程度 | 高/中/低 | 合约办理记录、转化率 |
| **溢出接受度** | 评估用户对套餐外付费的意愿 | 高/中/低 | 溢出费用、短期包订购 |
| **终端需求度** | 评估用户对更换/升级终端的需求 | 高/中/低 | 终端使用时长、5G支持 |
| **宽带依赖度** | 评估用户对宽带业务的依赖程度 | 高/中/低 | 宽带使用量、活跃度 |
| **视频偏好度** | 评估用户对视频内容的偏好 | 高/中/低 | ITV活跃、视频APP流量 |

### 4.1 策略字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 编号 | 策略编号 | 示例1 |

| 条件组合 | 标签组合 | 流量需求度:高 && 终端需求度:高 |
| 优先级 | 匹配优先级 | 1 |
| 策略层级 | 主推/次推/兜底 | 主推 |
| 策略内容 | 推荐产品 | 199全家福终端版 |
| 预期列收变化 | 收入影响 | +30元 |

| 适配触点 | 执行渠道 | 外呼/10000弹窗/厅店 |
| 话术要点 | 推销话术 | "您流量用得这么多..." |

### 4.2 条件组合格式

```
标签:取值 && 标签:取值 || 标签:取值

示例：
流量需求度:高 && 终端需求度:高
语音需求度:高 || 流量需求度:高
```
