## 一、电信业务体系概述

### 1.1 业务分类

```
电信业务体系
├── 基础通信业务
│   ├── 语音业务（打电话）
│   ├── 短信业务（发短信）
│   └── 数据业务（上网）
├── 增值业务
│   ├── 彩铃业务
│   ├── 来电显示
│   └── 通话秘书
├── 智能化业务
│   ├── AI智能配案
│   ├── 智能客服
│   └── 精准营销
└── 融合业务
    ├── 家庭融合（宽带+电视+手机）
    └── 企业融合（专线+云服务）
```

### 1.2 核心业务指标

| 指标类别 | 指标名称 | 说明 |
|----------|----------|------|
| 用户规模 | 用户数 | 在网用户总数 |
| | 新增用户 | 当月新增用户数 |
| | 流失用户 | 当月流失用户数 |
| 收入指标 | ARPU | 每用户平均收入 |
| | 总收入 | 月度/年度收入 |
| 使用量 | MOU | 每用户平均通话时长 |
| | DOU | 每用户平均流量使用 |
| 质量 | 接通率 | 通话接通成功率 |
| | 投诉率 | 用户投诉比例 |

### 5.2 核心数据表

```sql
-- 用户画像表
CREATE TABLE t_user_profile (
    id              BIGINT PRIMARY KEY COMMENT '主键ID',
    user_id         BIGINT NOT NULL COMMENT '用户ID',

    -- 基础特征
    age_range       VARCHAR(10) COMMENT '年龄段：18-25,26-35,36-45,46-55,55+',
    gender          TINYINT COMMENT '性别：1男 2女',
    region          VARCHAR(50) COMMENT '地域',
    vip_level       TINYINT COMMENT '会员等级：1-5',

    -- 使用特征
    avg_mou         DECIMAL(10,2) COMMENT '月均通话时长(分钟)',
    avg_dou         DECIMAL(10,2) COMMENT '月均流量使用(MB)',
    avg_sms_count   INT COMMENT '月均短信条数',
    peak_hour       VARCHAR(10) COMMENT '使用高峰时段',

    -- 消费特征
    arpu            DECIMAL(10,2) COMMENT 'ARPU值',
    recharge_freq   DECIMAL(10,2) COMMENT '充值频率(次/月)',
    price_sens      TINYINT COMMENT '价格敏感度：1低 2中 3高',

    -- 价值分数
    value_score     INT COMMENT '价值分数：0-100',

    create_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    UNIQUE KEY uk_user_id (user_id)
) COMMENT='用户画像表';

-- AI配案记录表
CREATE TABLE t_ai_recommend (
    id              BIGINT PRIMARY KEY COMMENT '主键ID',
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    recommend_type  TINYINT COMMENT '推荐类型：1套餐升级 2套餐变更 3增值业务',
    current_pkg_id  INT COMMENT '当前套餐ID',
    recommend_pkg_id INT COMMENT '推荐套餐ID',
    recommend_score DECIMAL(5,2) COMMENT '推荐分数',
    recommend_reason TEXT COMMENT '推荐理由',
    status          TINYINT DEFAULT 0 COMMENT '状态：0待处理 1已采纳 2已拒绝',
    create_time     DATETIME NOT NULL DEFAULT CURRENTETIME,

    KEY idx_user_id (user_id),
    KEY idx_create_time (create_time)
) COMMENT='AI配案记录表';
```

## 六、存量经营流程

### 6.1 用户生命周期管理

```
用户生命周期
│
├── 新用户期（0-3个月）
│   ├── 开户关怀
│   ├── 使用引导
│   └── 套餐推荐
│
├── 成长期（3-12个月）
│   ├── 使用习惯培养
│   ├── 增值业务推荐
│   └── 会员权益引导
│
├── 成熟期（12-24个月）
│   ├── 套餐优化
│   ├── 流量包推荐
│   └── 融合业务拓展
│
├── 衰退期（24个月+）
│   ├── 流失预警
│   ├── 挽留措施
│   └── 优惠关怀
│
└── 流失期
    ├── 流失原因分析
    ├── 回访挽留
    └── 归档记录
```

### 6.2 流失预警指标

| 指标 | 预警阈值 | 风险等级 |
|------|----------|:--------:|
| 话费余额 | <10元 | ⚠️ 中 |
| 最近充值时间 | >30天 | ⚠️ 中 |
| 通话次数下降 | >50% | 🔴 高 |
| 流量使用下降 | >50% | 🔴 高 |
| 投诉次数 | >3次/月 | 🔴 高 |
| 套餐到期 | <7天 | ⚠️ 中 |

## 七、业务系统架构

### 7.1 核心系统

```
电信业务系统架构
│
├── BSS（业务支撑系统）
│   ├── CRM系统 - 客户关系管理

│   └── 结算系统 - 网间结算
│
├── OSS（运营支撑系统）
│   ├── 网管系统 - 网络监控管理
│   ├── 资源系统 - 号码、SIM卡资源
│   └── 故障系统 - 故障处理
│
└── MSS（管理支撑系统）
    ├── ERP系统 - 企业资源管理
    ├── OA系统 - 办公自动化
    └── BI系统 - 商业智能分析
```
