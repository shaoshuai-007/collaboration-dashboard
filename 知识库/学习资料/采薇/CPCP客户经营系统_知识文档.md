## 二、系统架构

### 2.1 整体架构

```
评估中心
├── 事前预判：营销预评估、市场评估
├── 事中监控：工单运营分析、营销协同分析、全流程监控
└── 事后评估：业务发展分析、价值评估

数智驱动中心
├── 批流一体实时事件中心
├── 工单池
└── 统一智能调度中心

CPCP核心
├── 全息客户视图
├── 策略库
├── 运营协同
└── 权益营销资源

大数据湖
├── 存储：hdfs、hive、es、hbase、ozone、ck、Doris
├── 计算：spark、Flink-cdc、impala、Trion
└── 查询：汇总标签、底层标签
```

### 2.2 功能架构

```
场景支撑
├── 批量营销
├── 事件营销
└── 数智化事件驱动

事件中心
├── 原子事件、业务事件
├── 营销活动信息配置
└── 事件订阅触发

统一智能调度中心
├── 营销工单、服务工单
├── 规则引擎、群洞察、策略
└── 工单生成引擎、派单协同控制

评估中心
├── 整体监控、活动监控
├── 工单进度查询、渠道接触执行
└── 反馈接触结果、价值评估
```

## 四、2023年建设内容

### 4.1 一键系列

**一键诊断**：
- 3×3诊断要素（基本要素、套餐、连接、终端、应用、价值、活跃、时长、感知）
- 千人千面智能推荐
- 页面集成：全息客户视图、集中外呼、承包助手、厅店外呼、10000号客服、翼销售
- **诊断分析9956人次**

**一键点单**：
- 空间洞察、三步快速点单
- 营销闭环：跟踪、评估、管控
- **半月673次活动，派发7489笔工单，转化率11.2%**

**一键转派**：
- 跨触点协同灵活调度
- 支持承包助手、厅店外呼、智慧营维
- 变动实时通知

### 4.2 策略库建设

| 阶段 | 内容 |
|------|------|
| 第一阶段 | 关键词管理、场景目录管理、场景策略编排、策略对外开放 |
| 第二阶段 | 策略库与营销活动接入、画布能力、新UI样式 |
| 第三阶段 | 营销活动引用、诊断策略推荐 |

**建设成效**：
- 沉淀28个场景，31条策略
- 44个营销活动引用
- 一键诊断策略调用9956次，派单170913次

### 4.3 专项建设

**宽带预约拆机挽留**：
- 维挽成功率：30.7%
- 预约宽带拆机用户数：66967
- 维挽成功用户数：20559

**政企客户级派单**：
- 累计活动：157个
- 累计派单量：8万+
- 整体转化率：13%

**四级体系构建**：
- 省、市、区县、分支局四级分权分域管理
- 动态下放活动创建权限、分群创建权限

## 七、对AI+客户群洞察项目的启示

### 7.1 系统定位

- CPCP是**客户经营平台**，核心是**分群执行**
- AI+客户群洞察是**智能辅助系统**，核心是**分群决策**

### 7.2 能力互补

| CPCP能力 | AI+客户群洞察能力 |
|----------|-------------------|
| 标签取数 | 自然语言→标签推荐 |
| 目标分群配置 | 自然语言→分群规则生成 |
| 策略库 | 场景知识库+AI推荐 |
| 派单执行 | 智能触点推荐 |

### 7.3 AI增强方向

1. **分群门槛降低**
   - 从"3人半年"到"一人一天"
   - 自然语言描述→自动生成分群规则

2. **标签选择简化**
   - 从"6000+标签人工选择"到"AI智能推荐"
   - 场景描述→标签组合推荐

3. **场景知识沉淀**
   - 129个场景模板
   - SQL规则模板库
   - 一线话术库
