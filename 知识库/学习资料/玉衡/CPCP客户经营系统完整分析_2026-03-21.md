## 一、系统概述

### 1.1 全称与含义

**CPCP = Customer × Product × Channel × Promotion**

| 缩写 | 英文 | 含义 |
|:----:|------|------|
| **C** | Customer | 客户 |
| **P** | Product | 产品 |
| **C** | Channel | 渠道 |
| **P** | Promotion | 促销资源 |

**核心逻辑**：**客户 × 产品 × 渠道 × 促销资源 = 精准营销组合**

这是**4P营销理论**在电信客户经营中的落地应用！

### 1.2 定位

围绕客户全生命周期，构建**全场景、全流程、全触点**的数字智能化客户经营能力体系。

### 1.3 核心理念

```
洞察 → 定位 → 执行 → 评估 → 关怀 → 转化
```

**两大主线**：
- **营销数字化**：围绕客户经营智慧营销服务业务全流程
- **营服互促**：加强客户的长期关怀维系

## 二、系统架构

### 2.1 整体架构（四层）

```
┌─────────────────────────────────────────────────────────┐
│                    评估中心                               │
│   营销预评估 | 市场评估 | 工单运营分析 | 价值评估          │
├─────────────────────────────────────────────────────────┤
│              统一智能调度中心                             │
│   工单池 | 规则引擎 | 策略库 | 权益管理                   │
├─────────────────────────────────────────────────────────┤
│                CPCP核心能力                              │
│ ┌───────────┐ ┌───────────┐ ┌───────────┐ ┌───────────┐ │
│ │ C 全息    │ │ P 策略库  │ │ C 运营    │ │ P 权益    │ │
│ │ 客户视图  │ │           │ │ 协同      │ │ 营销资源  │ │
│ └───────────┘ └───────────┘ └───────────┘ └───────────┘ │
│ 客户画像 | 客户群洞察 | 标准化场景 | 统一协同 | 权益管理  │
├─────────────────────────────────────────────────────────┤
│                   大数据湖                               │
│      存储(hdfs/hive/es) + 计算(spark/Flink)             │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心模块

| 模块 | 功能 |
|------|------|
| **C - 全息客户视图** | 客户画像、客户群洞察、客户群统一管理 |
| **P - 策略库** | 策略规则库、智能策略编排、策略诊断 |
| **C - 运营协同** | 触点管理、统一协同派单、业务动作库 |
| **P - 权益营销资源** | 权益管理、促销资源、卡券资源 |

| 功能 | 目标 | 说明 |


**建设成效**：

**跨触点支持**：

**转派模式**：

## 四、策略库能力

### 4.1 建设背景

> 目前各类营销策略存在各个策划人员的脑子和本地表格里，缺乏统一标准管控、共享与复用。

### 4.2 建设目标

- 构建营销策略库
- 沉淀企业统一的标准化营销策略
- 实现策略智能调度与动态调整
- 满足**一户一案**运营

### 4.3 核心能力

| 能力 | 说明 |
|------|------|
| **策略运营沉淀** | 图形化场景策略编排 |
| **策略话术沉淀** | 自动汇聚历史话术，沉淀优秀话术 |
| **策略对外开放** | 标准化策略检索、引用、推荐API |

### 4.4 建设成效

| 指标 | 数值 |
|------|:----:|
| 场景数 | **28个** |
| 策略数 | **31条** |
| 活动引用 | **44个** |
| 诊断调用 | **9,956次** |
| 派单量 | **170,913次** |
