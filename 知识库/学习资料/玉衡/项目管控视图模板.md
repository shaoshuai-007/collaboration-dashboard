# 项目管控视图模板 - 建设类 V7.0

| 项目 | 内容 |
|------|------|

## 📊 模板结构

### 1. 总体视图

**用途**：项目组建设管控总视图，汇总各模块状态

**核心内容**：
- 序号、内容、汇报类型、更新频度、说明

---

### 2. 组织架构

**用途**：项目团队组织架构和职责分工

**核心字段**：
| 字段 | 说明 |
|------|------|
| 内外部 | 甲方/乙方 |
| 角色 | 职位名称 |
| 负责人 | 人员姓名 |
| 联系方式 | 电话/邮箱 |
| 职责描述 | 具体职责 |

---

### 3. 项目周报

**用途**：项目周报记录和跟踪

**核心字段**：
| 字段 | 说明 |
|------|------|
| 项目名称 | 项目全称 |
| 责任人 | 局方+乙方负责人 |
| 完成进度 | 百分比 |
| 偏差 | 正常/延迟/提前 |
| 项目总体进展 | 进度说明 |
| 本周完成情况 | 具体完成事项 |
| 下周工作计划 | 计划事项 |
| 问题与风险 | 待解决和协调事项 |

---

### 4. 里程碑计划

**用途**：项目里程碑监控

**核心字段**：
| 字段 | 说明 |
|------|------|
| 里程碑 | 关键节点名称 |
| 计划交付物 | 预期产出 |
| 完成标准 | 验收标准 |
| 计划完成时间 | 截止日期 |
| 进度% | 完成百分比 |
| 进度说明 | 当前状态 |
| 实际完成时间 | 实际日期 |

**使用规则**：
- 项目立项结束2周内制定完成
- 每周更新至少1次
- 每月至少1个里程碑（含初验、终验）

## 🎯 使用建议

### 项目启动阶段

1. **填写组织架构** → 明确团队职责
2. **制定里程碑计划** → 设定关键节点
3. **制定总体计划** → 任务分解

### 项目执行阶段

1. **每周更新周报** → 进度跟踪
2. **更新里程碑进度** → 节点监控
3. **更新各阶段计划** → 任务跟踪

### 项目监控阶段

1. **风险跟踪** → 识别和应对风险
2. **缺陷跟踪** → 问题管理
3. **外部依赖跟踪** → 协调外部资源

### 项目收尾阶段

1. **版本发布记录** → 交付物清单
2. **总结报告** → 经验沉淀

---

## ⚖️ 玉衡使用指南

作为项目经理，玉衡应：

1. **项目启动时**：
   - 基于此模板创建项目管控文件
   - 组织团队填写组织架构
   - 制定里程碑和总体计划

2. **项目执行中**：
   - 每周更新项目周报
   - 监控里程碑进度
   - 识别和跟踪风险

3. **项目交付时**：
   - 汇总版本发布记录
   - 整理项目文档
   - 编写项目总结
