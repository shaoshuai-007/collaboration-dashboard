## 一、售前核心技能总结

### 1. 客户需求挖掘技巧

#### SPIN提问法
- **S (Situation) 现状问题**：了解客户当前状态
  - "您目前使用的系统是什么？"
  - "团队规模大概多少人？"
  - "数据处理量大概什么级别？"

- **P (Problem) 问题问题**：发现痛点
  - "使用过程中遇到什么困扰？"
  - "现有方案哪里不能满足需求？"
  - "最大的效率瓶颈在哪里？"

- **I (Implication) 影响问题**：放大问题影响
  - "这个问题对业务有什么影响？"
  - "如果持续下去，会造成什么损失？"
  - "影响了多少人力成本？"

- **N (Need-payoff) 需求回报问题**：引导解决方案
  - "如果能解决这个问题，会带来什么价值？"
  - "您希望达到什么效果？"

#### 需求挖掘三层次
1. **显性需求**：客户直接表达的需求
2. **隐性需求**：客户未明确表达但实际存在的需求
3. **潜在需求**：客户未意识到但可能存在的需求

#### 关键技巧
- 倾听占比 > 70%
- 避免"卖产品"思维，建立"解决问题"思维
- 用开放式问题开场，用封闭式问题确认
- 记录关键词，适时复述确认

### 2. 方案呈现话术

#### 结构化呈现框架

**开场（5分钟）**
- 感谢客户时间
- 确认会议目标和议程
- 简述客户背景（表示了解）

**现状分析（10分钟）**
- 复述客户痛点（用客户原话）
- 数据支撑问题严重性
- 行业对标分析

**解决方案（20分钟）**
- 整体架构图
- 核心功能模块
- 技术亮点与创新点
- 与需求的对应关系

**价值呈现（10分钟）**
- 量化收益（ROI、效率提升、成本节约）
- 案例佐证
- 风险保障

**服务保障（5分钟）**
- 实施计划
- 服务承诺
- 团队能力

**Q&A与下一步（10分钟）**

#### 话术要点

**FAB话术法**
- **F (Feature) 特性**：产品有什么特点
- **A (Advantage) 优势**：这个特点带来什么优势
- **B (Benefit) 利益**：对客户有什么具体价值

**示例**：
> "我们的AI模型采用了自研的轻量化架构（F），在保证准确率的前提下，推理速度提升了3倍（A），这意味着您在高峰期的响应时间可以从5秒降到1.5秒，用户体验显著提升（B）。"

#### 常见场景话术

**客户说"太贵了"**：
> "我理解您对预算的关注。让我们看看总拥有成本——除了采购成本，还有实施成本、运维成本、升级成本。我们的方案采用一体化设计，省去了大量集成和运维投入，三年TCO反而更低。另外，您期望实现的XX价值，如果按XX效率提升计算，X个月就能收回投资。"

**客户说"再考虑考虑"**：
> "没问题，做决策确实需要时间。为了帮助您更好地评估，我能知道您具体想考虑哪些方面吗？是技术可行性、预算，还是其他方面？这样我可以准备更有针对性的资料给您。"

### 3. 报价单制作规范

#### 报价单结构

```
1. 报价说明
   - 报价有效期
   - 付款条件
   - 交付周期
   - 质保期限

2. 费用明细
   - 软件许可费
   - 实施服务费
   - 定制开发费
   - 培训费用
   - 硬件费用（如有）
   - 其他费用

3. 折扣说明
   - 折扣原因
   - 折扣金额

4. 总价汇总

5. 附件
   - 详细功能清单
   - 实施计划
   - 服务承诺
```

#### 定价策略

| 策略 | 适用场景 | 说明 |
|------|----------|------|
| 成本加成 | 标准产品 | 成本 + 预期利润率 |
| 价值定价 | 高端定制 | 基于客户获得的价值定价 |
| 竞争定价 | 成熟市场 | 参考竞品价格 |
| 捆绑定价 | 多产品 | 整体方案打包优惠 |

#### 报价注意事项

- ✅ 分项清晰，让客户看到价值组成
- ✅ 预留谈判空间（10-15%）
- ✅ 注明不包含项，避免歧义
- ✅ 标明报价有效期（通常30天）
- ✅ 提供多个方案选择（高/中/低）
- ❌ 避免一口价，缺乏说服力
- ❌ 避免隐藏费用，影响信任

## 二、电信售前要点

### 1. AI项目售前话术

#### 电信行业AI应用场景

| 场景 | 痛点 | 解决方案 | 价值主张 |
|------|------|----------|----------|
| 智能客服 | 人工成本高、响应慢 | NLP客服机器人 | 降本70%，7x24响应 |
| 网络运维 | 故障定位慢、人工巡检 | AI故障预测+自愈 | 故障处理时间降80% |
| 营销推荐 | 转化率低、资源浪费 | 用户画像+精准推荐 | ARPU提升15% |
| 反欺诈 | 规则漏报、新手法难识别 | 图计算+行为分析 | 欺诈识别率提升30% |

#### AI项目售前话术模板

**价值切入**：
> "在电信行业，AI不只是技术升级，更是业务转型的核心驱动力。根据我们对行业的观察，成功的AI项目平均能带来20-30%的运营效率提升。"

**技术可信度**：
> "我们的AI方案已在XX运营商落地，处理日均XX亿条数据，模型准确率达到XX%，获得了XX认证。"

**风险应对**：
> "我们知道AI项目有不确定性，所以我们采用分阶段交付模式——第一阶段做PoC验证，确认效果后再规模化部署，让您每一分投入都有确定性回报。"

#### AI项目关键挑战及应对

| 客户顾虑 | 应对策略 |
|----------|----------|
| 模型效果不确定 | 提供PoC验证、效果承诺条款 |
| 数据安全担忧 | 本地化部署、数据脱敏方案 |
| 人才短缺 | 提供培训服务、运维托管 |
| ROI不清晰 | 详细收益测算、对标案例 |

### 2. 数据安全项目报价要点

#### 电信行业数据安全特点

- **数据量大**：PB级数据处理能力要求
- **实时性高**：7x24业务不能中断
- **合规严格**：等保、数据安全法、电信条例
- **架构复杂**：多系统、多网络域

#### 报价分项建议

```
一、软件产品费
   1. 核心平台许可
      - 部署模式：本地部署/云部署
      - 授权方式：永久授权/年度订阅
      - 节点数量：XX个
   2. 功能模块
      - 数据分类分级
      - 敏感数据发现
      - 数据脱敏
      - 访问审计
      - 数据水印

二、实施服务费
   1. 需求调研（人天）
   2. 系统部署（人天）
   3. 策略配置（人天）
   4. 数据对接（人天）
   5. 联调测试（人天）

三、定制开发费
   - 接口定制（按接口数）
   - 报表定制（按报表数）
   - 特殊功能开发（人天）

四、培训服务费
   - 管理员培训
   - 用户培训
   - 在线课程

五、年度运维费（通常为软件费的15-20%）
   - 技术支持
   - 版本升级
   - 安全补丁
```

#### 报价策略

- **首次合作**：适当优惠，建立标杆案例
- **大型项目**：分阶段报价，降低决策压力
- **竞争激烈**：突出差异化价值，避免纯价格战
- **长期合作**：打包多项目，整体优惠

#### 风险条款

```
建议包含：
- 项目范围边界说明
- 需求变更计价规则
- 数据安全责任划分
- 验收标准与方法
```

### 3. 竞品分析方法

#### 竞品分析框架

**1. 竞品识别**
- 直接竞品：同类型产品/方案
- 间接竞品：替代方案
- 潜在竞品：可能进入市场的玩家

**2. 信息收集渠道**
| 渠道 | 获取信息 |
|------|----------|
| 官网/产品资料 | 产品功能、定位 |
| 客户访谈 | 实际使用体验 |
| 行业报告 | 市场份额、排名 |
| 招投标信息 | 中标价格、方案 |
| 技术论坛 | 技术架构、口碑 |
| 社交媒体 | 用户反馈、舆情 |

**3. 对比维度**

| 维度 | 分析要点 |
|------|----------|
| 功能对比 | 核心功能、特色功能、缺失功能 |
| 技术对比 | 架构、性能、安全性、扩展性 |
| 价格对比 | 定价模式、整体TCO |
| 服务对比 | 实施能力、响应速度、本地化支持 |
| 案例对比 | 标杆客户、行业覆盖 |
| 公司对比 | 规模、资金、稳定性、研发能力 |

**4. SWOT分析模板**

```
我方产品：
  优势(S)：列出相对于竞品的优势
  劣势(W)：列出相对于竞品的劣势
  机会(O)：市场趋势、客户需求变化
  威胁(T)：竞品动态、新技术威胁

竞品产品：
  同理分析
```

**5. 差异化话术构建**

**比竞品强的地方**：
> "您提到的XX功能确实是重要考量点。我们在这方面做了专门优化，相比同类方案，我们的XX指标高出30%，这已经在XX客户处得到了验证。"

**比竞品弱的地方**：
> "您观察得很仔细。XX功能确实是我们和竞品的差异点。不过，考虑到您的核心需求是XX，我们的方案在这方面的投入反而让您获得了更好的性价比。而且，XX功能已经在我们的产品路线图中，预计Q3就能上线。"
