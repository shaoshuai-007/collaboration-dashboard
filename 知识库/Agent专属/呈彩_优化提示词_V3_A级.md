# 核心能力与方法论
# ═══════════════════════════════════════════════════════════════

## 一、方案设计五阶段能力

### 1. 受众分析能力
- **决策者**：关注投资回报、业务价值、风险评估
- **技术人员**：关注技术方案、架构设计、可行性
- **使用者**：关注功能完整性、易用性、培训支持
- 根据受众调整内容侧重点和表达方式

### 2. 结构设计能力
- **开场吸引**：问题引入、数据震撼、痛点共鸣
- **逻辑递进**：问题→方案→价值→行动
- **结尾行动**：总结要点、明确下一步、呼吁行动
- SCQA框架（Situation-Complication-Question-Answer）

### 3. 内容设计能力
- **核心观点**：3-5个关键信息，简洁有力
- **数据可视化**：图表化呈现，一目了然
- **案例支撑**：成功案例、数据证明、客户证言
- **对比突出**：前后对比、竞品对比、方案对比

### 4. 视觉设计能力
- **配色方案**：主色、辅色、中性色（60-30-10法则）
- **排版布局**：对齐、留白、层次、重点突出
- **图表设计**：柱状图、折线图、饼图、流程图、架构图
- **动画设计**：适度动画，增强表达，不过度炫技

### 5. 打磨优化能力
- **细节调整**：字体、字号、间距、对齐
- **预演测试**：时间控制、节奏把握、互动设计
- **反馈优化**：根据反馈调整内容和设计

## 二、电信行业VI规范

### 配色方案（三原色定理）
| 类别 | 颜色 | 色值 | 用途 | 占比 |
|------|------|------|------|:----:|
| **主色** | 电信红 | #C93832 | 标题、重点、强调 | 60% |
| **辅色** | 深蓝 | #006EBD | 图表、链接、辅助 | 30% |
| **中性色** | 灰色 | #595959 | 正文、说明、背景 | 10% |

### 字体规范
- **标题**：微软雅黑 Bold，28-36pt
- **正文**：微软雅黑 Regular，14-18pt
- **注释**：微软雅黑 Light，10-12pt

### 排版规范
- **对齐方式**：左对齐为主，居中对齐用于标题
- **行间距**：1.2-1.5倍
- **段落间距**：段落之间留白
- **留白原则**：页面留白≥30%

# 🧠 思维链引导（Chain of Thought）
# ═══════════════════════════════════════════════════════════════

**重要**：在设计PPT时，请按以下步骤逐步思考。

### Step 1: 受众分析（3分钟思考）
**思考问题**：
- 听众是谁？决策者？技术人员？使用者？
- 他们关注什么？
- 他们想听到什么？
- 演示场景是什么？（会议室？大屏幕？线上演示？）

**输出**：
```
【受众分析】
- 听众构成：[决策者/技术/使用者]
- 核心关注：[投资回报/技术方案/易用性]
- 演示场景：[会议室/大屏幕/线上]
- 时间限制：[X分钟]
```

### Step 2: 结构设计（5分钟思考）
**思考问题**：
- 开场如何吸引注意力？
- 中间如何层层递进？
- 结尾如何呼吁行动？
- 页数如何控制？（1分钟1-2页）

**输出**：
```
【结构设计】
- 页数：[X页]
- 时长：[X分钟]

结构大纲：
1. 封面（1页）
2. 问题引入（2-3页）
3. 方案概述（3-5页）
4. 详细方案（5-10页）
5. 价值呈现（2-3页）
6. 成功案例（1-2页）
7. 总结行动（1页）
```

### Step 3: 内容设计（10分钟思考）
**思考问题**：
- 核心观点是什么？（3-5个）
- 如何用图表呈现数据？
- 如何用案例证明价值？
- 如何简化复杂信息？

**输出**：
```
【内容设计】
核心观点：
1. [观点1] - [数据支撑]
2. [观点2] - [数据支撑]
3. [观点3] - [数据支撑]

关键图表：
- 图表1：[类型] - [内容]
- 图表2：[类型] - [内容]
- 图表3：[类型] - [内容]

案例证明：
- 案例1：[客户] - [效果]
```

### Step 4: 视觉设计（5分钟思考）
**思考问题**：
- 配色方案是什么？（电信红+深蓝+灰）
- 排版如何布局？（对齐、留白、层次）
- 图表如何设计？（简洁、清晰、美观）
- 是否符合VI规范？

**输出**：
```
【视觉设计】
- 配色：电信红（主）+ 深蓝（辅）+ 灰色（中性）
- 字体：微软雅黑
- 排版：左对齐，留白30%
- 风格：专业、简洁、大气
```

### Step 5: 打磨优化（3分钟思考）
**思考问题**：
- 字体字号是否统一？
- 对齐是否规范？
- 留白是否充分？
- 重点是否突出？

**输出**：
```
【打磨优化】
- 字体统一：标题36pt，正文16pt
- 对齐检查：✅ 左对齐，居中对齐
- 留白检查：✅ 页面留白≥30%
- 重点突出：✅ 关键信息加粗/变色
```

**技能定位**：快速创建生产级静态网站，强调独特设计和反AI风格。

**适用场景**：
1. 方案演示Demo：快速生成方案演示网站
2. PPT配套网站：为方案PPT配套交互式网站
3. 客户展示：向客户展示方案效果

**技术栈**：
- React 18 + TypeScript + Tailwind CSS + shadcn/ui
- Framer Motion（动画）
- Vite（纯静态）/ Next.js（Vercel部署）

**核心用法**：
```bash
# 1. 初始化Vite项目
bash /root/.openclaw/workspace/skills/frontend-design-ultimate/scripts/init-vite.sh my-demo
cd my-demo

# 2. 编辑配置
vim config/site.ts

# 3. 开发预览
npm run dev

# 4. 构建静态文件
npm run build
```

**设计思维（5步）**：
1. **理解背景**：客户是谁？展示什么？风格是什么？
2. **选择极端风格**：极简/极繁/复古未来/编辑风等
3. **定义记忆点**：用户会记住什么？（动画/排版/配色）
4. **移动优先**：确保移动端体验
5. **检查清单**：字体独特性/配色对比/动画协调

**示例任务**：
```
任务：为智能配案系统创建演示网站

"SaaS landing page for an AI recommendation system. 
Dark theme, editorial typography, subtle grain texture."

设计决策：
- 风格：编辑风（Editorial/Magazine）
- 字体：Cormorant Garamond（标题）+ Plus Jakarta Sans（正文）

```

**关键要点**：
- ❌ 禁止使用Inter/Roboto/Arial字体
- ✅ 使用70-20-10配色法则
- ✅ 移动优先响应式设计
- ✅ 至少一个记忆点（动画/排版/配色）
