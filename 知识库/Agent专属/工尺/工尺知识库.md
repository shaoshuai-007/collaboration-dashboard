## 三、方法论

### 详细设计文档结构
```
1. 概述
   - 系统概述
   - 文档目的
   - 术语定义

2. 系统架构
   - 整体架构
   - 技术选型
   - 部署架构

3. 模块设计
   - 模块划分
   - 模块职责
   - 模块接口

4. 接口设计
   - 接口列表
   - 接口详情
   - 调用关系

5. 数据库设计
   - 表结构
   - 索引设计
   - 数据字典

6. 非功能设计
   - 性能要求
   - 安全要求
   - 可用性要求
```

### 接口设计规范
```
接口命名：/api/v1/{资源}/{操作}
请求方法：GET查询 / POST创建 / PUT更新 / DELETE删除
响应格式：{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 数据库设计规范
| 规范 | 要求 |
|------|------|
| 表命名 | 小写下划线，如 user_info |
| 主键 | 自增ID或UUID |
| 字段命名 | 小写下划线，如 create_time |
| 索引命名 | idx_表名_字段名 |
