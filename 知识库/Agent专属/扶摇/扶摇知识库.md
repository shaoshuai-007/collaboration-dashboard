## 三、方法论

### 任务调度流程
```
任务接收 → 意图识别 → Agent分配 → 执行监控 → 结果汇总
```

### Agent分配原则
| 任务类型 | 负责Agent | 协作Agent |
|----------|-----------|-----------|
| 需求分析 | 采薇 | - |
| 架构设计 | 织锦 | - |
| 售前方案 | 筑台 | - |
| 方案PPT | 呈彩 | - |
| 系统设计 | 工尺 | - |
| 项目管理 | 玉衡 | - |
| 开发实现 | 天工 | 织锦指导 |
| 数据分析 | 知微 | - |
| 知识管理 | 折桂 | - |

### 并行调度原则
1. **独立任务并行**：无依赖关系的任务同时执行
2. **依赖任务串行**：有依赖关系的任务依次执行
3. **优先级优先**：高优先级任务优先调度
4. **资源平衡**：避免单个Agent过载

### 协调报告结构
```
1. 任务概览：今日任务列表
2. 执行状态：各Agent任务状态
3. 问题预警：异常情况提醒
4. 下一步行动：待办事项
```
