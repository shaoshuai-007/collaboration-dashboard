# 步骤2：思维导图模板（左右结构版）

## 模板说明

**设计原则**：
- 采用左右结构，避免重叠
- 左侧：需求理解 + 痛点分析
- 右侧：解决方案 + 优先级评估
- 每个象限独立展示，清晰易读

**输出格式**：PNG图片

---

## 布局设计

```
┌─────────────────────────────────────────────────────────┐
│                    需求分析思维导图                        │
│                [项目名称] - [日期]                        │
├───────────────────────┬─────────────────────────────────┤
│   1. 需求理解          │    3. 解决方案想法               │
│   ├─ 1.1 核心需求      │    ├─ 3.1 方案1（P1）           │
│   ├─ 1.2 次要需求      │    ├─ 3.2 方案2（P1）           │
│   └─ 1.3 隐性需求      │    ├─ 3.3 方案3（P1）           │
│                       │    └─ 3.4 方案4（P2）           │
├───────────────────────┼─────────────────────────────────┤
│   2. 痛点分析          │    4. 优先级评估                 │
│   ├─ 2.1 效率痛点      │    ├─ 4.1 影响度评估            │
│   ├─ 2.2 准确性痛点    │    ├─ 4.2 可行性评估            │
│   └─ 2.3 成本痛点      │    └─ 4.3 综合评估结果          │
└───────────────────────┴─────────────────────────────────┘
```

---

## 内容要求

### 1. 需求理解（左上）

**1.1 核心需求（P1）**
- 列出3-5个核心功能需求
- 每个需求标注优先级（P1）
- 简要说明需求描述

**1.2 次要需求（P2）**
- 列出2-3个次要功能需求
- 每个需求标注优先级（P2）

**1.3 隐性需求**
- 列出2-3个隐性需求
- 如易用性、可扩展性、安全性等

---

### 2. 痛点分析（左下）

**2.1 效率痛点**
- 列出3-5个效率相关问题
- 标注影响程度（高/中/低）
- 说明根因

**2.2 准确性痛点**
- 列出3-5个准确性相关问题
- 标注影响程度（高/中/低）

**2.3 成本痛点**
- 列出2-3个成本相关问题
- 标注影响程度（高/中/低）

---

### 3. 解决方案想法（右上）

**每个方案包含**：
- 方案名称 + 优先级
- 技术方案简述
- 解决的痛点
- 预期效果
- 技术可行性（高/中/低）

**格式示例**：
```
3.1 RAG智能问答（P1）
    • RAG + 大模型 + 知识库
    • 解决：规则查询效率低
    • 预期：查询时间10分钟→30秒
    • 可行性：高
```

---

### 4. 优先级评估（右下）

**4.1 影响度评估**
- 评分标准说明
- 每个方案的影响度得分

**4.2 可行性评估**
- 评分标准说明
- 每个方案的可行性得分

**4.3 综合评估结果**
- 综合得分公式：影响度×0.6 + 可行性×0.4
- 最终P1/P2判定
- 优先级排序

---

## 视觉规范

**配色方案**：
- 标题：深蓝 #1E40AF
- 一级节点：蓝色 #3B82F6
- 二级节点：浅蓝 #60A5FA
- 内容：深灰 #1F2937
- 背景：白色 #FFFFFF
- 分隔线：灰色 #9CA3AF

**字体规范**：
- 标题：24pt 加粗
- 一级节点：18pt 加粗
- 二级节点：14pt
- 内容：12pt

**间距规范**：
- 左右分栏间距：50px
- 上下分栏间距：30px
- 节点间距：20px
- 内容行距：16px

## 质量检查

| 检查项 | 要求 | 分值 |
|--------|------|:----:|
| 需求理解完整 | 核心需求≥3个，次要需求≥2个 | 25 |
| 痛点分析深入 | 每类痛点≥3个，有影响程度 | 25 |
| 解决方案可行 | ≥3个方案，有技术方案和预期效果 | 20 |
| 优先级评估科学 | 影响度+可行性+综合评估 | 20 |
| 布局清晰无重叠 | 左右结构，无遮挡 | 10 |
| **总分** | **≥90分** | **100** |
