## 模板1：方案汇报模板

### 适用场景

- 方案汇报
- 项目提案
- 立项申请

### 结构框架

```
1. 封面页
   └── 方案标题 + 副标题 + 汇报人/日期

2. 目录页
   └── 一级目录，清晰展示汇报结构

3. 背景分析
   ├── 行业背景
   ├── 现状问题
   └── 需求分析

4. 方案概述
   ├── 方案目标
   ├── 核心价值
   └── 方案框架

5. 详细方案
   ├── 技术架构
   ├── 功能模块
   ├── 实施路径
   └── 预期效果

6. 案例支撑
   ├── 成功案例
   └── 效果数据

7. 实施计划
   ├── 阶段划分
   ├── 时间节点
   └── 资源需求

8. 投资回报
   ├── 投入预算
   └── 预期收益

9. 风险预案
   ├── 风险识别
   └── 应对措施

10. 总结页
    └── 核心观点重申 + 下一步行动
```

### 设计规范

| 要素 | 规范 |
|------|------|
| **配色** | 商务蓝+金或品牌色 |
| **字体** | 标题黑体，正文微软雅黑 |
| **标题** | 每页标题即结论 |
| **文字** | 每页不超过6行 |
| **图表** | 数据必图表化 |

## 模板2：竞品分析模板

### 适用场景

- 竞品对比分析
- 市场调研报告
- 差异化定位

### 结构框架

```
1. 封面页
   └── 竞品分析报告 + 分析对象

2. 分析概述
   ├── 分析目的
   ├── 分析维度
   └── 竞品选择

3. 竞品概览
   └── 各竞品基本信息对比表

4. 多维度对比
   ├── 产品功能对比
   ├── 技术架构对比
   ├── 价格策略对比
   ├── 市场份额对比
   └── 用户评价对比

5. SWOT分析
   ├── 我方优势/劣势
   └── 竞品机会/威胁

6. 差异化定位
   ├── 核心差异点
   ├── 竞争策略
   └── 话术支撑

7. 结论建议
   └── 总结 + 行动建议
```

### 设计规范

| 要素 | 规范 |
|------|------|
| **配色** | 对比色（如蓝vs橙） |
| **布局** | 对比页用左右/上下对比布局 |
| **表格** | 对比表用颜色区分各竞品 |
| **结论** | 每页给出明确对比结论 |

## 模板3：项目总结模板

### 适用场景

- 项目复盘
- 工作总结
- 阶段汇报

### 结构框架

```

   └── 项目名称 + 总结类型 + 时间

   ├── 项目背景

3. 完成情况
   ├── 目标达成率
   ├── 关键成果
   └── 里程碑回顾

   ├── 核心指标
   ├── 增长数据
   └── 效果对比

5. 工作回顾
   ├── 主要工作
   ├── 团队贡献
   └── 资源投入

6. 经验总结
   ├── 成功经验
   ├── 问题反思
   └── 改进建议

   ├── 后续工作
   └── 资源需求
```

### 设计规范

| 要素 | 规范 |
|------|------|
| **配色** | 稳重色调（深蓝/深灰） |
| **数据** | 用图表突出成果数据 |
| **时间轴** | 里程碑用时间轴呈现 |

```
   ├── 同比分析
   ├── 环比分析

   ├── 数据亮点
   ├── 问题发现
```

| 要素 | 规范 |
|------|------|

| **图表** | 每页一个核心图表 |

## 使用指南

### 模板选择流程

1. **明确场景** → 选择合适模板类型
2. **确定结构** → 根据模板框架填充内容
3. **设计风格** → 选择配色方案
4. **逐页设计** → 一页一主题
5. **检查统一** → 字体、配色、风格一致

### 南乔心法

> 模板是效率的加速器，不是创意的枷锁。
> 
> **选对模板，事半功倍；用对方法，质量翻倍。**
